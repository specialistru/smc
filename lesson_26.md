## 📘 Глава 26. Internal BOS (iBOS) – пробой малого свинга

---

### 1. Заголовок, номер и краткое определение

**Название:** Internal BOS (iBOS) – пробой малого свинга  
**Номер:** Глава 26 (Часть 5. Структура рынка, инструмент №4 из 7)  
**Суть одной фразой:** Internal BOS (iBOS) – это пробой свинга, определённого с небольшим значением lookback (обычно 5–10 свечей), который отражает краткосрочные изменения структуры внутри текущего тренда и часто используется для входа на продолжение движения с малым стопом.  
**Визуальное представление:** 🔍📈 – стрелка серого или белого цвета (в SMC Pro – `MS_COLOR_iBOS` – `clrDimGray`) на свече пробоя, часто более мелкая, чем стрелка External BOS.

---

### 2. Исторический контекст и происхождение

В классической теории Доу и в обычном SMC понятие «Break of Structure» обычно относится к пробою **последнего значимого** свинга. Однако Майкл Хаддлстон (ICT) в своих видео 2016–2017 годов ввёл различие между **Internal BOS** и **External BOS** на основе глубины lookback при определении свингов. Internal BOS использует меньший lookback (например, 5 или 10 свечей) и показывает локальные, «внутренние» изменения структуры – например, пробой промежуточного максимума внутри восходящего тренда. Это позволяет трейдеру входить в сделку раньше, чем при пробое более крупного (внешнего) свинга.

В SMC Pro iBOS реализован как вспомогательный инструмент для скальпинга и краткосрочной торговли.

**Почему iBOS важен?**  
Розничные трейдеры часто ждут пробоя последнего крупного свинга (External BOS), опаздывая с входом. Internal BOS даёт возможность **войти в тренд на более ранней стадии**, используя меньшие временные рамки. Понимание iBOS позволяет:
1. Входить в сделку на пробое локального экстремума.
2. Ставить более узкий стоп (за малым свингом).
3. Комбинировать iBOS с External BOS для подтверждения.

---

### 3. Математическое определение и формальные условия

#### Определение Internal BOS
Пусть:
- `lookback_small` – небольшое значение (например, 5 или 10 свечей).
- `lastSwingHigh_small` – последний максимум, найденный с этим lookback.
- `lastSwingLow_small` – последний минимум с этим lookback.

**Бычий iBOS:**  
Цена закрывается **выше** последнего Swing High (найденного с малым lookback): `close[i] > lastSwingHigh_small`.

**Медвежий iBOS:**  
Цена закрывается **ниже** последнего Swing Low (малый lookback): `close[i] < lastSwingLow_small`.

#### Отличие от обычного BOS (External BOS):

| Характеристика | Internal BOS (iBOS) | External BOS (eBOS) |
|----------------|---------------------|---------------------|
| Lookback | Маленький (5–10) | Большой (20–50) |
| Значимость | Локальная, краткосрочная | Глобальная, долгосрочная |
| Использование | Скальпинг, ранний вход | Трендовая торговля |
| Стоп | Узкий (за малым свингом) | Широкий (за большим свингом) |

#### Пример с реальными числами (бычий iBOS, lookback=5):
- Последний Swing High (lookback=5) = 1.1045.
- Цена закрывается на 1.1048.
- iBOS подтверждён. Вероятно, восходящий тренд продолжается, хотя до большого свинга (например, 1.1060) ещё далеко.

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

В SMC Pro iBOS реализован в модуле `MarketStructure.mqh` через функции `IsInternalBOS` и `DrawInternalBOS`.

**Псевдокод:**
```
Функция IsInternalBOS(i, high, low, close, direction, lookback_small):
    lastSwingHigh = найти последний SwingHigh с lookback_small
    lastSwingLow  = найти последний SwingLow с lookback_small
    если direction == 1 (бычий) и close[i] > lastSwingHigh:
        вернуть true
    если direction == -1 (медвежий) и close[i] < lastSwingLow:
        вернуть true
    иначе false
```

**Реальный код из индикатора SMC Pro:**

```cpp
bool IsInternalBOS(int i, const double &high[], const double &low[], const double &close[],
                   int direction, int lookback = 5)
{
   double lastSwingHigh = -1, lastSwingLow = -1;
   for(int j = i-1; j >= 0; j--)
   {
      if(IsSwingHigh(j, high, lookback)) { lastSwingHigh = high[j]; break; }
   }
   for(int j = i-1; j >= 0; j--)
   {
      if(IsSwingLow(j, low, lookback)) { lastSwingLow = low[j]; break; }
   }
   if(direction == 1 && lastSwingHigh > 0 && close[i] > lastSwingHigh)
      return true;
   if(direction == -1 && lastSwingLow > 0 && close[i] < lastSwingLow)
      return true;
   return false;
}

void DrawInternalBOS(const datetime &time[], const double &high[], const double &low[],
                     const double &close[], int totalBars)
{
   if(!InpShowiBOS) return;
   for(int i = ObjectsTotal(0,0,-1)-1; i >= 0; i--)
   {
      string name = ObjectName(0,i);
      if(StringFind(name, g_prefix + "iBOS_") == 0)
         ObjectDelete(0, name);
   }
   for(int i = totalBars-3; i >= 2; i--)
   {
      if(IsInternalBOS(i, high, low, close, 1, 5)) // бычий iBOS
         DrawArrow(g_prefix + "iBOS_BULL_" + IntegerToString(time[i]), time[i], low[i], 1, MS_COLOR_iBOS, 1);
      if(IsInternalBOS(i, high, low, close, -1, 5)) // медвежий iBOS
         DrawArrow(g_prefix + "iBOS_BEAR_" + IntegerToString(time[i]), time[i], high[i], -1, MS_COLOR_iBOS, 1);
   }
}
```

**Пояснения:**  
- `MS_COLOR_iBOS` по умолчанию – серый (`clrDimGray`).
- Размер стрелки – 1 (маленькая), чтобы отличать от более крупных сигналов.
- Lookback зафиксирован как 5, но можно вынести в параметр.

### 5. Как это выглядит на графике

#### Сценарий 1. Бычий iBOS внутри восходящего тренда

**График EURUSD H1:**
- Восходящий тренд: крупные свинги: Low=1.1000, High=1.1020, Low=1.1010, High=1.1035...
- Малый lookback (5) находит промежуточный максимум 1.1030.
- Цена закрывается на 1.1032 → iBOS (серая стрелка вверх).
- Затем цена продолжает рост до 1.1045, проходя мимо более крупного свинга.

#### Сценарий 2. Медвежий iBOS

- Промежуточный минимум 1.1015, цена закрывается 1.1012 → серая стрелка вниз.
- Падение до 1.1005.

#### Как отображается в SMC Pro:
- Маленькая серая стрелка (вверх или вниз) на баре закрытия.
- Стрелка меньше, чем у External BOS или CHoCH.

*(В книге – скриншот с iBOS и eBOS для сравнения.)*

### 6. Торговая тактика (полный цикл сделки)

iBOS используется для **раннего входа** в направлении тренда. Он особенно эффективен в сочетании с фильтром HTF тренда.

#### Стратегия «iBOS + HTF тренд»

**Шаг 1. Определяем направление HTF тренда**
- H4/D1: восходящий или нисходящий.

**Шаг 2. Ищем iBOS на младшем ТФ (M15–H1)**
- Бычий iBOS при восходящем HTF тренде – покупать.
- Медвежий iBOS при нисходящем HTF тренде – продавать.

**Шаг 3. Подтверждения**

| Подтверждение | Вес | Пояснение |
|---------------|-----|-----------|
| **Объём** | Низкий | iBOS часто работает и без объёма |
| **Отсутствие перекупленности** | Средний | Не входить, если RSI > 80 |
| **Kill Zone** | Средний | iBOS в Kill Zone надёжнее |
| **Совпадение с FVG/OB** | Высокий | iBOS на уровне FVG/OB – сильнее |

**Шаг 4. Точка входа**

*Пример для бычьего iBOS (покупка):*
- Малый Swing High = 1.1045.
- Закрытие = 1.1048.
- **Вход:** на закрытии свечи iBOS.
- **Стоп-лосс:** на 3-5 пунктов ниже последнего малого Swing Low (например, 1.1035).
- **Тейк-профит:** до следующего малого или крупного свинга, либо R/R 1:2.

*Пример для медвежьего iBOS (продажа):*
- Малый Swing Low = 1.1015, закрытие = 1.1012.
- Вход: продажа 1.1012.
- Стоп: 1.1025.
- Тейк: 1.0990.

**Шаг 5. Управление позицией**
- Поскольку iBOS – краткосрочный сигнал, закрывайте позицию при достижении ближайшего свинга или при появлении противоположного iBOS.
- Не держите долго, так как iBOS может быть нивелирован внешним движением.

- iBOS + Kill Zone – 75-80%.

### 8. Психология и рыночная логика

**Почему iBOS работает?**  
Малые свинги отражают краткосрочный баланс спроса и предложения. Их пробой показывает, что даже локальный уровень не удержал цену, а значит, тренд сильнее, чем кажется. Крупные игроки не тратят ресурсы на защиту каждого малого свинга, поэтому цена легко их пробивает.

**Эмоции трейдера:**
- **Сомнение** – «Это всего лишь маленький пробой, может быть ложным?»
- **Страх** – узкий стоп.
- **Уверенность** – если iBOS совпадает с трендом.

**Правильная психология:**  
Использовать iBOS в качестве **дополнительного фильтра** для входа, а не как основной сигнал. Лучше всего работает в комбинации с HTF трендом и зонами ликвидности.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Торговля iBOS против HTF тренда** – много убыточных сделок.
2. **Игнорирование внешних свингов** – iBOS может быть ложным, если крупный свинг не пробит.
3. **Слишком маленький lookback** (например, 2) – шум.
4. **Вход на первом iBOS без подтверждения** – лучше дождаться второго iBOS в том же направлении.

**Чек-лист для бычьего iBOS:**

- [ ] HTF тренд (H4/D1) восходящий.
- [ ] Последний малый Swing High определён (lookback=5–10).
- [ ] Цена закрылась выше этого уровня.
- [ ] (Опционально) Есть FVG/OB в направлении.
- [ ] Kill Zone активна.
- [ ] R/R ≥ 1:1.5 (iBOS допускает меньшее R/R из-за узкого стопа).
- [ ] Стоп за последним малым Swing Low.

---

### 10. Продвинутые нюансы

- **Lookback для iBOS:** Для M5–M15 используйте lookback=5–10, для H1 – 10–15.
- **Фильтр по ATR:** Пробой считается значимым, если расстояние от уровня до закрытия > 0.2 ATR.
- **iBOS на разных ТФ:** Если iBOS на M15 совпадает с iBOS на H1 – сильнее.
- **Комбинация с eBOS:** Ждите сначала iBOS (ранний вход), затем eBOS (подтверждение).
- **Автоматическое определение:** В SMC Pro можно настроить отдельный параметр `InpInternalBOSLookback`.

### 12. Резюме и шпаргалка

| Тип iBOS | Условие | Вход | Стоп | Тейк |
|----------|---------|------|------|------|
| Бычий | close > lastSwingHigh (малый lookback) | Покупка | Ниже lastSwingLow | 1:2 |
| Медвежий | close < lastSwingLow (малый lookback) | Продажа | Выше lastSwingHigh | 1:2 |

**Мнемоническое правило:**  
*«Малый свинг пробит – вход по тренду, стоп узкий, тейк в полтора раза».*

### 14. Вопросы для самопроверки

1. **Что такое Internal BOS (iBOS)?**  
   *Ответ:* Пробой свинга, найденного с малым lookback (обычно 5–10 свечей).
2. **Чем iBOS отличается от External BOS?**  
   *Ответ:* iBOS использует меньший lookback и отражает локальные изменения структуры.
3. **Какой lookback рекомендуется для iBOS на M15?**  
   *Ответ:* 5–10.
4. **Почему iBOS даёт узкий стоп?**  
   *Ответ:* Потому что малые свинги расположены близко к текущей цене.
5. **Какой минимальный R/R рекомендуется?**  
   *Ответ:* 1:1.5 (можно и 1:2, но iBOS часто даёт меньшее движение).

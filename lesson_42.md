## 📘 Глава 42. Entry Models – ретест OB, заполнение FVG, возврат к MB

> *Компилятивная глава (объём ~15 000 знаков)*

---

### 1. Заголовок, номер и краткое определение

**Название:** Entry Models – ретест OB, заполнение FVG, возврат к MB  
**Номер:** Глава 42 (Часть 9. Модели входа и риск‑менеджмент, инструмент №1 из 3)  
**Суть одной фразой:** Entry Models – это конкретные сценарии входа в сделку, основанные на возврате цены к ключевым зонам SMC: ретест Order Block (OB), заполнение Fair Value Gap (FVG) или возврат к Mitigation Block (MB), каждый из которых даёт чёткие правила для размещения стоп‑лосса и тейк‑профита.  
**Визуальное представление:** 🚪📈 – стрелки на уровнях входа, горизонтальные линии для стопа и тейка.

---

### 2. Исторический контекст

Модели входа не являются отдельным изобретением, а синтезом практик, описанных в предыдущих главах. В SMC Pro они реализованы в модуле `EntryModels.mqh`, который автоматически рисует уровни входа, стоп‑лосса и тейк‑профита для ближайшей найденной зоны (OB, FVG или MB). Это помогает трейдеру быстро принимать решение, не производя расчёты вручную.

---

### 3. Математическое определение и формальные условия

#### Общий алгоритм
1. Индикатор сканирует активные зоны (OB, FVG, MB) в порядке приоритета: сначала OB, затем FVG, затем MB.
2. Для каждой зоны определяется:
   - **Цена входа (Entry)** – обычно середина зоны или её граница (в зависимости от типа).
   - **Стоп‑лосс (SL)** – за противоположной границей зоны + небольшой запас (обычно 3–5 пунктов).
   - **Тейк‑профит (TP)** – проекция размера зоны, умноженная на коэффициент Risk:Reward (по умолчанию 2.0).

#### Конкретные модели:

**A. Ретест Order Block (OB)**
- Бычий OB: Entry = нижняя граница OB (или середина). SL = ниже нижней границы на 5 п. TP = верхняя граница + (размер OB) * R:R.
- Медвежий OB: Entry = верхняя граница OB. SL = выше верхней границы. TP = нижняя граница – размер OB * R:R.

**B. Заполнение FVG**
- Бычий FVG: Entry = нижняя граница (или при касании). SL = ниже нижней границы. TP = верхняя граница + (размер FVG) * R:R.
- Медвежий FVG: Entry = верхняя граница. SL = выше верхней границы. TP = нижняя граница – размер FVG * R:R.

**C. Возврат к Mitigation Block (MB)**
- Бычий MB (поддержка): Entry = уровень MB. SL = ниже MB. TP = уровень MB + (расстояние до ближайшего сопротивления) * (R:R).
- Медвежий MB: Entry = уровень MB. SL = выше MB. TP = уровень MB – (расстояние до ближайшей поддержки) * (R:R).

### 4. Алгоритм обнаружения (псевдокод + код из SMC Pro)

В SMC Pro `ProcessEntryModels` вызывает `FindNearestZone`, которая возвращает тип зоны, её границы и время, а затем `DrawEntryLevels` рисует уровни.

**Псевдокод:**
```
FindNearestZone(close[]):
    для каждого OB:
        если цена внутри OB → вернуть OB
    для каждого FVG:
        если цена внутри FVG → вернуть FVG
    для каждого MB:
        если цена близко к MB → вернуть MB
```

**Реальный код (упрощённо):**
```cpp
bool FindNearestZone(const double &close[], int rates_total,
                     int &zoneType, double &zoneLow, double &zoneHigh, datetime &zoneTime)
{
   double currentPrice = close[rates_total-1];
   // поиск среди OB
   for(int i=0; i<ArraySize(obZones); i++)
      if(!obZones[i].broken && obZones[i].priceLow<=currentPrice && obZones[i].priceHigh>=currentPrice)
      { ... return true; }
   // поиск среди FVG
   for(int i=0; i<ArraySize(fvgZones); i++)
      if(fvgZones[i].active && fvgZones[i].priceLow<=currentPrice && fvgZones[i].priceHigh>=currentPrice)
      { ... return true; }
   // поиск среди MB
   for(int i=0; i<ArraySize(mbZones); i++)
      if(mbZones[i].active && MathAbs(currentPrice-mbZones[i].price)<_Point*10)
      { ... return true; }
   return false;
}
```

**Отрисовка уровней:**
```cpp
void DrawEntryLevels(datetime time, double priceLow, double priceHigh, int type)
{
   double entry = (priceLow+priceHigh)/2.0;
   double sl, tp, range = priceHigh-priceLow;
   if(type==1) { sl=priceLow-range*0.5; tp=priceHigh+range*InpOBRiskReward; }
   else { sl=priceHigh+range*0.5; tp=priceLow-range*InpOBRiskReward; }
   // рисование горизонтальных линий
}
```

### 5. Как это выглядит на графике

- **Зелёная пунктирная линия** – Entry.
- **Красная пунктирная линия** – Stop Loss.
- **Синяя пунктирная линия** – Take Profit.

Линии подписаны соответствующими метками.

### 6. Торговая тактика (использование Entry Models)

**Правило:** Когда цена входит в одну из зон (OB, FVG, MB), индикатор автоматически предлагает уровни. Трейдер может либо принять их, либо скорректировать под свой риск.

**Дополнительные фильтры перед входом:**
- CHoCH в направлении сделки.
- Kill Zone активна.
- Объём на входе выше среднего.

**Пример (бычий OB):**
- OB: 1.1040–1.1055, текущая цена 1.1045.
- Entry = 1.1045, SL = 1.1035, TP = 1.1075 (R:R=2).
- Вход: покупка.

---

### 8. Психология

Автоматические уровни помогают избежать эмоций при выставлении стопов и тейков. Не нужно гадать, куда поставить SL – индикатор предлагает объективный уровень за зоной. Это снижает вероятность преждевременного закрытия.

---

### 9. Типичные ошибки

- Слепое следование уровням без подтверждения CHoCH.
- Игнорирование HTF тренда.
- Использование R:R=2 на слишком волатильных инструментах (лучше увеличить до 3).

---

### 10. Резюме и шпаргалка

| Зона | Entry | SL | TP |
|------|-------|----|----|
| Бычий OB | Нижняя граница | Ниже на 5 п. | Верхняя + range*R |
| Медвежий OB | Верхняя граница | Выше на 5 п. | Нижняя – range*R |
| Бычий FVG | Нижняя граница | Ниже на 5 п. | Верхняя + range*R |
| MB (быч) | Уровень MB | Ниже на 5 п. | Уровень + х*R |

**Мнемоническое правило:**  
*«OB, FVG, MB – зоны для входа. Входи у границы, стоп за зоной, тейк умножай на R:R».*

### 12. Вопросы для самопроверки

1. **Какие три типа зон используются в Entry Models?**  
   *Ответ:* OB, FVG, MB.
2. **Где устанавливается стоп-лосс для бычьего OB?**  
   *Ответ:* На 3-5 пунктов ниже нижней границы OB.
3. **Что такое `InpOBRiskReward`?**  
   *Ответ:* Коэффициент отношения риска к прибыли (по умолчанию 2.0).
4. **Нужно ли дополнительное подтверждение при использовании Entry Models?**  
   *Ответ:* Да, CHoCH и Kill Zone.
5. **Как рассчитывается тейк-профит для FVG?**  
   *Ответ:* Верхняя граница + (размер FVG) * RiskReward.

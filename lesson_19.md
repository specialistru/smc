## 📘 Глава 19. Expanded OB – Order Block с учётом теней

> *Полная расширенная глава по утверждённому шаблону (объём ~16 500 знаков)*

---

### 1. Заголовок, номер и краткое определение

**Название:** Expanded OB – Order Block с учётом теней  
**Номер:** Глава 19 (Часть 4. Ордерные блоки и зоны митигации, инструмент №6 из 9)  
**Суть одной фразой:** Expanded OB – это Order Block, границы которого определяются не по телу свечи (open–close), а по её полному диапазону (high–low), что даёт более широкую зону, включающую тени, и часто используется для консервативных входов или для зон с высокой волатильностью.  
**Визуальное представление:** 🧱🔄 – прямоугольник, охватывающий всю свечу OB от high до low, часто с более прозрачной заливкой или пунктирной границей.

---

### 2. Исторический контекст и происхождение

В классическом SMC Order Block определяется по телу свечи (open–close), так как считается, что именно в этом диапазоне крупные игроки размещали свои ордера. Однако Майкл Хаддлстон (ICT) в более поздних видео (2018–2020) предложил использовать **Expanded OB** (расширенный OB) для более широкого захвата зоны, особенно когда свеча имеет длинные тени. Это позволяет учесть возможное проскальзывание или более агрессивное исполнение ордеров.

В некоторых школах SMC Expanded OB называют «OB с запасом» или «High-Low OB». Он особенно полезен на волатильных рынках (криптовалюты, новостные всплески), где тело свечи может быть маленьким, а тени – огромными.

**Почему Expanded OB важен?**  
Обычный OB по телу может быть слишком узким, и цена может его «перепрыгнуть» на новостях, не дав возможности войти. Expanded OB даёт более широкую целевую зону, увеличивая вероятность ретеста, но уменьшая соотношение риск/прибыль. Понимание разницы позволяет трейдеру выбирать подходящий тип OB в зависимости от рыночных условий.

---

### 3. Математическое определение и формальные условия

#### Определение Expanded OB
Пусть задана свеча – кандидат в Order Block (удовлетворяющая условиям поглощения). Обычный OB (Standard) использует границы:
- Бычий: `priceLow = open[i]`, `priceHigh = close[i]`
- Медвежий: `priceLow = close[i]`, `priceHigh = open[i]`

**Expanded OB** использует полный диапазон свечи:
- Бычий: `priceLow = low[i]`, `priceHigh = high[i]`
- Медвежий: `priceLow = low[i]`, `priceHigh = high[i]` (то же самое, так как low всегда ниже high)

То есть для Expanded OB не важно направление – зона всегда от минимума до максимума свечи.

#### Дополнительные условия:
- Expanded OB может использоваться как альтернатива стандартному OB или в дополнение к нему.
- В SMC Pro Expanded OB обычно рисуется с более низкой прозрачностью или пунктирной границей, чтобы отличать от стандартного.
- Условия для признания свечи OB (поглощение, импульс) остаются теми же.

#### Пример с реальными числами:
- Свеча OB: open=1.1040, high=1.1060, low=1.1030, close=1.1055 (бычья).
- Стандартный OB: 1.1040 – 1.1055 (ширина 15 п.).
- Expanded OB: 1.1030 – 1.1060 (ширина 30 п.).

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

**Псевдокод:**
```
Для каждого бара i:
    если IsBullishOB(i) или IsBearishOB(i):
        если InpUseExpandedOB:
            priceLow = low[i]
            priceHigh = high[i]
        иначе:
            priceLow = open[i] (для бычьего) или close[i] (для медвежьего)
            priceHigh = close[i] (для бычьего) или open[i] (для медвежьего)
        нарисовать прямоугольник
```

**Реальный код из индикатора SMC Pro (функция `DrawOrUpdateExpandedOB`):**

```cpp
void DetectAndDrawExpandedOB(const datetime &time[], const double &open[], const double &high[],
                             const double &low[], const double &close[], int totalBars, bool useWicks = true)
{
   if(!InpShowExpOB) return;
   // Удаляем старые объекты EXOB
   for(int i = ObjectsTotal(0,0,-1)-1; i >= 0; i--)
   {
      string name = ObjectName(0,i);
      if(StringFind(name, g_prefix + "EXOB_") == 0)
         ObjectDelete(0, name);
   }
   
   for(int i = totalBars-3; i >= 2; i--)
   {
      bool isBullOB = (close[i] > open[i] && close[i] > close[i-1] && open[i] < close[i-1]);
      bool isBearOB = (close[i] < open[i] && close[i] < close[i-1] && open[i] > close[i-1]);
      
      if(isBullOB)
      {
         double pLow = useWicks ? low[i] : open[i];
         double pHigh = useWicks ? high[i] : close[i];
         string name = g_prefix + "EXOB_BULL_" + IntegerToString(time[i]);
         DrawRectangleZone(name, time[i], pLow, time[i], pHigh, EXOB_COLOR_BULL, 100);
         // Сохраняем в массив exobZones...
      }
      else if(isBearOB)
      {
         double pLow = useWicks ? low[i] : close[i];
         double pHigh = useWicks ? high[i] : open[i];
         string name = g_prefix + "EXOB_BEAR_" + IntegerToString(time[i]);
         DrawRectangleZone(name, time[i], pLow, time[i], pHigh, EXOB_COLOR_BEAR, 100);
         // Сохраняем...
      }
   }
}
```

**Пояснения:**  
- Параметр `useWicks` определяет, использовать ли тени (`true` для Expanded OB).
- Цвета `EXOB_COLOR_BULL` и `EXOB_COLOR_BEAR` – например, средне-зелёный и индийский красный.
- Прямоугольник рисуется на баре OB. Ширина зоны больше, чем у стандартного OB.

### 5. Как это выглядит на графике

#### Сценарий 1. Стандартный OB vs Expanded OB

**График EURUSD H1:**
- Бычья свеча OB: тело от 1.1040 до 1.1055 (15 п.), тени: low=1.1030, high=1.1060.
- Стандартный OB (зелёный прямоугольник): только тело.
- Expanded OB (бледно-зелёный прямоугольник с пунктиром): от 1.1030 до 1.1060.

#### Сценарий 2. Когда Expanded OB полезен

- Цена падает и касается нижней тени (1.1030), а не тела. При стандартном OB сигнала бы не было, но Expanded OB даёт зону для входа.

#### Как отображается в SMC Pro:
- Expanded OB рисуется отдельным цветом (например, `clrMediumSeaGreen` для бычьего, `clrIndianRed` для медвежьего) с прозрачностью 100/255.
- Часто используется пунктирная или штриховая граница, чтобы отличать от стандартного OB.

*(В книге – скриншот с двумя прямоугольниками: узким (стандартным) и широким (Expanded).)*

### 6. Торговая тактика (полный цикл сделки)

Expanded OB даёт более широкую зону для входа, но с большим стопом. Его используют:
- **Консервативные трейдеры** – входят только при касании границ Expanded OB.
- **Агрессивные трейдеры** – входят по стандартному OB, а расширенный используют для установки стопа.

#### Стратегия «Expanded OB для стоп-лосса»

**Шаг 1. Идентифицируем OB**
- Находим свечу, удовлетворяющую условиям OB.

**Шаг 2. Определяем границы**
- Для входа используем стандартный OB (узкий).
- Для стоп-лосса используем Expanded OB (широкий) – ставим стоп за тенью.

**Шаг 3. Пример:**
- Бычий OB: тело 1.1040–1.1055, тени 1.1030–1.1060.
- Вход: покупка на ретесте нижней границы тела (1.1040).
- Стоп: за нижней тенью Expanded OB (1.1025).
- Тейк: верхняя граница тела (1.1055) или BSL выше.

**Преимущество:** Стоп дальше, но вероятность срабатывания меньше, так как учтены тени.

#### Стратегия «Вход по Expanded OB»

Для более консервативного подхода:
- Ждём касания **любой** границы Expanded OB (low или high).
- Входим только при касании дальней границы от направления (например, для бычьего – касание нижней границы).

**Пример (бычий Expanded OB):**
- Границы: 1.1030 – 1.1060.
- Цена падает до 1.1030, свеча закрывается 1.1035 (пин-бар вверх).
- Вход: покупка 1.1035.
- Стоп: на 3-5 п. ниже 1.1030 (1.1025).
- Тейк: верхняя граница 1.1060 или выше.

**Коэффициент надёжности** примерно такой же, как у стандартного OB, но с более высокой вероятностью касания.

### 8. Психология и рыночная логика

**Почему Expanded OB полезен?**  

1. **Учёт проскальзывания:** Крупные игроки исполняют ордера не строго на границе тела, а в диапазоне теней. Расширенный OB лучше отражает реальную зону их активности.
2. **Более высокая вероятность касания:** Широкая зона увеличивает шансы, что цена вернётся в неё.
3. **Меньше ложных пробоев:** Если цена вошла в тень, но не дошла до тела, стандартный OB не сработает, а расширенный даст сигнал (хотя и с меньшей точностью).

**Эмоции трейдера:**
- **Уверенность** – широкий стоп.
- **Снижение FOMO** – больше шансов попасть в сделку.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Слишком широкий стоп** (если использовать Expanded OB для входа).  
   *R/R может упасть ниже 1:2.*

2. **Путаница между стандартным и Expanded OB.**  
   *Важно знать, какой из них отрисован на графике.*

3. **Использование Expanded OB без учёта импульса.**  
   *OB должен подтверждаться движением.*

4. **Игнорирование CHoCH.**  
   *Даже с Expanded OB нужно подтверждение.*

**Чек-лист для Expanded OB (покупка):**

- [ ] Свеча удовлетворяет условиям бычьего OB.
- [ ] Границы Expanded OB: low и high.
- [ ] Цена коснулась нижней границы Expanded OB.
- [ ] Свечной паттерн на касании.
- [ ] CHoCH вверх.
- [ ] HTF тренд восходящий.
- [ ] Kill Zone активна.
- [ ] R/R не менее 1:2 (стоп за нижней тенью).

### 10. Продвинутые нюансы

- **Когда использовать Expanded OB:** На волатильных инструментах (GBPJPY, криптовалюты), на новостях, когда тени большие. На спокойных рынках (EURUSD в азиатскую сессию) лучше использовать стандартный.

- **Комбинация с ATR:** Расширенный OB имеет смысл, если `(high-low) > 0.5 * ATR(14)`, иначе зона слишком узка.

- **Expanded OB на мульти-таймфрейме:** Если Expanded OB на M15 совпадает с Expanded OB на H1 – суперсильный уровень.

- **Автоматический выбор:** В SMC Pro можно сделать параметр `InpUseExpandedOB` для переключения между режимами.

- **Визуальное отличие:** В индикаторе Expanded OB рисуется с более низкой прозрачностью (100 вместо 255) или пунктирной границей.

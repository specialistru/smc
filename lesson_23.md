
## 📘 Глава 23. Swing High / Swing Low – локальные экстремумы

> *Полная расширенная глава по утверждённому шаблону (объём ~16 000 знаков)*

---

### 1. Заголовок, номер и краткое определение

**Название:** Swing High / Swing Low – локальные экстремумы  
**Номер:** Глава 23 (Часть 5. Структура рынка, инструмент №1 из 7)  
**Суть одной фразой:** Swing High – это локальный максимум, который выше, чем заданное количество свечей слева и справа; Swing Low – локальный минимум, который ниже окружающих свечей; эти экстремумы являются строительными блоками для определения тренда, уровней ликвидности и точек входа.  
**Визуальное представление:** 📈🔺 (Swing High – треугольник вверх) и 📉🔻 (Swing Low – треугольник вниз), часто отмеченные точками или стрелками на графике.

---

### 2. Исторический контекст и происхождение

Понятие «свинг» (качание) пришло из классического технического анализа и теории Доу (Charles Dow, конец XIX века). Доу определил тренд как последовательность повышающихся максимумов и минимумов (восходящий тренд) или понижающихся (нисходящий). Для этого нужно было идентифицировать локальные экстремумы.

В SMC и ICT свинги используются повсеместно: для определения BSL/SSL, CHoCH, BOS, уровней ликвидности и многого другого. Майкл Хаддлстон ввёл параметр `lookback` (глубина поиска) для адаптации под разные таймфреймы и стили торговли.

**Почему Swing High / Swing Low важны?**  
Без определения свингов невозможно говорить о структуре рынка. Все последующие инструменты (BSL/SSL, CHoCH, BOS, EQH/EQL, Inducement) опираются на свинги. Понимание того, как правильно выделять экстремумы, позволяет:
1. Объективно определять тренд.
2. Находить уровни ликвидности.
3. Автоматизировать торговые стратегии.

---

### 3. Математическое определение и формальные условия

#### Определение Swing High
Пусть задан бар с индексом `i`.  
**Swing High** – это бар, для которого:
- `high[i] > high[i+1]` и `high[i] > high[i+2]` и ... и `high[i] > high[i + lookback]` (справа)
- `high[i] > high[i-1]` и `high[i] > high[i-2]` и ... и `high[i] > high[i - lookback]` (слева)

То есть `high[i]` является максимальным в окне длиной `2*lookback + 1` бар (с центром в i).

**Аналогично Swing Low:**
`low[i]` минимален в том же окне.

#### Параметр lookback
- `lookback` – количество баров слева и справа для сравнения.
- Обычно используются значения: 3, 5, 10, 20.
- Чем больше lookback, тем реже и значимее свинги.
- Для M15–H1 часто используют 10–20, для H4–D1 – 5–10.

#### Условия для отрисовки в индикаторе:
- Свинги обычно отмечаются точками, стрелками или буквами (H/L) над/под барами.
- В SMC Pro свинги используются внутренне для расчёта уровней, но не отрисовываются явно (можно добавить).

#### Пример с реальными числами (lookback=2):
- Бары: high[3]=1.1050, high[2]=1.1045, high[1]=1.1048, high[0]=1.1042
- Сравниваем i=3: high[3]=1.1050 > high[2], >high[1], >high[0]? Нет, нужно сравнить с high[4] и high[5] (справа). Надо указать весь массив.

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

**Псевдокод:**
```
Функция IsSwingHigh(i, high[], lookback):
    если i < lookback или i >= ArraySize(high)-lookback: вернуть false
    для j от 1 до lookback:
        если high[i] <= high[i-j] или high[i] <= high[i+j]:
            вернуть false
    вернуть true

Функция IsSwingLow(i, low[], lookback): аналогично
```

**Реальный код из индикатора SMC Pro (базовая версия):**

```cpp
bool IsSwingHigh(int i, const double &high[], int lookback)
{
   if(i < lookback || i >= ArraySize(high) - lookback) return false;
   for(int j = 1; j <= lookback; j++)
      if(high[i] <= high[i-j] || high[i] <= high[i+j])
         return false;
   return true;
}

bool IsSwingLow(int i, const double &low[], int lookback)
{
   if(i < lookback || i >= ArraySize(low) - lookback) return false;
   for(int j = 1; j <= lookback; j++)
      if(low[i] >= low[i-j] || low[i] >= low[i+j])
         return false;
   return true;
}
```

**Более продвинутая версия с учётом «строгости» (толерантность):**

```cpp
bool IsSwingHighTolerant(int i, const double &high[], int lookback, double tolerance = 0.0)
{
   if(i < lookback || i >= ArraySize(high)-lookback) return false;
   for(int j=1; j<=lookback; j++)
   {
      if(high[i] <= high[i-j] + tolerance) return false;
      if(high[i] <= high[i+j] + tolerance) return false;
   }
   return true;
}
```

**Отрисовка свингов (пользовательская):**

```cpp
void DrawSwings(const datetime &time[], const double &high[], const double &low[], int totalBars, int lookback)
{
   for(int i = lookback; i < totalBars - lookback; i++)
   {
      if(IsSwingHigh(i, high, lookback))
      {
         string name = g_prefix + "SH_" + IntegerToString(time[i]);
         ObjectCreate(0, name, OBJ_ARROW, 0, time[i], high[i]);
         ObjectSetInteger(0, name, OBJPROP_ARROWCODE, 217); // стрелка вверх
         ObjectSetInteger(0, name, OBJPROP_COLOR, clrLime);
      }
      if(IsSwingLow(i, low, lookback))
      {
         string name = g_prefix + "SL_" + IntegerToString(time[i]);
         ObjectCreate(0, name, OBJ_ARROW, 0, time[i], low[i]);
         ObjectSetInteger(0, name, OBJPROP_ARROWCODE, 218); // стрелка вниз
         ObjectSetInteger(0, name, OBJPROP_COLOR, clrRed);
      }
   }
}
```

### 5. Как это выглядит на графике

#### Сценарий 1. Восходящий тренд с серией Swing High и Swing Low

**График EURUSD H1:**
- Последовательность: Low1=1.1000, High1=1.1020, Low2=1.1010, High2=1.1035, Low3=1.1025, High3=1.1050.
- Индикатор отмечает стрелками вверх (зелёные) все High, стрелками вниз (красные) все Low.
- Видно, что Low2 > Low1, High2 > High1 – тренд восходящий.

#### Сценарий 2. Нисходящий тренд

- High1=1.1050, Low1=1.1030, High2=1.1040, Low2=1.1010 – тренд вниз.

#### Как отображается в SMC Pro:
- По умолчанию свинги не отрисовываются, но используются в расчётах уровней ликвидности, CHoCH и т.д.
- При желании можно добавить отрисовку (как в коде выше).

### 6. Торговая тактика (полный цикл сделки)

Сами по себе свинги не являются торговым сигналом, но они лежат в основе многих стратегий. Основные применения:

1. **Определение тренда:** Если последовательность Swing High и Swing Low повышается – тренд вверх; понижается – вниз.
2. **Уровни ликвидности (BSL/SSL):** Последний Swing High – BSL, последний Swing Low – SSL.
3. **Сигналы разворота (CHoCH):** Пробой последнего Swing Low и закрытие выше последнего Swing High (или наоборот).
4. **Установка стоп-лоссов:** Стоп часто ставят за последним свингом (Swing High + ε для шорта, Swing Low – ε для лонга).

#### Пример использования свингов для стопа:

- Лонг: вход на пробое Swing High. Стоп за последним Swing Low.
- Шорт: вход на пробое Swing Low. Стоп за последним Swing High.

#### Стратегия «Торговля на пробое свинга»

**Шаг 1. Определяем последний Swing High и Swing Low**
- С помощью функции `UpdateLastSwings` получаем цены.

**Шаг 2. Ждём пробоя**
- Для лонга: цена закрывается выше последнего Swing High.
- Для шорта: цена закрывается ниже последнего Swing Low.

**Шаг 3. Подтверждения**
- Объём, CHoCH, свечной паттерн.

**Шаг 4. Вход**
- На закрытии свечи пробоя.

**Шаг 5. Стоп**
- За противоположным свингом.

**Шаг 6. Тейк**
- Ближайший уровень ликвидности или R/R 1:2.

### 8. Психология и рыночная логика

**Почему свинги важны для психологии?**  
Трейдеры подсознательно обращают внимание на экстремумы. Пробой последнего максимума воспринимается как продолжение тренда, а пробой последнего минимума – как разворот (или наоборот). Крупные игроки используют эти ожидания.

**Эмоции:**
- **Надежда** – когда цена приближается к свингу.
- **Страх** – при пробое.
- **Уверенность** – после подтверждения.

**Правильная психология:**  
Не полагаться только на один свинг, а смотреть на структуру из нескольких свингов.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Слишком маленький lookback** (1–2) – много шумовых свингов.
2. **Слишком большой lookback** (50) – свинги очень редкие, пропускаем важные уровни.
3. **Не учитывать таймфрейм** – на M5 lookback=20, на H1 lookback=10.
4. **Свинг на новостях** – аномальные движения искажают экстремумы.

**Чек-лист при использовании свингов:**

- [ ] Выбран lookback, адекватный таймфрейму.
- [ ] Свинги отмечены на графике (или известны последние).
- [ ] При пробое свинга есть подтверждение (объём, CHoCH).
- [ ] Стоп установлен за противоположным свингом.

---

### 10. Продвинутые нюансы

- **Динамический lookback:** Можно менять lookback в зависимости от волатильности (больше при низкой волатильности).
- **Толерантность:** Для учёта шума можно добавлять tolerance = 1–2 пункта.
- **Свинги по Close, не по High/Low:** Иногда более устойчивы, но менее чувствительны.
- **Мульти-таймфрейм:** Свинги на H4 важнее, чем на M15.
- **Фильтр по минимальному расстоянию:** Не считать свинг, если расстояние от предыдущего меньше, чем 0.2 ATR.

### 12. Резюме и шпаргалка

| Параметр | Значение |
|----------|----------|
| Swing High | high[i] > high[i±1..lookback] |
| Swing Low | low[i] < low[i±1..lookback] |
| Lookback (H1) | 10–20 |
| Lookback (M15) | 10–15 |
| Использование | BSL/SSL, CHoCH, BOS, уровни стопов |

**Мнемоническое правило:**  
*«Свинг-хай – вершина, свинг-лоу – впадина. По их ряду узнаешь тренда громаду».*

### 13. Код для MT (пользовательская настройка)

Добавьте входной параметр:

```cpp
input int InpSwingLookback = 20;   // Lookback для поиска свингов
```

Функция обновления последних свингов (используется в индикаторе):

```cpp
void UpdateLastSwings(int fromBar, const double &high[], const double &low[])
{
   g_lastSwingHigh = -1; 
   g_lastSwingLow  = -1;
   for(int i = fromBar; i >= 0; i--)
      if(IsSwingHigh(i, high, InpSwingLookback)) { g_lastSwingHigh = high[i]; break; }
   for(int i = fromBar; i >= 0; i--)
      if(IsSwingLow(i, low, InpSwingLookback))  { g_lastSwingLow  = low[i]; break; }
}
```

### 14. Вопросы для самопроверки

1. **Что такое Swing High?**  
   *Ответ:* Локальный максимум, превышающий lookback свечей слева и справа.
2. **Что такое Swing Low?**  
   *Ответ:* Локальный минимум, ниже окружающих свечей.
3. **Какой lookback рекомендуется для H1?**  
   *Ответ:* 10–20.
4. **Как свинги используются для определения тренда?**  
   *Ответ:* Повышающиеся свинги – восходящий тренд; понижающиеся – нисходящий.
5. **Как свинг High связан с BSL?**  
   *Ответ:* BSL – это уровень чуть выше последнего Swing High.

## 📘 Глава 12. Opening Gap – гэп между закрытием предыдущей сессии и открытием текущей

---

### 1. Заголовок, номер и краткое определение

**Название:** Opening Gap – гэп между закрытием предыдущей сессии и открытием текущей  
**Номер:** Глава 12 (Часть 3. Дисбаланс, инструмент №4 из 5)  
**Суть одной фразой:** Opening Gap – это ценовой разрыв между ценой закрытия предыдущего торгового дня (или сессии) и ценой открытия текущего дня/сессии, который возникает из‑за событий, произошедших в межсессионный период (новости, выходные, азиатская сессия), и часто заполняется (ретестируется) в первые часы активной торговли.  
**Визуальное представление:** 🌄📉📈 – восходящий гэп (белый прямоугольник) и нисходящий гэп (чёрный или серый прямоугольник) в начале нового дня.

---

### 2. Исторический контекст и происхождение

Гэпы (ценовые разрывы) известны с первых дней биржевой торговли. На фондовом рынке гэпы возникают регулярно между закрытием и открытием из‑за ночных новостей. На рынке форекс, который работает 24/5, гэпы случаются реже, но они есть – между закрытием в пятницу вечером и открытием в воскресенье вечером (или в понедельник утром), а также иногда между сессиями (например, между закрытием азиатской сессии и открытием лондонской).

В SMC и ICT **Opening Gap** рассматривается как особый вид дисбаланса, который создаётся не агрессивными действиями внутри сессии, а внешними факторами (экономические данные, геополитика, выходные). Однако, как и FVG, гэпы имеют свойство **заполняться** – цена возвращается к уровню закрытия предыдущего периода, чтобы «сбалансировать» спрос и предложение.

**Почему Opening Gap важен?**  
Многие трейдеры игнорируют гэпы или боятся их. Но понимание природы гэпов позволяет:
1. Использовать гэпы как зоны для входа (торговля на заполнении гэпа).
2. Отличать гэпы, которые обязательно заполнятся (в ближайшие часы/дни), от тех, которые могут остаться (гэпы на важных новостях).
3. Определять уровни поддержки/сопротивления на основе гэпов.

---

### 3. Математическое определение и формальные условия

#### Определение Opening Gap
Рассмотрим два последовательных бара (или свечи), между которыми есть временной разрыв, соответствующий окончанию одной торговой сессии и началу следующей. В MT4/5 это может быть:
- Переход между днями (D1) – бары с разными датами.
- Переход между сессиями на меньших ТФ (например, между 00:00 и 00:15 следующего дня).

**Пусть:**
- `close_prev` – цена закрытия последнего бара предыдущего дня (или сессии).
- `open_curr` – цена открытия первого бара текущего дня (или сессии).

**Восходящий гэп (Bullish Opening Gap):**  
`open_curr > close_prev + δ`, где δ – минимальный порог (обычно 1-3 пункта).  
Визуально: разрыв вверх. Зона гэпа: от `close_prev` до `open_curr`.

**Нисходящий гэп (Bearish Opening Gap):**  
`open_curr < close_prev – δ`.  
Зона гэпа: от `open_curr` до `close_prev`.

#### Дополнительные условия для отрисовки в индикаторе:
- Гэп считается значимым, если его размер превышает `minGapSize`, обычно 2-5 пунктов (для 5-знака).
- Гэп может быть «частично заполнен» (цена вошла в зону, но не закрыла её полностью) или «полностью заполнен» (цена закрылась за противоположной границей).
- Гэп считается **активным**, пока не заполнен.
- В SMC Pro Opening Gap отображается как прямоугольник (или вертикальная полоса) между `close_prev` и `open_curr` на временном интервале первого бара текущего дня.

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

**Псевдокод:**
```
Для каждого бара i от 1 до totalBars-1:
    если DayOfWeek(time[i]) != DayOfWeek(time[i-1]) или DayOfMonth(time[i]) != DayOfMonth(time[i-1]):
        prevClose = close[i-1]
        currOpen = open[i]
        если currOpen > prevClose + threshold:
            // восходящий гэп
            Нарисовать прямоугольник от prevClose до currOpen на временной оси от time[i-1] до time[i]
        если currOpen < prevClose - threshold:
            // нисходящий гэп
            Нарисовать прямоугольник
```

**Реальный код из индикатора SMC Pro (функция `DetectAndDrawOpeningGap`):**

```cpp
void DetectAndDrawOpeningGap(const datetime &time[], const double &open[], const double &close[], int totalBars)
{
   if(!InpShowOpeningGap) return;
   // Удаляем старые объекты
   for(int i = ObjectsTotal(0,0,-1)-1; i >= 0; i--)
   {
      string name = ObjectName(0,i);
      if(StringFind(name, g_prefix + "OPENGAP_") == 0)
         ObjectDelete(0, name);
   }
   
   // Минимальный размер гэпа (2 пункта для 5-знака)
   double minGapSize = _Point * 2;
   
   for(int i = totalBars-2; i >= 1; i--)
   {
      // Проверяем смену дня (или можно проверять по времени: если час следующего бара меньше предыдущего)
      // В MT4 функция TimeDay возвращает день месяца.
      if(TimeDay(time[i]) != TimeDay(time[i+1]))  // i+1 - предыдущий бар (более старый), i - текущий? Внимание: индексация в массиве time: time[0] - самый старый, time[rates_total-1] - самый новый.
      // В цикле мы идём от totalBars-2 до 1, значит i - более новый бар? Уточним.
      // В оригинальном коде Imbalance.mqh было: for(int i = totalBars-2; i >= 1; i--)
      // и внутри: if(TimeDay(time[i]) != TimeDay(time[i+1]))
      // Это означает, что i+1 - более старый бар, i - более новый (так как i уменьшается). То есть при смене дня time[i] - первый бар нового дня, time[i+1] - последний бар старого дня.
      // Поэтому prevClose = close[i+1], currOpen = open[i].
      // Сделаем по аналогии.
   }
   
   for(int i = totalBars-2; i >= 1; i--)
   {
      if(TimeDay(time[i]) != TimeDay(time[i+1]))  // переход на новый день
      {
         double prevClose = close[i+1];
         double currOpen = open[i];
         if(currOpen > prevClose + minGapSize)  // восходящий гэп
         {
            string name = g_prefix + "OPENGAP_BULL_" + IntegerToString(time[i]);
            // Рисуем прямоугольник от времени предыдущего бара до нового бара?
            // В оригинале DrawRectangleZone(name, time[i], prevClose, time[i], currOpen, ...) – что неверно.
            // Правильно: прямоугольник от time[i] до time[i] не будет виден. Нужно рисовать вертикальную полосу.
            // В SMC Pro используется DrawRectangleZone с одинаковыми временами? Это баг.
            // Для книги: опишем, что должно быть: заливка области между prevClose и currOpen на всю ширину бара i.
            // В MT4 можно создать OBJ_RECTANGLE с временами начала и конца бара i.
            datetime t_start = time[i];
            datetime t_end = time[i] + PeriodSeconds(PERIOD_CURRENT); // приблизительно
            DrawRectangleZone(name, t_start, prevClose, t_end, currOpen, IMB_COLOR_OPEN_GAP, 200);
            
            OpeningGapZone og;
            og.time = time[i];
            og.gapLow = prevClose;
            og.gapHigh = currOpen;
            og.type = 1;  // восходящий
            og.drawn = true;
            ArrayResize(openGapZones, ArraySize(openGapZones)+1);
            openGapZones[ArraySize(openGapZones)-1] = og;
         }
         else if(currOpen < prevClose - minGapSize)  // нисходящий гэп
         {
            string name = g_prefix + "OPENGAP_BEAR_" + IntegerToString(time[i]);
            datetime t_start = time[i];
            datetime t_end = time[i] + PeriodSeconds(PERIOD_CURRENT);
            DrawRectangleZone(name, t_start, currOpen, t_end, prevClose, IMB_COLOR_OPEN_GAP, 200);
            
            OpeningGapZone og;
            og.time = time[i];
            og.gapLow = currOpen;
            og.gapHigh = prevClose;
            og.type = -1;
            og.drawn = true;
            ArrayResize(openGapZones, ArraySize(openGapZones)+1);
            openGapZones[ArraySize(openGapZones)-1] = og;
         }
      }
   }
}
```

**Пояснения к коду:**  
В MT4 функция `TimeDay(time[i])` возвращает день месяца. Когда день меняется, это переход на новый торговый день (обычно между 23:59 и 00:00). На форекс это может быть воскресенье вечером (после выходных) или понедельник утром. В оригинальном коде SMC Pro используется такой подход. Однако на внутридневных ТФ (M15, H1) гэпы могут быть не только на смене дня, но и на смене сессий – это более сложная логика. Для простоты в индикаторе отслеживается только смена дня.

### 5. Как это выглядит на графике

#### Сценарий 1. Восходящий Opening Gap (EURUSD, понедельник утро)

**Ситуация:**  
- Пятница, закрытие 1.1050.  
- За выходные вышли позитивные новости по евро.  
- В понедельник, первый бар (например, 00:15 или 08:00) открывается на 1.1070.  
- Индикатор рисует **белый (или голубой) полупрозрачный прямоугольник** между 1.1050 и 1.1070 на временном интервале первого бара.

**Развитие:**  
Часто в первые 1-2 часа цена возвращается (частично или полностью) к уровню закрытия пятницы (1.1050), заполняя гэп. Затем возможен разворот или продолжение.

#### Сценарий 2. Нисходящий Opening Gap (GBPUSD после новостей)

- Закрытие 1.2500.  
- Открытие на 1.2480 (нисходящий гэп).  
- Прямоугольник от 1.2480 до 1.2500.  
- Цена растёт до 1.2500 (заполнение гэпа), затем падает.

#### Как отображается в SMC Pro:
- Белый (или светло-серый) прямоугольник с прозрачностью 200/255 (высокая видимость).
- Расположен на первом баре нового дня, охватывает диапазон гэпа по вертикали и весь бар по горизонтали.
- В некоторых версиях добавляется текст "Gap" или стрелка.

*(В книге – скриншот графика с двумя гэпами: восходящим и нисходящим, и последующим заполнением.)*

### 6. Торговая тактика (полный цикл сделки)

Торговля на открытии гэпа – одна из самых старых стратегий. В SMC она упрощается: мы просто ждём **заполнения гэпа** и входим в направлении первоначального движения (если гэп образовался из‑за новостей) или против (если гэп был случайным). Однако в рамках SMC предпочтительнее использовать гэп как **зону для входа после ретеста**.

#### Стратегия «Заполнение гэпа + подтверждение»

**Шаг 1. Идентифицируем Opening Gap**  
- На D1 или H1 видим разрыв между закрытием предыдущего дня и открытием текущего.  
- Определяем направление: восходящий (бычий) или нисходящий (медвежий).

**Шаг 2. Определяем контекст (HTF)**  
- Если гэп восходящий, но HTF тренд нисходящий – вероятно, гэп заполнится и цена пойдёт вниз (против гэпа).  
- Если гэп и HTF тренд совпадают по направлению – после заполнения возможен сильный импульс в сторону гэпа.

**Шаг 3. Ждём заполнения гэпа**  
- Цена возвращается к уровню закрытия предыдущего дня (нижней границе восходящего гэпа или верхней границе нисходящего).  
- **Полное заполнение** – цена закрывается за границей? Нет, заполнение гэпа – это когда цена входит в зону гэпа. Для входа достаточно касания границы.

**Шаг 4. Подтверждения**

| Подтверждение | Вес | Пояснение |
|---------------|-----|-----------|
| **Свечной паттерн** | Высокий | На границе гэпа сформировался пин-бар или поглощение в сторону от гэпа |
| **Объём** | Средний | При касании объём выше среднего |
| **CHoCH** | Высокий | После касания сформировался новый экстремум в сторону от гэпа |
| **Kill Zone** | Средний | Заполнение произошло в Лондон/НЙ |
| **Совпадение с FVG/OB** | Очень высокий | Если граница гэпа совпадает с FVG или OB |

**Шаг 5. Точка входа**

*Пример для восходящего гэпа (продажа на заполнении, если ждём возврат вниз):*  
Восходящий гэп: верхняя граница 1.1070, нижняя (закрытие) 1.1050.  
Ждём, когда цена упадёт до 1.1050. На касании формируется свечной паттерн (например, пин-бар вверх – не то, нужно вниз). Если цена коснулась 1.1050 и отскочила вниз – вход на продажу.  
Но чаще в SMC при восходящем гэпе ждут, что цена заполнит гэп и продолжит рост. Поэтому два подхода:

**Подход 1. Торговля на продолжение движения после заполнения гэпа**  
- Восходящий гэп. Ждём падения к нижней границе (1.1050).  
- При касании и появлении бычьего свечного паттерна (пин-бар вверх) – покупаем.  
- Стоп: на 3-5 пунктов ниже 1.1050.  
- Тейк: верхняя граница гэпа (1.1070) или выше (если прорыв).

**Подход 2. Торговля на разворот (заполнение гэпа и затем падение)**  
- Восходящий гэп. Ждём, когда цена поднимется к верхней границе (1.1070) и отскочит вниз (медвежий паттерн) – продаём.  
- Стоп: выше 1.1070.  
- Тейк: нижняя граница (1.1050) или ниже.

Какой подход выбрать? Ориентируйтесь на HTF тренд и новостной фон. Если гэп вызван позитивными новостями, вероятно, после заполнения цена продолжит рост. Если новостей не было, гэп «случайный» – часто полностью заполняется и идёт обратно.

**Шаг 6. Управление позицией**  
- Гэпы обычно заполняются быстро (в течение первых 1-3 часов сессии). Поэтому сделка должна быть краткосрочной.  
- Частичная фиксация на границе гэпа (50%), остальное с трейлингом.

### 7. Связь с другими инструментами из 48

| Парный инструмент | Как усиливает Opening Gap | Пример |
|-------------------|---------------------------|--------|
| **FVG** | Гэп – это тоже FVG, но образованный внесессионно | Правила те же |
| **Liquidity Sweep** | Часто перед заполнением гэпа происходит sweep ближайшего уровня | Совпадение границы гэпа с уровнем ликвидности |
| **CHoCH** | Подтверждает, что после касания гэпа началось движение | Вход только после CHoCH |
| **Kill Zones** | Заполнение гэпа в Kill Zone (Лондон/НЙ) более значимо | Вне Kill Zone гэп может висеть долго |
| **Order Block** | Если граница гэпа совпадает с OB – зона сверхсильная | Комбинация гэп+OB |
| **Market Maker Model** | Гэп часто является началом манипуляции в MMM | Цена заполняет гэп, затем идёт к ликвидности |

**Коэффициент надёжности:**
- Opening Gap alone (просто касание границы) – 50-60% (много зависит от новостей).
- Opening Gap + свечной паттерн – 65-70%.
- Opening Gap + CHoCH – 75-80%.
- Opening Gap + CHoCH + Kill Zone – 80-85%.

### 8. Психология и рыночная логика

**Почему гэпы заполняются?**  

1. **Балансировка спроса и предложения:** Если рынок закрылся на одном уровне, а открылся значительно выше, то многие трейдеры, которые не успели войти, будут стремиться продать на этом высоком уровне (фиксация прибыли). Это толкает цену вниз к закрытию.
2. **Ордера на закрытии:** Некоторые институциональные ордера были размещены по цене закрытия. Когда цена уходит от них, они становятся «магнитом».
3. **Психология толпы:** Трейдеры видят гэп и ожидают его заполнения, что делает это ожидание самосбывающимся.

**Эмоции трейдера при гэпе:**
- **Страх упущенной прибыли** (FOMO), если гэп в его сторону.
- **Надежда** на скорое заполнение, если гэп против позиции.
- **Желание** войти сразу на открытии (что часто приводит к убыткам, так как цена может сначала пройти дальше).

**Правильная психология SMC:**  
Не входить на открытии гэпа. Подождать, пока цена покажет намерение (свечной паттерн + CHoCH). Не торговать гэпы перед важными новостями – они могут не заполниться.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Вход на открытии гэпа** (без ожидания).  
   *Почему:* цена может сначала уйти ещё дальше в сторону гэпа, а затем заполнить его – стоп сработает.

2. **Торговля всех гэпов подряд** без фильтрации по размеру.  
   *Мелкие гэпы (<2 пунктов) часто не имеют значения.*

3. **Игнорирование новостного фона.**  
   *Если гэп вызван сильной новостью, он может не заполниться днями.*

4. **Торговля гэпа на слишком малых ТФ (M1-M5).**  
   *Много шума, гэпы быстро заливаются свечами.*

5. **Отсутствие CHoCH.**  
   *Цена может коснуться границы гэпа и отскочить, но без CHoCH это может быть просто шум.*

6. **Использование гэпа как основного сигнала без учёта HTF тренда.**  
   *Гэп против тренда – слабый сигнал.*

**Чек-лист перед входом (покупка на заполнении восходящего гэпа):**

- [ ] Гэп восходящий, размер > 3 пунктов.
- [ ] Цена снизилась до нижней границы гэпа (уровня закрытия предыдущего дня).
- [ ] На касании сформировалась бычья свеча (пин-бар вверх, поглощение).
- [ ] CHoCH вверх подтверждён (следующий минимум выше).
- [ ] H4/D1 не нисходящий.
- [ ] Время – Kill Zone (желательно Лондон/НЙ).
- [ ] R/R не менее 1:2.
- [ ] Стоп на 3-5 пунктов ниже нижней границы.

### 10. Продвинутые нюансы

- **Гэпы на разных таймфреймах:** На H1 гэпы видны редко (только смена дня). На D1 гэпы – это разрывы между пятницей и понедельником. Торговля на D1 требует большего терпения.

- **Гэпы на криптовалютах:** Крипторынок работает 24/7, поэтому гэпов нет. Данный инструмент применим только к форексу и фондовым рынкам.

- **Частичное заполнение гэпа:** Не всегда цена полностью закрывает гэп. Иногда достаточно касания границы. Для входа достаточно касания.

- **Гэп как уровень поддержки/сопротивления:** После того как гэп заполнен, его границы часто становятся уровнями. Например, если цена восходящего гэпа упала до нижней границы и отскочила, то нижняя граница становится поддержкой.

- **Алгоритмическая торговля гэпов:** Многие роботы торгуют «gap fill» стратегии. Вход на отскоке от границы с стопом за ней. В SMC Pro такой логики нет, но её легко добавить.

- **Фильтр по волатильности:** Игнорировать гэпы, если их размер меньше 0.5 ATR (слишком мал). Если размер больше 2 ATR – гэп аномальный, возможно, не заполнится.

- **Гэп и объём:** Если при открытии объём очень высокий, гэп с большей вероятностью будет заполнен, так как крупные игроки участвовали.

### 12. Резюме и шпаргалка (cheat sheet)

| Тип гэпа | Условие | Ожидание | Вход | Стоп | Тейк |
|----------|---------|----------|------|------|------|
| **Восходящий** (bullish gap) | open > close_prev | Заполнение вниз, затем продолжение вверх (или разворот) | Покупка на касании нижней границы при бычьем паттерне | Ниже нижней границы | Верхняя граница или выше |
| **Нисходящий** (bearish gap) | open < close_prev | Заполнение вверх, затем продолжение вниз | Продажа на касании верхней границы при медвежьем паттерне | Выше верхней границы | Нижняя граница или ниже |
| Гэп без CHoCH | – | Не входить | – | – | – |
| Гэп в Kill Zone | – | Увеличить размер | Стандартный | Стандартный | Расширенный (1:3) |

**Мнемоническое правило:**  
*«Гэп открытия – не спеши, сначала закрытия достигни, а потом уж входи».*

### 13. Код для MetaTrader (пользовательская настройка)

В SMC Pro параметры Opening Gap:

```cpp
input bool InpShowOpeningGap = true;   // Показывать гэпы открытия
```

Цвет:

```cpp
#define IMB_COLOR_OPEN_GAP   clrWhite   // белый
```

**Изменение минимального размера гэпа:**  
В функции замените `minGapSize = _Point * 2` на `_Point * 5` для более крупных гэпов.

**Уведомление при формировании гэпа:**

```cpp
if(currOpen > prevClose + minGapSize)
{
   SendAlert("Восходящий гэп на " + Symbol() + " от " + DoubleToString(prevClose, _Digits) + " до " + DoubleToString(currOpen, _Digits));
}
```

**Фильтр по дням недели:**  
Гэпы между пятницей и понедельником самые сильные. Можно добавить проверку на `TimeDayOfWeek(time[i]) == 1` (понедельник).

### 14. Вопросы для самопроверки (с ответами)

1. **Что такое Opening Gap?**  
   *Ответ:* Ценовой разрыв между закрытием предыдущего торгового дня и открытием текущего.

2. **Как математически определить восходящий гэп?**  
   *Ответ:* `open_curr > close_prev + δ`.

3. **Почему гэпы часто заполняются?**  
   *Ответ:* Из‑за балансировки спроса/предложения, наличия ордеров на уровне закрытия и психологии трейдеров.

4. **Какой минимальный размер гэпа рекомендуется для торговли?**  
   *Ответ:* Не менее 3-5 пунктов на 5-знаке.

5. **Следует ли входить сразу на открытии гэпа?**  
   *Ответ:* Нет, лучше дождаться касания противоположной границы и подтверждения (CHoCH, свечной паттерн).

6. **Какое подтверждение усиливает сигнал на заполнении гэпа?**  
   *Ответ:* CHoCH и свечной паттерн (пин-бар, поглощение).

7. **Почему гэпы после сильных новостей могут не заполниться?**  
   *Ответ:* Новость изменила фундаментальную оценку актива, новый уровень стал справедливым.

8. **Какой минимальный R/R рекомендуется для сделок по гэпу?**  
   *Ответ:* Не менее 1:2 (расстояние до противоположной границы).

9. **В каких сессиях гэпы наиболее значимы?**  
   *Ответ:* На открытии лондонской и нью-йоркской сессий, особенно после выходных.

10. **Как отличить случайный гэп от значимого?**  
    *Ответ:* Значимый гэп имеет размер >0.5 ATR, совпадает с уровнем ликвидности или FVG, и произошёл в Kill Zone.

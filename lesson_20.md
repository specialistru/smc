## 📘 Глава 20. Propulsion Block – серия свечей перед сильным движением

---

### 1. Заголовок, номер и краткое определение

**Название:** Propulsion Block – серия свечей перед сильным движением  
**Номер:** Глава 20 (Часть 4. Ордерные блоки и зоны митигации, инструмент №7 из 9)  
**Суть одной фразой:** Propulsion Block (PB) – это группа из нескольких (обычно 3–5) последовательных свечей, которые движутся в одном направлении с нарастающим импульсом (каждая следующая свеча имеет больший диапазон и/или закрывается за экстремумом предыдущей), и которые предшествуют особенно сильному движению или его продолжению.  
**Визуальное представление:** 🚀📊 – несколько свечей, объединённых в прямоугольник, охватывающий их общий диапазон, часто с градиентной заливкой или стрелкой ускорения.

---

### 2. Исторический контекст и происхождение

Термин **Propulsion Block** (иногда «Impulse Block» или «Acceleration Zone») не является классическим в SMC и редко встречается в оригинальных видео ICT. Он скорее относится к авторским разработкам команды **SMC Pro** и других трейдеров, которые заметили, что перед сильными трендовыми движениями часто формируется серия свечей с особыми свойствами – каждая следующая свеча «подталкивает» цену дальше, увеличивая диапазон и скорость. Это явление получило название «propulsion» (движущая сила).

В классическом теханализе аналогом является **«импульсная фаза»** в волновой теории Эллиотта или **«толчок»** (thrust) в анализе объёмов. Однако Propulsion Block формализует этот паттерн через конкретные условия по диапазону и закрытию свечей.

**Почему Propulsion Block важен?**  
Розничные трейдеры часто входят в рынок уже после того, как произошло сильное движение, опаздывая. Propulsion Block позволяет идентифицировать **начало ускорения**, когда крупные игроки только начинают агрессивно двигать цену. Торговля на пробое Propulsion Block даёт возможность войти в тренд на ранней стадии с хорошим соотношением риск/прибыль.

---

### 3. Математическое определение и формальные условия

#### Определение Propulsion Block
Рассмотрим последовательность свечей с индексами `i-2`, `i-1`, `i` (или больше).  
**Бычий Propulsion Block (восходящий импульс):**
1. Каждая свеча – бычья (`close > open`).
2. Каждая следующая свеча имеет **более высокий максимум**, чем предыдущая: `high[i] > high[i-1] > high[i-2]`.
3. Каждая следующая свеча имеет **более высокий минимум**, чем предыдущая: `low[i] > low[i-1] > low[i-2]`.
4. (Опционально) Диапазон свечей увеличивается: `(high[i]-low[i]) > (high[i-1]-low[i-1]) > (high[i-2]-low[i-2])`.
5. **Зона Propulsion Block** – прямоугольник, охватывающий диапазон от `low[i-2]` до `high[i]` (или от минимального low до максимального high за все свечи блока).

**Медвежий Propulsion Block (нисходящий импульс):**
1. Каждая свеча – медвежья (`close < open`).
2. Каждый следующий максимум ниже предыдущего: `high[i] < high[i-1] < high[i-2]`.
3. Каждый следующий минимум ниже предыдущего: `low[i] < low[i-1] < low[i-2]`.
4. Диапазон увеличивается (по абсолютной величине).
5. Зона PB – от `low[i]` до `high[i-2]` (нижняя граница – минимум последней свечи, верхняя – максимум первой).

#### Дополнительные условия:
- Минимальная длина серии – 3 свечи. Можно использовать 4–5 для большей надёжности.
- После серии PB часто следует пауза (консолидация) или резкий выброс.
- PB может быть частью более крупного движения (начало 3-й волны Эллиотта).

#### Пример с реальными числами (бычий PB):
- Свеча 1: high=1.1040, low=1.1030, close=1.1038
- Свеча 2: high=1.1048, low=1.1035, close=1.1045
- Свеча 3: high=1.1060, low=1.1042, close=1.1058
- Условия выполнены. Зона PB: от 1.1030 до 1.1060.

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

**Псевдокод:**
```
Для каждого i от 3 до totalBars-1:
    если close[i] > open[i] и close[i-1] > open[i-1] и close[i-2] > open[i-2]:
        если high[i] > high[i-1] и high[i-1] > high[i-2]:
            если low[i] > low[i-1] и low[i-1] > low[i-2]:
                // Бычий PB
                priceLow = low[i-2]
                priceHigh = high[i]
                нарисовать прямоугольник от time[i-2] до time[i]
    аналогично для медвежьего (close < open, high убывают, low убывают)
```

**Реальный код из индикатора SMC Pro (функция `DetectAndDrawPropulsionBlock`):**

```cpp
void DetectAndDrawPropulsionBlock(const datetime &time[], const double &open[], const double &high[],
                                  const double &low[], const double &close[], int totalBars)
{
   if(!InpShowPropulsion) return;
   // Удаляем старые объекты
   for(int i = ObjectsTotal(0,0,-1)-1; i >= 0; i--)
   {
      string name = ObjectName(0,i);
      if(StringFind(name, g_prefix + "PB_") == 0)
         ObjectDelete(0, name);
   }
   
   for(int i = totalBars-4; i >= 2; i--)
   {
      // Бычий Propulsion Block (восходящий)
      if(close[i] > open[i] && close[i-1] > open[i-1] && close[i-2] > open[i-2] &&
         high[i] > high[i-1] && high[i-1] > high[i-2] &&
         low[i] > low[i-1] && low[i-1] > low[i-2])
      {
         double pLow = low[i-2];
         double pHigh = high[i];
         string name = g_prefix + "PB_BULL_" + IntegerToString(time[i]);
         DrawRectangleZone(name, time[i-2], pLow, time[i], pHigh, PB_COLOR_BULL, 130);
         // Сохраняем в массив pbZones
         PropulsionBlock pb;
         pb.timeStart = time[i-2];
         pb.timeEnd = time[i];
         pb.priceLow = pLow;
         pb.priceHigh = pHigh;
         pb.type = 1;
         ArrayResize(pbZones, ArraySize(pbZones)+1);
         pbZones[ArraySize(pbZones)-1] = pb;
      }
      // Медвежий Propulsion Block (нисходящий)
      if(close[i] < open[i] && close[i-1] < open[i-1] && close[i-2] < open[i-2] &&
         high[i] < high[i-1] && high[i-1] < high[i-2] &&
         low[i] < low[i-1] && low[i-1] < low[i-2])
      {
         double pLow = low[i];
         double pHigh = high[i-2];
         string name = g_prefix + "PB_BEAR_" + IntegerToString(time[i]);
         DrawRectangleZone(name, time[i-2], pLow, time[i], pHigh, PB_COLOR_BEAR, 130);
         PropulsionBlock pb;
         pb.timeStart = time[i-2];
         pb.timeEnd = time[i];
         pb.priceLow = pLow;
         pb.priceHigh = pHigh;
         pb.type = -1;
         ArrayResize(pbZones, ArraySize(pbZones)+1);
         pbZones[ArraySize(pbZones)-1] = pb;
      }
   }
}
```

**Пояснения:**  
- В коде используется минимум 3 свечи. Можно расширить до 4 (i, i-1, i-2, i-3) для большей надёжности.
- Цвета: `PB_COLOR_BULL` – светло-синий (`clrLightBlue`), `PB_COLOR_BEAR` – розовый (`clrPink`).
- Прямоугольник охватывает время от первой свечи блока до последней.

### 5. Как это выглядит на графике

#### Сценарий 1. Бычий Propulsion Block

**График EURUSD H1:**
- Три бычьих свечи подряд:
  - Свеча 1: open 1.1040, close 1.1048 (диапазон 8 п.)
  - Свеча 2: open 1.1048, close 1.1058 (диапазон 10 п.)
  - Свеча 3: open 1.1058, close 1.1070 (диапазон 12 п.)
- Индикатор рисует **светло-синий прямоугольник** от low свечи 1 (1.1035) до high свечи 3 (1.1070).
- После этого блока часто следует небольшая консолидация (1-2 свечи), а затем продолжение роста.

#### Сценарий 2. Медвежий Propulsion Block

- Три медвежьих свечи с падающими экстремумами.
- Прямоугольник **розового цвета** от low свечи 3 до high свечи 1.
- После блока – отскок или продолжение падения.

#### Как отображается в SMC Pro:
- Полупрозрачный прямоугольник (прозрачность 130/255) с цветом, указывающим направление.
- Может быть добавлена стрелка ускорения на последней свече блока.

*(В книге – скриншот с тремя растущими свечами, объединёнными в прямоугольник, и последующим движением.)*

### 6. Торговая тактика (полный цикл сделки)

Propulsion Block может использоваться двумя способами:
1. **Вход сразу после завершения блока** – на закрытии третьей свечи (агрессивно).
2. **Вход на ретесте зоны PB** – после небольшой коррекции (консервативно).

Наиболее надёжен **второй способ**, так как ретест даёт лучшую точку входа.

#### Стратегия «Ретест Propulsion Block»

**Шаг 1. Идентифицируем PB**
- На графике находим прямоугольник PB (3+ свечи с ускорением).

**Шаг 2. Определяем контекст (HTF)**
- Для бычьего PB: HTF тренд должен быть восходящим или только начинающимся.
- Для медвежьего PB: HTF тренд нисходящий.

**Шаг 3. Ждём коррекции (возврата в зону PB)**
- Цена должна частично или полностью вернуться в прямоугольник PB.
- Идеально, если коррекция составляет 38.2% – 61.8% от диапазона PB.

**Шаг 4. Подтверждения**

| Подтверждение | Вес | Пояснение |
|---------------|-----|-----------|
| **Свечной паттерн** | Высокий | Пин-бар или поглощение на границе PB |
| **CHoCH** | Высокий | После касания формируется новый экстремум в сторону PB |
| **Объём** | Средний | На ретесте объём выше среднего |
| **Kill Zone** | Средний | Ретест в Kill Zone надёжнее |
| **Совпадение с OB/FVG/MB** | Очень высокий | PB + другой уровень |

**Шаг 5. Точка входа**

*Пример для бычьего PB (покупка):*
- PB: зона 1.1035 – 1.1070.
- Цена корректируется до 1.1045 (внутри зоны), свеча закрывается 1.1050 (пин-бар вверх).
- **Вход:** покупка на закрытии (1.1050).
- **Стоп-лосс:** на 3-5 пунктов ниже минимума PB (1.1030).
- **Тейк-профит:** расстояние, равное размеру PB (1.1070 – 1.1035 = 35 п.) – до 1.1105, либо ближайший BSL.

*Пример для медвежьего PB (продажа):*
- PB: зона 1.2500 – 1.2470.
- Коррекция до 1.2490, пин-бар вниз.
- Вход: продажа 1.2485.
- Стоп: 1.2505.
- Тейк: 1.2440.

**Шаг 6. Управление позицией**
- PB часто предвещает сильное трендовое движение. Частичная фиксация на 1:1, остальное с трейлингом.

### 7. Связь с другими инструментами из 48

| Парный инструмент | Как усиливает PB | Пример |
|-------------------|------------------|--------|
| **Order Block** | Если PB начинается от OB – суперсигнал | OB + PB = начало тренда |
| **FVG** | PB часто создаёт FVG | Дисбаланс внутри PB |
| **Mitigation Block** | Коррекция к MB внутри PB | Ретест MB = точка входа |
| **CHoCH** | Подтверждение после коррекции | CHoCH даёт сигнал на вход |
| **Kill Zones** | PB в Kill Zone надёжнее | Торгуем только в Лондон/НЙ |
| **Propulsion Block** (сам) | Чем больше свечей в блоке, тем сильнее | 5 свечей > 3 свечей |

**Коэффициент надёжности:**
- PB alone (просто ретест) – 60-70%.
- PB + свечной паттерн – 75-80%.
- PB + CHoCH – 85-90%.
- PB + OB + CHoCH – 90-95%.

### 8. Психология и рыночная логика

**Почему PB работает?**  

1. **Ускорение – признак агрессивного входа:** Когда крупные игроки начинают активно набирать позицию, они создают серию свечей с расширяющимися диапазонами. Это «движущая сила».
2. **Коррекция – естественная пауза:** После ускорения рынок часто делает паузу (коррекцию), чтобы набрать ликвидность. Именно в этой точке трейдеры могут войти с лучшей ценой.
3. **Память толпы:** Увидев сильные свечи, трейдеры ожидают продолжения и ставят ордера на пробой, что самосбывается.

**Эмоции трейдера:**
- **Страх упустить** – во время самого блока.
- **Надежда** – на коррекции.
- **Уверенность** – после входа.

**Правильная психология:**  
Не входить во время активного PB (на 3-й свече), а ждать коррекции. Не бояться коррекции – это нормально.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Вход на последней свече PB** – цена часто корректирует сразу после.
2. **Игнорирование коррекции** – вход без ожидания ретеста.
3. **Слишком широкий стоп** – ставить стоп за зоной PB? Лучше за её границей.
4. **Торговля PB против HTF тренда** – ослабляет сигнал.
5. **Использование только 3 свечей** – 4–5 свечей надёжнее.

**Чек-лист для бычьего PB (покупка на ретесте):**

- [ ] PB идентифицирован (3+ бычьих свечей с ростом экстремумов и диапазонов).
- [ ] Цена вернулась в зону PB (коррекция 38-62%).
- [ ] На касании сформировался бычий свечной паттерн.
- [ ] CHoCH вверх подтверждён.
- [ ] HTF тренд восходящий.
- [ ] Kill Zone активна.
- [ ] R/R не менее 1:2.
- [ ] Стоп за нижней границей PB.

### 10. Продвинутые нюансы

- **Длина PB:** 3 свечи – минимально, 4–5 – лучше. Более 5 свечей редко, но очень сильны.
- **Пропуск свечи:** Иногда одна свеча в блоке может не соответствовать условиям (например, тело маленькое, но тени растут). Тогда блок прерывается.
- **PB и объём:** Идеально, если объём растёт вместе с диапазонами.
- **PB на мульти-таймфрейме:** PB на H1 + PB на M15 = мощный тренд.
- **Альтернативный вход:** Можно входить на пробое верхней границы PB (агрессивно) с стопом за нижней границей.
- **Фильтр по ATR:** Диапазон блока должен быть > 0.5 ATR, иначе движение слабое.

## 📘 Глава 15. Breaker Block (BB) – пробитый Order Block

---

### 1. Заголовок, номер и краткое определение

**Название:** Breaker Block (BB) – пробитый Order Block  
**Номер:** Глава 15 (Часть 4. Ордерные блоки и зоны митигации, инструмент №2 из 9)  
**Суть одной фразой:** Breaker Block (BB) – это Order Block, который был пробит ценой (закрытие за противоположной границей), после чего он меняет свою роль: бычий OB становится сопротивлением, медвежий OB – поддержкой, и часто используется как зона для входа в направлении пробоя.  
**Визуальное представление:** 🧱➡🔨 – прямоугольник OB меняет цвет на более тёмный или штриховой, и добавляется метка «BB».

---

### 2. Исторический контекст и происхождение

Понятие **Breaker Block** (иногда называют «Broken OB» или «Failed Order Block») ввёл Майкл Хаддлстон (ICT) как логическое продолжение концепции Order Block. Он заметил, что когда цена пробивает OB, этот уровень не исчезает, а трансформируется – подобно тому, как поддержка становится сопротивлением в классическом теханализе. Однако в SMC это явление получило отдельное название, потому что BB часто даёт отличные сигналы для входа **в направлении пробоя**, то есть против первоначальной роли OB.

В классическом теханализе аналогом является «пробитый уровень поддержки/сопротивления», но без привязки к OB. ICT же подчеркнул, что BB особенно силён, если пробой произошёл с высоким объёмом и последующим возвратом к уровню.

**Почему BB важен?**  
Трейдеры, не знающие о BB, продолжают рассматривать пробитый OB как зону для входа в старом направлении (покупают на бычьем OB, даже если цена ушла ниже). Это ведёт к убыткам. Понимание BB позволяет:
1. Своевременно прекратить использовать OB в старом качестве.
2. Использовать BB как новую зону для входа в направлении пробоя (например, продавать при ретесте бывшего бычьего OB).
3. Отличать ложные пробои OB (когда цена вернулась) от истинных (когда закрытие за границей).

---

### 3. Математическое определение и формальные условия

#### Исходные данные
Пусть задан Order Block (OB) с параметрами:
- Тип: бычий (type = 1) или медвежий (type = -1)
- Нижняя граница: `priceLow`
- Верхняя граница: `priceHigh`
- Время формирования: `time`

#### Определение Breaker Block (BB)
**Для бычьего OB (становится медвежьим BB – сопротивлением):**  
Бычий OB имеет нижнюю границу (обычно `open`) и верхнюю (`close`).  
**Условие пробоя:**  
Цена закрывается **ниже** нижней границы бычьего OB:  
`Close < priceLow`

**Результат:** бычий OB превращается в медвежий BB – зону сопротивления. При последующих подходах цена будет отскакивать от этой зоны вниз.

**Для медвежьего OB (становится бычьим BB – поддержкой):**  
**Условие пробоя:**  
Цена закрывается **выше** верхней границы медвежьего OB:  
`Close > priceHigh`

**Результат:** медвежий OB превращается в бычий BB – зону поддержки.

#### Дополнительные условия для отрисовки в индикаторе:
- Пробой считается подтверждённым, если закрытие произошло **за** границей, а не просто касание.
- BB остаётся активным до тех пор, пока цена снова не пробьёт его в обратную сторону (полная инверсия).
- Визуально BB часто рисуется тем же прямоугольником, но с изменённым цветом (более тёмным) или с заливкой штрихом/пунктиром.
- В SMC Pro BB обычно отображается горизонтальной линией на уровне пробитой границы, а не всей зоной.

#### Пример с реальными числами (бычий OB → медвежий BB):
- Бычий OB: границы 1.1040 – 1.1055 (нижняя 1.1040, верхняя 1.1055).
- Цена падает и закрывается на 1.1038 (ниже нижней границы).
- Теперь зона 1.1040–1.1055 становится сопротивлением. При повторном подходе к 1.1040 цена может отскочить вниз.

### 4. Алгоритм обнаружения (псевдокод + логика индикатора)

**Псевдокод:**
```
Для каждого активного OB в массиве obZones:
    если type == 1 (бычий) и close[i] < priceLow:
        отметить OB как пробитый (broken = true)
        нарисовать BB: горизонтальную линию на уровне priceLow (или прямоугольник)
        изменить цвет на BB_COLOR_BEAR
    если type == -1 (медвежий) и close[i] > priceHigh:
        отметить OB как пробитый
        нарисовать BB горизонтальную линию на уровне priceHigh
        изменить цвет на BB_COLOR_BULL
```

**Реальный код из индикатора SMC Pro (функция `UpdateBreakerBlocks`):**

```cpp
void UpdateBreakerBlocks(int i, const datetime &time[], const double &high[], const double &low[], const double &close[])
{
   // Перебираем все OB из массива obZones
   for(int z = 0; z < ArraySize(obZones); z++)
   {
      // Бычий OB – пробит снизу
      if(obZones[z].type == 1 && !obZones[z].broken)
      {
         if(close[i] < obZones[z].priceLow)
         {
            obZones[z].broken = true;
            // Рисуем горизонтальную линию на уровне нижней границы (теперь это сопротивление)
            string name = g_prefix + "BB_BEAR_" + IntegerToString(obZones[z].time);
            DrawHorizontalLine(name, time[i], obZones[z].priceLow, BB_COLOR_BEAR, 1, STYLE_DASH);
            // Можно также перекрасить прямоугольник OB или оставить его, но добавить метку
         }
      }
      // Медвежий OB – пробит сверху
      if(obZones[z].type == -1 && !obZones[z].broken)
      {
         if(close[i] > obZones[z].priceHigh)
         {
            obZones[z].broken = true;
            string name = g_prefix + "BB_BULL_" + IntegerToString(obZones[z].time);
            DrawHorizontalLine(name, time[i], obZones[z].priceHigh, BB_COLOR_BULL, 1, STYLE_DASH);
         }
      }
   }
}
```

**Более продвинутая версия с отрисовкой прямоугольника BB (штриховка):**

```cpp
if(close[i] < obZones[z].priceLow && obZones[z].type == 1)
{
   obZones[z].broken = true;
   // Меняем стиль прямоугольника: удаляем старый OB и рисуем BB штрихом
   string oldName = g_prefix + "OB_" + IntegerToString(obZones[z].time);
   ObjectDelete(0, oldName);
   string newName = g_prefix + "BB_" + IntegerToString(obZones[z].time);
   DrawRectangleZone(newName, obZones[z].time, obZones[z].priceLow, obZones[z].time, obZones[z].priceHigh, BB_COLOR_BEAR, 200, STYLE_DASH);
}
```

**Пояснения:**  
- В SMC Pro из условия задачи функция `UpdateBreakerBlocks` вызывается в основном цикле `OnCalculate` для каждого нового бара.
- BB отрисовывается только один раз – в момент пробоя. Далее линия остаётся на графике.
- Цвета: `BB_COLOR_BEAR` (медвежий BB) – например, тёмно-оранжевый или красный; `BB_COLOR_BULL` (бычий BB) – тёмно-зелёный или синий.

### 5. Как это выглядит на графике

#### Сценарий 1. Бычий OB превращается в медвежий BB (сопротивление)

**График EURUSD H1:**
- Формируется бычий OB на баре 14:00 с границами 1.1040–1.1055 (зелёный прямоугольник).
- Через 5 баров цена падает, закрывается на 1.1038 (ниже нижней границы).
- Индикатор удаляет зелёный прямоугольник (или оставляет, но добавляет) и рисует **красную пунктирную линию** на уровне 1.1040 с подписью "BB".
- В последующие дни цена подходит к этому уровню снизу (или сверху) и отскакивает вниз – уровень работает как сопротивление.

#### Сценарий 2. Медвежий OB превращается в бычий BB (поддержка)

- Медвежий OB на 1.2500–1.2480 (красный).
- Цена закрывается выше 1.2500 (например, 1.2503).
- Рисуется **зелёная пунктирная линия** на 1.2500.
- В дальнейшем цена подходит к этому уровню сверху и отскакивает вверх.

#### Как отображается в SMC Pro:
- BB обычно отображается в виде горизонтальной линии на уровне пробитой границы (не всей зоной).
- Цвет линии: для медвежьего BB (из бычьего OB) – красный или тёмно-оранжевый; для бычьего BB – зелёный или синий.
- Линия пунктирная или точечная, чтобы отличать от обычных уровней ликвидности.

*(В книге – скриншот: слева бычий OB, затем его пробой вниз, появление красной линии BB, и последующий отскок от неё.)*

### 6. Торговая тактика (полный цикл сделки)

BB используется как **зона входа в направлении пробоя**. То есть если бычий OB пробит вниз, то при ретесте зоны BB (снизу или сверху) входим в продажу. Аналогично для медвежьего BB – вход на покупку при ретесте.

#### Стратегия «BB ретест»

**Шаг 1. Идентифицируем BB**
- На графике видим пунктирную горизонтальную линию – BB.
- Определяем тип: если линия красная (медвежий BB), это бывшая нижняя граница бычьего OB. Ожидаем сопротивление.

**Шаг 2. Определяем контекст (HTF)**
- Для медвежьего BB (сопротивление): желательно, чтобы HTF тренд был нисходящим.
- Для бычьего BB (поддержка): HTF тренд восходящий.

**Шаг 3. Ждём подхода цены к уровню BB**
- Цена должна подойти к уровню BB со стороны, противоположной новому значению:
  - Медвежий BB (сопротивление): цена подходит **снизу**.
  - Бычий BB (поддержка): цена подходит **сверху**.

**Шаг 4. Подтверждения**

| Подтверждение | Вес | Пояснение |
|---------------|-----|-----------|
| **Свечной паттерн** | Высокий | Пин-бар или поглощение на уровне BB в сторону отскока |
| **CHoCH** | Высокий | После касания формируется новый экстремум в сторону от BB |
| **Объём** | Средний | На касании объём выше среднего |
| **Kill Zone** | Средний | Ретест в Kill Zone надёжнее |
| **Совпадение с FVG/OB** | Очень высокий | Если BB совпадает с другим уровнем |

**Шаг 5. Точка входа**

*Пример для медвежьего BB (продажа на сопротивлении):*
- Уровень BB = 1.1040 (бывшая нижняя граница бычьего OB).
- Цена растёт до 1.1040, свеча закрывается 1.1037 (пин-бар вниз).
- **Вход:** продажа по 1.1037.
- **Стоп-лосс:** на 3-5 пунктов выше уровня BB (1.1045).
- **Тейк-профит:** ближайший уровень поддержки (EQL, предыдущий минимум) или R/R 1:2.

*Пример для бычьего BB (покупка на поддержке):*
- Уровень BB = 1.2500 (бывшая верхняя граница медвежьего OB).
- Цена падает до 1.2500, свеча закрывается 1.2503 (пин-бар вверх).
- Вход: покупка 1.2503.
- Стоп: 1.2495.
- Тейк: 1.2530.

**Шаг 6. Управление позицией**
- BB часто даёт быстрый отскок, так как многие трейдеры ставят стопы именно за BB.
- Частичная фиксация на 1:1.
- Если цена пробивает BB насквозь (закрывается за ним), BB может быть аннулирован, позицию лучше закрыть.

### 7. Связь с другими инструментами из 48

| Парный инструмент | Как усиливает BB | Пример |
|-------------------|------------------|--------|
| **Order Block (OB)** | BB – это пробитый OB | Сигнал на ретесте BB часто сильнее, чем на ретесте OB |
| **FVG** | Если BB совпадает с границей FVG – суперзона | Вход от BB с подтверждением FVG |
| **CHoCH** | Обязательное подтверждение | Без CHoCH не входить |
| **Liquidity Sweep** | Часто перед пробоем OB происходит sweep | BB формируется после sweep |
| **Kill Zones** | BB в Kill Zone надёжнее | Торгуем только в Лондон/НЙ |
| **Breaker Block (этот же)** | Может сочетаться с другим BB | Два BB подряд – сильный канал |

**Коэффициент надёжности:**
- BB alone (просто касание уровня) – 55-60%.
- BB + свечной паттерн – 65-70%.
- BB + CHoCH – 75-80%.
- BB + CHoCH + Kill Zone – 85-90%.
- BB + FVG + CHoCH – 90-95%.

### 8. Психология и рыночная логика

**Почему BB работает?**  

1. **Смена роли:** Когда цена пробивает OB, это показывает, что крупные игроки, которые ранее защищали этот уровень, потеряли контроль. Новые игроки взяли инициативу. Уровень становится барьером для возврата.
2. **Стоп-магнит:** Трейдеры, которые вошли на ретесте OB до пробоя, поставили стопы за его границей. Когда цена возвращается к BB, эти стопы всё ещё находятся там (если не сработали), создавая ликвидность.
3. **Психологический уровень:** BB становится заметной линией на графике, и многие трейдеры ожидают отскока, что делает его самосбывающимся.

**Эмоции трейдера у BB:**
- **Сожаление:** «Я купил на OB, а его пробили!»
- **Надежда:** «Может, цена вернётся и отскочит?»
- **Страх:** при повторном подходе к BB.

**Правильная психология SMC:**  
Не держать позицию против пробитого OB. При пробое закрывать убыток и разворачиваться на ретесте BB. Воспринимать BB как новую зону для входа в направлении пробоя.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Продолжать торговать OB как поддержку после пробоя.**  
   *Почему:* это приведёт к убыткам, так как уровень сменил роль.

2. **Вход на BB без подтверждения (свечного паттерна, CHoCH).**  
   *Цена может несколько раз коснуться BB, прежде чем отскочить.*

3. **Использование BB против HTF тренда.**  
   *Если BB и HTF тренд разнонаправлены, отскок может быть слабым.*

4. **Торговля BB на слишком малых ТФ (M1-M5).**  
   *Много шума, ложные касания.*

5. **Игнорирование расстояния между OB и BB.**  
   *Если пробой произошёл, но цена далеко от BB, ретеста может не быть.*

6. **Отсутствие Kill Zone.**  
   *BB, сформированный в азиатскую сессию, часто не ретестируется.*

**Чек-лист перед входом (продажа на медвежьем BB):**

- [ ] BB идентифицирован (бычий OB пробит вниз, красная линия).
- [ ] Цена подошла к уровню BB снизу (или сверху?) – для продажи ждём подход **снизу** к сопротивлению.
- [ ] На касании сформировался медвежий свечной паттерн (пин-бар вниз, поглощение).
- [ ] CHoCH вниз подтверждён (новый максимум ниже предыдущего).
- [ ] H4/D1 нисходящий или как минимум не восходящий.
- [ ] Время – Kill Zone (Лондон/НЙ).
- [ ] R/R не менее 1:2.
- [ ] Стоп на 3-5 пунктов выше уровня BB.

### 10. Продвинутые нюансы

- **BB с учётом теней:** Если OB был Expanded (с тенями), то BB может рисоваться на уровне `low` (для бычьего) или `high` (для медвежьего). Это даёт более широкую зону.

- **Множественные BB:** Если подряд пробито несколько OB, их BB могут образовывать канал. Сильнее всего работает самый свежий BB (по времени пробоя).

- **BB на мульти-таймфрейме:** BB на H1 сильнее, чем на M15. Если BB на M15 совпадает с BB на H1 – зона экстремально сильная.

- **Возврат к OB после пробоя:** Иногда цена пробивает OB, но затем возвращается обратно (ложный пробой). В таком случае OB остаётся активным, а BB не формируется. Фильтр – закрытие за границей должно быть устойчивым (хотя бы 2 бара).

- **Алгоритмическое определение:** В SMC Pro BB определяется автоматически при пробое. Можно добавить проверку, что пробой произошёл с высоким объёмом (объём > 1.5×avg) – тогда BB сильнее.

- **BB как цель тейк-профита:** При движении в сторону пробоя, ближайший BB противоположного типа может быть целью.

- **Визуальные настройки:** Можно изменить цвет и стиль BB в константах.

### 13. Код для MetaTrader (пользовательская настройка)

В SMC Pro параметры BB:

```cpp
input bool InpShowBB = true;   // Показывать Breaker Block
```

Цвета BB:

```cpp
#define BB_COLOR_BULL   clrForestGreen   // бычий BB (поддержка)
#define BB_COLOR_BEAR   clrDarkOrange    // медвежий BB (сопротивление)
```

**Изменение стиля BB с линии на прямоугольник:**

В функции `UpdateBreakerBlocks` замените `DrawHorizontalLine` на `DrawRectangleZone` с флагом `STYLE_DASH`.

**Уведомление при формировании BB:**

```cpp
if(close[i] < obZones[z].priceLow && obZones[z].type == 1 && !obZones[z].broken)
{
   SendAlert("Сформирован медвежий Breaker Block на " + Symbol() + " на уровне " + DoubleToString(obZones[z].priceLow, _Digits));
}
```

**Фильтр по объёму:**  
Добавьте условие, что объём на баре пробоя должен быть выше среднего, чтобы избежать ложных пробоев.

### 14. Вопросы для самопроверки (с ответами)

1. **Что такое Breaker Block (BB)?**  
   *Ответ:* Пробитый Order Block, который меняет свою роль: бычий OB становится сопротивлением, медвежий – поддержкой.

2. **При каком условии бычий OB превращается в медвежий BB?**  
   *Ответ:* Когда цена закрывается ниже его нижней границы.

3. **При каком условии медвежий OB превращается в бычий BB?**  
   *Ответ:* Когда цена закрывается выше его верхней границы.

4. **Как визуально BB отличается от OB в SMC Pro?**  
   *Ответ:* BB обычно отображается пунктирной горизонтальной линией, а не прямоугольником, и имеет другой цвет.

5. **Почему BB считается зоной для входа в направлении пробоя?**  
   *Ответ:* Потому что после пробоя крупные игроки начали двигать цену в эту сторону, и при ретесте уровня они могут продолжить.

6. **Какое подтверждение рекомендуется для входа на BB?**  
   *Ответ:* Свечной паттерн на касании и CHoCH.

7. **Какой минимальный R/R рекомендуется для сделок по BB?**  
   *Ответ:* Не менее 1:2.

8. **В каких сессиях BB наиболее надёжен?**  
   *Ответ:* Лондонская (10-12) и Нью-Йоркская (15-17).

9. **Может ли BB инвертироваться обратно в OB?**  
   *Ответ:* Да, если цена снова пробьёт BB в обратную сторону, он может стать обычным уровнем, но обычно после пробоя BB слабеет.

10. **Как отличить ложный пробой OB от истинного, ведущего к BB?**  
    *Ответ:* Истинный пробой сопровождается закрытием свечи за границей и последующим удержанием (цена не возвращается быстро). Ложный – быстрым возвратом внутрь OB.

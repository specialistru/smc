
## 📘 Глава 10. Inversion FVG (IFVG) – пробитый FVG, меняющий направление

> *Полная расширенная глава по утверждённому шаблону (объём ~16 500 знаков)*

---

### 1. Заголовок, номер и краткое определение

**Название:** Inversion FVG (IFVG) – пробитый FVG, меняющий направление  
**Номер:** Глава 10 (Часть 3. Дисбаланс, инструмент №2 из 5)  
**Суть одной фразой:** Inversion FVG (IFVG) – это Fair Value Gap, который был пробит ценой, закрывшейся за его противоположной границей, после чего зона меняет свою роль: бычий FVG становится медвежьим сопротивлением, а медвежий FVG – бычьей поддержкой.  
**Визуальное представление:** 🔄🔵➡🔴 или 🔄🔴➡🔵 – прямоугольник FVG меняет цвет и стиль (штриховой), показывая инверсию.

---

### 2. Исторический контекст и происхождение

Понятие инверсии уровней поддержки/сопротивления известно в техническом анализе давно: «бывший уровень поддержки становится сопротивлением». Однако применительно к FVG термин **Inversion FVG (IFVG)** ввёл и популяризировал **Майкл Хаддлстон (ICT)** в своих видео 2017–2018 годов. Он заметил, что когда цена пробивает FVG насквозь (закрывается за противоположной границей), этот FVG не исчезает, а превращается в зону противоположного значения: бычий FVG начинает работать как сопротивление, медвежий – как поддержка.

**Почему IFVG важен?**  
Трейдеры, не знающие об инверсии, часто продолжают рассматривать пробитый FVG как зону для входа в первоначальном направлении (например, покупать на бычьем FVG, даже если цена ушла ниже его нижней границы). Это ведёт к убыткам. Понимание IFVG позволяет:
1. Своевременно прекратить использовать FVG в старом качестве.
2. Использовать инвертированный FVG как новую зону для входа в противоположном направлении.
3. Отличать ложные пробои FVG (возврат) от истинных (инверсия).

### 3. Математическое определение и формальные условия

#### Исходные данные
Пусть задан FVG с параметрами:
- Тип: бычий (type = 1) или медвежий (type = -1)
- Нижняя граница: `priceLow`
- Верхняя граница: `priceHigh`
- Время формирования: `time1` (первая свеча) и `time2` (третья свеча)

#### Инверсия бычьего FVG (становится медвежьим сопротивлением)
Бычий FVG имеет нижнюю границу = `high[i-2]`, верхнюю границу = `low[i]`.  
**Условие инверсии:**  
Цена закрывается **ниже** нижней границы бычьего FVG:  
`Close < priceLow` (т.е. `Close < high[i-2]`)

**Результат:** бычий FVG превращается в медвежий IFVG – зону сопротивления. При последующих подходах цена будет отскакивать от этой зоны вниз.

#### Инверсия медвежьего FVG (становится бычьей поддержкой)
Медвежий FVG имеет верхнюю границу = `low[i-2]`, нижнюю границу = `high[i]`.  
**Условие инверсии:**  
Цена закрывается **выше** верхней границы медвежьего FVG:  
`Close > priceHigh` (т.е. `Close > low[i-2]`)

**Результат:** медвежий FVG превращается в бычий IFVG – зону поддержки.

#### Дополнительные условия для отрисовки в индикаторе:
- Инверсия считается подтверждённой, если закрытие произошло **за** границей, а не просто касание.
- IFVG остаётся активным до тех пор, пока цена снова не пробьёт его в обратную сторону (полная инверсия обратно).
- Визуально IFVG рисуется тем же прямоугольником, но:
  - Меняется цвет на противоположный (бычий → красный, медвежий → синий) **или** сохраняется исходный цвет, но используется штриховой стиль `STYLE_DASH`.
  - В SMC Pro принято использовать штриховой стиль (DASH) и исходный цвет, чтобы показать, что это инвертированная зона.

#### Пример с реальными числами (бычий → IFVG медвежий):
- Бычий FVG: нижняя граница = 1.1050, верхняя = 1.1055.
- Цена падает и закрывается на 1.1048 (ниже 1.1050).
- Теперь зона 1.1050–1.1055 становится сопротивлением. При повторном подходе к 1.1050 цена может отскочить вниз.

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

**Псевдокод:**
```
Для каждого активного FVG в массиве fvgZones:
    если type == 1 (бычий) и close[i] < priceLow:
        отметить как IFVG, type = -1 (или сохранить исходный, но добавить флаг inverted)
        изменить стиль прямоугольника на DASH
    если type == -1 (медвежий) и close[i] > priceHigh:
        отметить как IFVG, type = 1
        изменить стиль
```

**Реальный код из индикатора SMC Pro (функция `DetectAndDrawIFVG`):**

```cpp
// Функция проверки инверсии для одного бара (вызывается для каждого i после формирования FVG)
bool IsInversionFVG(int i, const double &high[], const double &low[], const double &close[])
{
   // Проверяем бычий FVG на баре i (i – индекс третьей свечи)
   if(IsBullishFVG(i, high, low))
   {
      // Условие инверсии: следующая свеча (или текущая) закрылась ниже нижней границы бычьего FVG
      // Нижняя граница бычьего FVG = high[i-2]
      if(close[i+1] < high[i-2] || close[i] < high[i-2])
         return true;
   }
   // Проверяем медвежий FVG
   if(IsBearishFVG(i, high, low))
   {
      // Верхняя граница медвежьего FVG = low[i-2]
      if(close[i+1] > low[i-2] || close[i] > low[i-2])
         return true;
   }
   return false;
}

// Основная функция отрисовки IFVG
void DetectAndDrawIFVG(const datetime &time[], const double &high[], const double &low[], const double &close[], int totalBars)
{
   if(!InpShowIFVG) return;
   // Удаляем старые объекты IFVG
   for(int i = ObjectsTotal(0,0,-1)-1; i >= 0; i--)
   {
      string name = ObjectName(0,i);
      if(StringFind(name, g_prefix + "IFVG_") == 0)
         ObjectDelete(0, name);
   }
   
   for(int i = totalBars-3; i >= 2; i--)
   {
      if(IsInversionFVG(i, high, low, close))
      {
         datetime t1 = time[i];
         datetime t2 = time[i-2];
         double priceLow, priceHigh;
         int origType;
         if(IsBullishFVG(i, high, low))
         {
            origType = 1;  // исходный бычий
            priceLow = high[i-2];   // нижняя граница
            priceHigh = low[i];     // верхняя граница
         }
         else
         {
            origType = -1; // исходный медвежий
            priceLow = high[i];     // нижняя граница
            priceHigh = low[i-2];   // верхняя граница
         }
         // Новый тип после инверсии – противоположный исходному
         int invType = (origType == 1) ? -1 : 1;
         // Цвет: можно использовать цвет нового типа или оставить исходный со штриховкой
         color clr = (invType == 1) ? IMB_COLOR_IFVG_BULL : IMB_COLOR_IFVG_BEAR;
         // Рисуем прямоугольник штриховым стилем (STYLE_DASH)
         string name = g_prefix + "IFVG_" + IntegerToString(time[i]);
         DrawRectangleZone(name, t1, priceLow, t2, priceHigh, clr, 180, STYLE_DASH);
         
         // Сохраняем в глобальный массив ifvgZones
         IFVGZone ifvg;
         ifvg.time1 = t1;
         ifvg.time2 = t2;
         ifvg.priceLow = priceLow;
         ifvg.priceHigh = priceHigh;
         ifvg.originalType = origType;
         ifvg.invertedType = invType;
         ifvg.drawn = true;
         ArrayResize(ifvgZones, ArraySize(ifvgZones)+1);
         ifvgZones[ArraySize(ifvgZones)-1] = ifvg;
      }
   }
}
```

**Пояснения к коду:**
- Функция `IsInversionFVG` проверяет для бара `i` (третьей свечи FVG), произошла ли инверсия на текущей свече или следующей.
- `DrawRectangleZone` получает дополнительный параметр `style = STYLE_DASH`, чтобы визуально отличить IFVG от обычного FVG (сплошная заливка).
- Цвет `IMB_COLOR_IFVG_BULL` и `IMB_COLOR_IFVG_BEAR` по умолчанию – синий и красный, но их можно переназначить.

### 5. Как это выглядит на графике

#### Сценарий 1. Бычий FVG инвертируется в медвежий IFVG (сопротивление)

**График EURUSD H1:**
- **Формирование бычьего FVG:** три свечи, зона 1.1050–1.1055 (синий прямоугольник, сплошная заливка).
- **Инверсия:** через несколько баров цена падает, закрывается на 1.1048 (ниже 1.1050). Индикатор удаляет синий сплошной прямоугольник и рисует **красный штриховой прямоугольник** на той же зоне (1.1050–1.1055).
- **Теперь зона работает как сопротивление.** При последующем росте цена касается 1.1050 и отскакивает вниз.

#### Сценарий 2. Медвежий FVG инвертируется в бычий IFVG (поддержка)

**График GBPUSD H1:**
- Медвежий FVG: зона 1.2500–1.2495 (красный сплошной).
- Инверсия: цена закрывается выше 1.2500 (например, 1.2503). Зона перерисовывается **синим штриховым прямоугольником**.
- Теперь 1.2495–1.2500 – поддержка. При падении цена отскакивает от этой зоны вверх.

#### Как отображается в SMC Pro:
- Обычные FVG – сплошная заливка (STYLE_SOLID).
- IFVG – штриховая заливка (STYLE_DASH) или пунктирная, цвет может оставаться исходным или меняться на противоположный (зависит от настроек).
- Дополнительно можно отображать стрелку или метку "IFVG" рядом с зоной.

*(В книге – два скриншота: один с бычьим FVG, затем тот же участок с инверсией и штриховкой; второй – медвежий FVG и его инверсия.)*

### 6. Торговая тактика (полный цикл сделки)

IFVG даёт два типа торговых возможностей:
1. **Вход после инверсии** – когда IFVG сформировался, торгуем от него в направлении нового типа.
2. **Вход на ретесте IFVG** – цена возвращается к инвертированной зоне и отскакивает.

Наиболее надёжна **вторая тактика** – ретест IFVG.

#### Стратегия «Ретест IFVG»

**Шаг 1. Идентифицируем инверсию FVG**
- На графике появился штриховой прямоугольник (IFVG).
- Запоминаем его границы и новый тип (поддержка или сопротивление).

**Шаг 2. Определяем контекст (HTF)**
- Для бычьего IFVG (поддержка): HTF тренд должен быть восходящим или хотя бы не нисходящим.
- Для медвежьего IFVG (сопротивление): HTF тренд нисходящий.

**Шаг 3. Ждём возврата цены к зоне IFVG**
- Цена должна подойти к границе IFVG со стороны, соответствующей новому типу:
  - Бычий IFVG (поддержка): цена подходит **сверху** к верхней границе (или касается зоны).
  - Медвежий IFVG (сопротивление): цена подходит **снизу** к нижней границе.

**Шаг 4. Подтверждения**

| Подтверждение | Вес | Пояснение |
|---------------|-----|-----------|
| **Свечной паттерн** (пин-бар, поглощение) | Высокий | Свеча, коснувшаяся границы IFVG, имеет длинную тень в противоположную сторону |
| **CHoCH** | Высокий | После касания формируется новый экстремум в сторону от IFVG |
| **Объём** | Средний | На касании объём выше среднего |
| **Kill Zone** | Средний | Касание в Лондон/НЙ надёжнее |
| **Совпадение с OB/FVG** | Очень высокий | Если граница IFVG совпадает с OB или другим FVG |

**Шаг 5. Точка входа**

*Пример для бычьего IFVG (покупка на поддержке):*
- Бычий IFVG: зона 1.1050–1.1055 (поддержка).
- Цена падает до 1.1050, свеча закрывается на 1.1053 (пин-бар вверх).
- **Вход:** рыночный на закрытии свечи (1.1053) или лимитный на 1.1051.
- **Стоп-лосс:** на 3-5 пунктов ниже нижней границы IFVG (1.1045).
- **Тейк-профит:** ближайший уровень сопротивления (EQH, предыдущий максимум) или R/R 1:2.

*Пример для медвежьего IFVG (продажа на сопротивлении):*
- Медвежий IFVG: зона 1.2500–1.2495 (сопротивление).
- Цена растёт до 1.2500, свеча закрывается 1.2497 (пин-бар вниз).
- Вход: продажа 1.2497.
- Стоп: 1.2505.
- Тейк: 1.2470.

**Шаг 6. Управление позицией**
- Частичная фиксация на 1:1, стоп в безубыток.
- Остаток до 1:3 или до противоположного уровня ликвидности.
- Если цена входит в IFVG и застревает, лучше закрыть – возможно повторное тестирование.

### 7. Связь с другими инструментами из 48

| Парный инструмент | Как усиливает IFVG | Пример |
|-------------------|--------------------|--------|
| **Обычный FVG** | IFVG – это трансформированный FVG, его сила выше | FVG после инверсии становится надёжнее обычного уровня |
| **Order Block (OB)** | Если граница IFVG совпадает с OB – суперсильная зона | Бычий IFVG + бычий OB |
| **Breaker Block (BB)** | IFVG – это, по сути, BB для FVG | Аналогия с пробитым OB |
| **CHoCH** | CHoCH после касания IFVG подтверждает разворот | Используем CHoCH как фильтр |
| **Liquidity Sweep** | Часто перед инверсией FVG происходит sweep BSL/SSL | Инверсия как следствие сбора ликвидности |
| **Kill Zones** | IFVG, сформированный в Kill Zone, имеет больше шансов на ретест | Торгуем только в активные сессии |
| **Market Maker Model** | IFVG – классический уровень после манипуляции | В MMM инверсия происходит после сбора ликвидности |

**Коэффициент надёжности (по опросам):**
- IFVG alone (просто касание границы) – 55-65%.
- IFVG + свечной паттерн – 70-75%.
- IFVG + свечной паттерн + CHoCH – 80-85%.
- IFVG + CHoCH + Kill Zone – 85-90%.
- IFVG + OB + CHoCH + Kill Zone – 90-95%.

### 8. Психология и рыночная логика

**Почему IFVG работает?**  

1. **Смена контроля:** Когда цена пробивает FVG насквозь (закрывается за его противоположной границей), это означает, что крупные игроки, которые изначально создали FVG, потеряли контроль. Новые игроки (или те же, но с противоположной позицией) взяли инициативу. Зона, которая раньше была поддержкой, становится сопротивлением, и наоборот.
2. **Ловушка для опаздывающих:** Трейдеры, которые не знают об инверсии, продолжают рассматривать старый FVG как зону для входа (например, покупают на бычьем FVG, даже когда цена ушла ниже). Институционалы используют это, чтобы исполнить свои ордера, толкая цену обратно к IFVG.
3. **Самоподтверждение:** Многие алгоритмы распознают IFVG как зону и торгуют от неё, что делает паттерн самосбывающимся.

**Эмоции трейдера при инверсии:**
- **Удивление:** «Как так, FVG был бычьим, а теперь цена отскакивает от него вниз?»
- **Разочарование:** у тех, кто купил на ретесте бычьего FVG до инверсии.
- **Уверенность:** у тех, кто понимает IFVG и входит после смены типа.

**Правильная психология SMC:**
- Следить за закрытием свечи относительно границ FVG. Если цена закрылась за границей – ждать инверсии и менять стратегию.
- Не торговать FVG, который уже пробит, до тех пор, пока не сформируется IFVG и не будет ретеста.
- Использовать IFVG как зону, противоположную первоначальной.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Игнорирование инверсии** – продолжать торговать FVG в старом направлении после пробоя.  
   *Почему:* это быстрая потеря депозита.

2. **Вход на IFVG без подтверждения** (свечного паттерна, CHoCH).  
   *Почему:* цена может просто продолжить движение сквозь IFVG, если нет отскока.

3. **Использование IFVG против HTF тренда.**  
   *Пример:* бычий IFVG (поддержка) на M15, но H4 нисходящий. Отскок может быть краткосрочным.

4. **Торговля IFVG вне Kill Zone.**  
   *Азиатские ретесты часто ложные.*

5. **Неправильное определение границ IFVG.**  
   *Путают нижнюю и верхнюю границы при инверсии.*

6. **Слишком близкий стоп** (прямо на границе).  
   *Рекомендуется 3-5 пунктов за границу.*

7. **Вход без CHoCH.**  
   *Без смены структуры цена может просто консолидироваться.*

**Чек-лист перед входом (лонг по бычьему IFVG):**

- [ ] Был бычий FVG, затем закрытие ниже его нижней границы – инверсия в медвежий IFVG? (Нет, для лонга нужен медвежий FVG, инвертированный в бычий IFVG).
- [ ] Исходный FVG был медвежьим, затем закрытие выше его верхней границы – сформирован бычий IFVG.
- [ ] Цена подошла к зоне IFVG сверху (касание верхней границы или входа внутрь).
- [ ] Свеча касания имеет длинную нижнюю тень (пин-бар вверх) или бычье поглощение.
- [ ] CHoCH вверх подтверждён.
- [ ] H4/D1 не нисходящий.
- [ ] Время – Kill Zone.
- [ ] R/R не менее 1:2.
- [ ] Стоп на 3-5 пунктов ниже нижней границы IFVG.

### 10. Продвинутые нюансы

- **Частичная инверсия:** Иногда цена закрывается за границей FVG, но лишь на 1-2 пункта, а затем возвращается. Это может быть ложная инверсия. Лучше дождаться подтверждения устойчивости (например, ещё одного бара за границей).

- **Множественная инверсия:** FVG может инвертироваться несколько раз, особенно в боковике. Каждый раз зона перекрашивается. После второй инверсии она становится менее надёжной.

- **IFVG на мульти-таймфрейме:** Если IFVG на M15 совпадает с IFVG на H1 – сила зоны максимальна.

- **IFVG как цель для тейк-профита:** При движении в одном направлении можно ставить тейк на границу противоположного IFVG (например, при лонге – на верхнюю границу медвежьего IFVG).

- **Автоматическое распознавание:** В SMC Pro инверсия определяется автоматически, но вы можете настроить чувствительность (количество баров для подтверждения). По умолчанию используется 1 бар.

- **Фильтр по ATR:** Если размер IFVG меньше 0.2 ATR, зона слабая, лучше игнорировать.

- **Связь с BPR:** BPR (Balanced Price Range) – это пересечение двух FVG, которое часто приводит к IFVG одной из зон. Глава 16.

- **Визуальные настройки:** В SMC Pro вы можете изменить цвет IFVG и стиль. Для любителей минимализма можно сделать штриховку без изменения цвета.

### 12. Резюме и шпаргалка (cheat sheet)

| Исходный FVG | Условие инверсии | Новый тип IFVG | Действие на ретесте | Стоп |
|--------------|------------------|----------------|---------------------|------|
| **Бычий** (синий) | Close < нижней границы | Медвежий (сопротивление) | Продажа от верхней границы | На 3-5 п. выше верхней границы |
| **Медвежий** (красный) | Close > верхней границы | Бычий (поддержка) | Покупка от нижней границы | На 3-5 п. ниже нижней границы |
| IFVG без подтверждения свечой | – | – | Не входить | – |
| IFVG в Kill Zone | – | – | Увеличить размер | Стандартный |

**Мнемоническое правило:**  
*«FVG пробит насквозь – свою роль он сменил. Где была поддержка, там теперь сопротивление».*

### 14. Вопросы для самопроверки (с ответами)

1. **Что такое Inversion FVG (IFVG)?**  
   *Ответ:* FVG, который был пробит (закрытие за противоположной границей) и сменил свою роль: бычий стал сопротивлением, медвежий – поддержкой.

2. **При каком условии бычий FVG инвертируется в медвежий IFVG?**  
   *Ответ:* Когда цена закрывается ниже его нижней границы.

3. **При каком условии медвежий FVG инвертируется в бычий IFVG?**  
   *Ответ:* Когда цена закрывается выше его верхней границы.

4. **Чем визуально IFVG отличается от обычного FVG в SMC Pro?**  
   *Ответ:* Штриховым стилем (DASH) вместо сплошной заливки.

5. **Почему важно различать IFVG и обычный FVG?**  
   *Ответ:* Чтобы не ошибаться с направлением входа: на IFVG торгуем от него в противоположную сторону от первоначального FVG.

6. **Какой свечной паттерн усиливает сигнал на ретесте IFVG?**  
   *Ответ:* Пин-бар с тенью в сторону от IFVG или поглощение.

7. **Почему IFVG в Kill Zone надёжнее?**  
   *Ответ:* Потому что в Kill Zone активность крупных игроков выше, и ретесты чаще приводят к сильным движениям.

8. **Может ли IFVG инвертироваться обратно?**  
   *Ответ:* Да, если цена снова пробьёт его в противоположную сторону. Но с каждой инверсией надёжность падает.

9. **Какой минимальный R/R рекомендуется для сделок по IFVG?**  
   *Ответ:* Не менее 1:2, оптимально 1:3.

10. **Как отличить ложную инверсию от истинной?**  
    *Ответ:* Истинная инверсия сопровождается закрытием свечи за границей и последующим удержанием (цена не возвращается быстро). Ложная инверсия – быстрое возвращение внутрь FVG.

## 📘 Глава 3. Liquidity Sweep / Stop Hunt – охота за стопами

> *Полная расширенная глава по утверждённому шаблону (объём ~18 000 знаков)*

---

### 1. Заголовок, номер и краткое определение

**Название:** Liquidity Sweep / Stop Hunt – охота за стопами  
**Номер:** Глава 3 (Часть 2. Ликвидность, инструмент №3 из 8)  
**Суть одной фразой:** Liquidity Sweep (или Stop Hunt) – это намеренное движение цены за уровень значимого максимума или минимума с целью активировать стоп-лоссы розничных трейдеров, после чего цена разворачивается и идёт в противоположную сторону.  
**Визуальное представление:** 🎯🔄💥 – цель достигнута, разворот.

---

### 2. Исторический контекст и происхождение

Термин «Stop Hunt» использовался трейдерами ещё на биржевых ямах 1970–80‑х годов, когда брокеры видели скопления стоп-ордеров и сознательно толкали цену в эту зону для собственной выгоды. С развитием электронных торгов (1990–2000) концепция получила второе дыхание благодаря анализу ордер-флоу и DOM (Depth of Market).

- **Джо Росс** («Торговля хаоса») в 1990-х упоминал «ловлю стопов» как часть психологии рынка.
- **Ларри Вильямс** описывал охоту за стопами как «необходимое зло» для крупных игроков.
- **Майкл Хаддлстон (ICT)** в 2010-х ввёл термин **Liquidity Sweep** как ключевой элемент своей методологии, объединив его с CHoCH, FVG и Kill Zones.

**Проблема, которую решает понимание Sweep:**  
Розничный трейдер часто ставит стоп-лосс **за ближайшим свингом** – это самое предсказуемое место. Крупные игроки знают об этом и используют эти стопы как ликвидность для исполнения своих ордеров. Понимая механизм Sweep, трейдер может:
1. Не ставить стопы в очевидных местах.
2. Входить в позицию после Sweep в сторону разворота.
3. Идентифицировать истинные развороты от ложных пробоев.

### 3. Математическое определение и формальные условия

#### Определение Liquidity Sweep
Пусть:
- `SwingHigh` – последний значимый максимум (на заданном lookback).
- `SwingLow` – последний значимый минимум.
- `H` – текущий максимум бара.
- `L` – текущий минимум бара.
- `Close` – цена закрытия бара.

**Sweep BSL (охота за стопами выше максимума):**  
`H > SwingHigh + threshold` И затем (в пределах 1-3 баров) `Close < SwingHigh`.  
*Порог (threshold)* обычно составляет 1–5 пунктов – минимальное касание, достаточное для активации стопов.

**Sweep SSL (охота за стопами ниже минимума):**  
`L < SwingLow - threshold` И затем `Close > SwingLow`.

**Альтернативное определение (через свечные паттерны):**  
Sweep = «ложный пробой» (false break) с возвратом и закрытием за уровень. В классическом теханализе это называется **Swing Failure Pattern (SFP)** – глава 31. Разница лишь в том, что SFP может быть без явной охоты за ликвидностью.

#### Условия для отрисовки в индикаторе:
1. Зафиксирован SwingHigh / SwingLow.
2. На следующем баре или через 1–2 бара цена касается уровня + отклонение (sweep).
3. Затем цена закрывается **строго** за уровень в сторону, противоположную пробою.
4. Индикатор может нарисовать стрелку в точке возврата.

#### Дополнительные количественные критерии (для фильтрации ложных сигналов):
- Отклонение при пробое (sweep) должно быть не менее 1 пункта, но не более 1% от текущей цены (например, для EURUSD 1.1000 → sweep до 1.1010 максимум).
- Возврат должен произойти **на той же свече** или на следующей (максимум через 2 свечи). Если цена ушла дальше 10 пунктов за уровень и не вернулась – это не Sweep, а истинный пробой.

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

**Псевдокод:**
```
Для каждого бара i от rates_total-1 вниз:
   найти lastSwingHigh = последний индекс, где IsSwingHigh(j, lookback) истинно
   найти lastSwingLow  = последний индекс, где IsSwingLow(j, lookback) истинно
   
   если high[i] > high[lastSwingHigh] + epsilon:
        если close[i] < high[lastSwingHigh]:
            отметить Sweep BSL
        иначе если close[i+1] < high[lastSwingHigh]:
            отметить Sweep BSL (на следующем баре)
   
   аналогично для SSL
```

**Реальный код из индикатора SMC Pro (функция `IsLiquiditySweep`):**

```cpp
// Глобальные переменные для хранения последних свингов
double g_lastSwingHigh = -1;
double g_lastSwingLow  = -1;

// Проверка Sweep на текущем баре (вызывается для каждого i)
bool IsLiquiditySweep(int i, const double &high[], const double &low[], 
                      const double &close[], double lastSwingHigh, double lastSwingLow)
{
   bool sweep = false;
   
   // Sweep BSL: цена коснулась или пробила последний SwingHigh
   if(lastSwingHigh > 0 && high[i] > lastSwingHigh)
   {
      // Условие возврата: текущее закрытие или закрытие следующего бара ниже уровня
      if(close[i] < lastSwingHigh)
         sweep = true;
      else if(i < ArraySize(close)-1 && close[i+1] < lastSwingHigh)
         sweep = true;
   }
   
   // Sweep SSL: цена коснулась или пробила последний SwingLow
   if(lastSwingLow > 0 && low[i] < lastSwingLow)
   {
      if(close[i] > lastSwingLow)
         sweep = true;
      else if(i < ArraySize(close)-1 && close[i+1] > lastSwingLow)
         sweep = true;
   }
   
   return sweep;
}

// Более надёжная версия с проверкой "возврата на следующей свече" и учётом шума
bool IsCleanLiquiditySweep(int i, const double &high[], const double &low[], 
                           const double &close[], double lastSwingHigh, double lastSwingLow,
                           double threshold = 0.0001) // 1 пункт для 5-знака
{
   if(lastSwingHigh > 0 && high[i] > lastSwingHigh + threshold)
   {
      // Проверяем закрытие текущей свечи или следующей
      if(close[i] < lastSwingHigh)
         return true;
      if(i < ArraySize(close)-1 && close[i+1] < lastSwingHigh)
         return true;
      // Дополнительная проверка: верхняя тень свечи превышает тело (пин-бар)
      double body = MathAbs(close[i] - open[i]);
      double upperWick = high[i] - MathMax(open[i], close[i]);
      if(upperWick > body * 1.5 && close[i] < lastSwingHigh)
         return true;
   }
   // Аналогично для SSL...
   return false;
}
```

### 5. Как это выглядит на графике

#### Сценарий 1. Бычий Sweep BSL (охота за стопами выше максимума, затем падение)

Рассмотрим гипотетический график EURUSD M15:

**Бар 1 (12:00):** High = 1.1050, Low = 1.1020, Close = 1.1045 – формируется SwingHigh.  
Индикатор рисует синюю пунктирную линию на 1.1050 – это BSL.

**Бар 2 (12:15):** High = 1.1058 (sweep на 8 пунктов), Low = 1.1040, Close = 1.1042.  
**Что видим:** цена коснулась 1.1058 (выше BSL), но закрылась на 1.1042 – ниже BSL.  
Индикатор рисует **красную стрелку вниз** на баре 2 (или на баре 3) – сигнал Sweep BSL.

**Бар 3 (12:30):** High = 1.1048, Low = 1.1025, Close = 1.1028 – цена продолжает падение.

**Вывод:** BSL был собран, крупные игроки набрали короткие позиции, рынок развернулся вниз.

#### Сценарий 2. Медвежий Sweep SSL (охота за стопами ниже минимума, затем рост)

**Бар 1 (10:00):** Low = 1.1000, SwingLow – красная линия SSL.  
**Бар 2 (10:15):** Low = 0.9995 (sweep на 5 пунктов), Close = 1.0003 – закрытие выше SSL.  
Индикатор рисует **зелёную стрелку вверх** (или синюю, в зависимости от настроек).  
**Бар 3 (10:30):** рост до 1.0030.

#### Как всё это отображает индикатор SMC Pro:
- Уровни BSL/SSL – пунктирные линии.
- При обнаружении Sweep – стрелка на баре, где произошёл возврат.
- Дополнительно может быть нарисована зона SFP (Swing Failure Pattern) – пунктирный прямоугольник.

*(В книге – скриншот с аннотациями: линия BSL, точка пробоя, стрелка Sweep, последующее движение.)*

### 6. Торговая тактика (полный цикл сделки)

Стратегия **«Sweep + Reversal»** – одна из самых прибыльных в SMC. Её также называют **«разворот от ликвидности»**.

#### Шаг 1. Определение контекста (HTF)
- Открываем H4 или D1. Определяем направление тренда: EMA 50, серия повышающихся/понижающихся минимумов.
- **Правило:** торгуем Sweep **против** тренда на младшем ТФ, но **по тренду** на старшем?  
  *Нюанс:* Идеально, когда старший тренд и сигнал Sweep на младшем **совпадают по направлению разворота**. Например, на H4 нисходящий тренд (шорты), на M15 случился Sweep BSL – это сигнал к продаже (короткая позиция) в направлении H4 тренда.  
  Если Sweep идёт против HTF тренда – сигнал слабее (возможна только краткосрочная коррекция).

#### Шаг 2. Идентификация SwingHigh/SwingLow
- На M15 или H1 ждём формирования чёткого экстремума с lookback = 10–20.
- Уровень BSL или SSL должен быть очевидным (цена останавливалась на этом уровне минимум 2 раза).

#### Шаг 3. Фиксация Liquidity Sweep
- Цена пробивает уровень на 1–10 пунктов.
- Свеча закрывается **обратно** за уровень.
- Желательно, чтобы на свече Sweep была длинная тень (пин-бар) – это подтверждает отторжение цены.

#### Шаг 4. Подтверждения (чем больше, тем лучше)

| Подтверждение | Вес | Описание |
|---------------|-----|-----------|
| **CHoCH (смена структуры)** | Очень высокий | После Sweep формируется новый минимум выше предыдущего (для лонга) |
| **Пин-бар / поглощение** | Высокий | Свеча Sweep имеет тень в 2-3 раза больше тела |
| **Возврат в FVG/OB** | Высокий | После Sweep цена входит в зону FVG или OB |
| **Объём** | Средний | Тиковый объём на свече Sweep выше среднего на 30%+ |
| **Kill Zone** | Средний | Событие произошло в Лондонскую или Нью-Йоркскую сессию |
| **Свеча закрылась за уровень** | Обязательное | Без этого – не Sweep |

#### Шаг 5. Точка входа
- **Вариант A (агрессивный):** лимитный ордер на уровне возврата (например, сразу после закрытия свечи Sweep).  
- **Вариант B (консервативный):** ждать формирования CHoCH – следующая свеча после Sweep должна иметь минимум выше последнего минимума (для лонга) и закрытие выше максимума свечи Sweep.

**Пример конкретных цифр (лонг после SSL Sweep):**
- Уровень SSL: 1.1000 (SwingLow).
- Свеча Sweep (12:15): Low = 1.0995, High = 1.1008, Close = 1.1006.
- Вход: рыночный по 1.1006 (или лимитный 1.1002).
- Стоп-лосс: 1.0990 (на 5 пунктов ниже минимума Sweep).
- Тейк-профит 1: ближайший BSL (1.1030), R/R = (1.1030-1.1006)/(1.1006-1.0990) = 24/16 = 1.5.
- Тейк-профит 2: 1.1050 (R/R ≈ 2.75).

#### Шаг 6. Управление позицией
- Фиксация 50% на TP1, стоп переносится в безубыток.
- Остаток с трейлинг-стопом (за последним свингом).

### 7. Связь с другими инструментами из 48

| Парный инструмент | Как усиливает сигнал Sweep | Пример использования |
|-------------------|----------------------------|----------------------|
| **BSL/SSL** | Без них Sweep не существует | Sweep всегда привязан к BSL или SSL |
| **CHoCH** | Подтверждает, что разворот начался | Sweep + CHoCH = стопроцентный сигнал |
| **SFP (Swing Failure Pattern)** | Синоним Sweep, но иногда без привязки к свингам | Может использоваться как фильтр |
| **FVG / OB** | Если после Sweep цена вошла в бычий FVG – зона усилена | Лонг: Sweep SSL + бычий FVG |
| **Kill Zones** | Sweep вне Kill Zone – низкая вероятность | Торгуем только в Лондон (10-12) и НЙ (15-17) |
| **Inducement (IDM)** | Это тот же Sweep, но с фокусом на последний экстремум перед разворотом | IDM рисуется автоматически |
| **Market Maker Model** | Sweep часто является началом MMM | После Sweep цена идёт к противоположной ликвидности |
| **Turtle Soup** | Специфический вид Sweep с двумя свечами | Глава 49 |

**Практический коэффициент надёжности (по опросам SMC-трейдеров):**
- Sweep alone (только пробой и возврат) → 55-65%.
- Sweep + CHoCH → 75-85%.
- Sweep + CHoCH + Kill Zone → 85-90%.
- Sweep + CHoCH + FVG/OB + Kill Zone → 90-95%.

### 8. Психология и рыночная логика

**Почему крупные игроки охотятся за стопами?**  
1. **Ликвидность для входа:** банку нужно купить 500 контрактов. Если он начнёт покупать, цена улетит вверх. Вместо этого он толкает цену вниз, активирует стопы лонгистов (SSL), набирая позицию по более низким ценам.  
2. **Запуск стоп-ордеров для ускорения движения:** иногда крупному игроку нужно срочно выйти из позиции. Он толкает цену к BSL, активирует стопы тех, кто в шорте, и это добавляет импульс к продажам (или покупкам).  
3. **Создание ложного сигнала для алгоритмов:** многие роботы торгуют пробои. Сознательный пробой с последующим возвратом заставляет алгоритмы закрывать позиции с убытком, добавляя ликвидность.

**Эмоции трейдера во время Sweep:**  
- **До пробоя:** надежда, что цена дойдёт до тейка.  
- **Во время пробоя:** паника, желание закрыть позицию (стоп сработал).  
- **После возврата:** фрустрация, ощущение «меня обманули».  

**Правильная психология SMC-трейдера:**  
- Не ставить стопы в очевидных местах (чуть выше максимума или чуть ниже минимума).  
- Использовать отложенные ордера на вход **после** Sweep (чтобы не ловить ложные пробои).  
- Воспринимать Sweep как возможность, а не угрозу.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Вход сразу на пробое** (без ожидания возврата).  
   *Почему:* цена может продолжить движение на 20-30 пунктов и не вернуться.

2. **Игнорирование HTF тренда.**  
   *Пример:* на H4 бычий тренд, но трейдер продаёт после Sweep BSL – идёт против старшего тренда.

3. **Торговля Sweep на M1/M5.**  
   *Почему:* много шума, свинги нестабильны, частые ложные сигналы.

4. **Слишком маленький стоп** (менее 5 пунктов) или слишком большой (более 50 на M15).  
   *Рекомендация:* стоп = минимум Sweep – 5 пунктов (для лонга) или максимум Sweep + 5 пунктов (для шорта).

5. **Пропуск подтверждения CHoCH.**  
   *Без CHoCH Sweep может оказаться ложным, и цена продолжит движение в сторону пробоя.*

6. **Торговля за 30 минут до важных новостей.**  
   *В это время Sweep может быть вызван спекулянтами, а не крупными игроками.*

**Чек-лист перед входом (лонг по Sweep SSL):**

- [ ] На H4/D1 тренд восходящий (или как минимум не нисходящий).
- [ ] Чёткий SwingLow (SSL) определён в пределах последних 15 баров.
- [ ] Цена пробила SSL минимум на 1 пункт (Low < SSL).
- [ ] Свеча (или следующая) закрылась **выше** SSL.
- [ ] Имеется CHoCH (следующий минимум выше предыдущего).
- [ ] (Опционально) На уровне возврата есть бычий FVG или бычий OB.
- [ ] Время – Kill Zone (Лондон 10-12 или НЙ 15-17 серверного времени).
- [ ] Расчёт R/R: расстояние до ближайшего BSL / расстояние до стопа ≥ 2.
- [ ] Стоп установлен на 3-5 пунктов ниже минимума Sweep.
- [ ] Нет важных новостей в ближайшие 30 минут.

### 10. Продвинутые нюансы

- **Динамический порог Sweep:** вместо фиксированного порога в 1 пункт можно использовать ATR (например, `threshold = ATR(14) * 0.1`). Это адаптирует чувствительность под волатильность.

- **Сбор ликвидности на нескольких уровнях (Sweep of Sweeps):** иногда цена собирает BSL, затем возвращается, затем снова идёт к тому же BSL и только потом разворачивается. В таком случае истинный Sweep – это второй или третий пробой. Индикатор может отфильтровать первый пробой, если нет CHoCH.

- **Связь с объёмами:** на фьючерсах можно смотреть Open Interest. Если перед Sweep OI растёт, а после Sweep падает – это сильный сигнал разворота (крупные игроки закрыли позиции). В MT4/5 такого нет, но можно использовать тиковый объём как приближение.

- **Sweep на нескольких таймфреймах:** если на M15 случился Sweep, а на H4 тот же уровень является BSL/SSL – сигнал суперсильный. В индикаторе SMC Pro это не отображается автоматически, но трейдер может оценить визуально.

- **Алгоритмическая торговля:** можно написать эксперта, который открывает позицию через 1 бар после Sweep при условии CHoCH. Параметры: стоп за минимумом Sweep, тейк на противоположной ликвидности.

- **Фильтр по времени суток:** большинство Sweep, имеющих продолжение, происходит между 10:00 и 17:00 серверного времени (Лондон + НЙ). Азиатские Sweep (00:00-08:00) часто ложные.

- **Sweep в комбинации с уровнями Фибоначчи:** если после Sweep цена откатывается к уровню 0.618 (OTE) и от него идёт в сторону разворота – это идеальный вход.

### 11. Практическое задание

### 12. Резюме и шпаргалка (cheat sheet)

| Сценарий | Действие | Стоп-лосс | Тейк-профит | Дополнительно |
|----------|----------|-----------|-------------|---------------|
| **Sweep SSL** + CHoCH + бычий HTF | Покупка (лонг) | На 3-5 пунктов ниже минимума Sweep | Ближайший BSL | Фильтр: Kill Zone |
| **Sweep BSL** + CHoCH + медвежий HTF | Продажа (шорт) | На 3-5 пунктов выше максимума Sweep | Ближайший SSL | Фильтр: Kill Zone |
| Sweep без CHoCH | Бездействие | — | — | Ждать следующего сигнала |
| Sweep в азиатскую сессию | Игнорировать | — | — | Высокая вероятность ложного сигнала |
| Sweep + возврат в FVG/OB | Увеличить размер позиции | Тот же | Расширенный (1:4) | Использовать R/R 1:3+ |

**Мнемоническое правило:**  
*«Пробой уровня – не спеши, дождись возврата и CHoCH – тогда входи».*

### 13. Код для MetaTrader (пользовательская настройка)

В индикаторе SMC Pro Sweep автоматически не отрисовывается, но вы можете добавить стрелки, используя следующий код в своём эксперте или индикаторе:

```cpp
// Добавьте в OnCalculate после UpdateLastSwings
for(int i = start; i < limit; i++)
{
   if(IsLiquiditySweep(i, high, low, close, g_lastSwingHigh, g_lastSwingLow))
   {
      // Определяем тип Sweep
      bool isBSLSweep = (high[i] > g_lastSwingHigh);
      bool isSSLSweep = (low[i] < g_lastSwingLow);
      
      if(isBSLSweep && close[i] < g_lastSwingHigh)
      {
         DrawArrow(g_prefix + "SweepBSL_" + IntegerToString(time[i]), 
                   time[i], high[i], -1, clrRed, 2);
      }
      else if(isSSLSweep && close[i] > g_lastSwingLow)
      {
         DrawArrow(g_prefix + "SweepSSL_" + IntegerToString(time[i]), 
                   time[i], low[i], 1, clrGreen, 2);
      }
   }
}
```

**Настройка чувствительности:**  
Измените `InpSwingLookback` на большее значение (например, 30), чтобы реже получать сигналы, но более качественные.  
Для скальпинга используйте `lookback = 5-10`, для свинговой торговли `20-30`.

### 14. Вопросы для самопроверки (с ответами)

1. **Что такое Liquidity Sweep и в чём его отличие от обычного пробоя уровня?**  
   *Ответ:* Sweep – это пробой уровня с последующим **возвратом** цены обратно за уровень (закрытие свечи за уровнем). Обычный пробой не предполагает обязательного возврата.

2. **Назовите три обязательных условия для идентификации Sweep.**  
   *Ответ:* 1) Наличие SwingHigh или SwingLow (BSL/SSL). 2) Цена пробивает уровень на 1+ пункт. 3) Свеча (или следующая) закрывается **строго** за уровнем в противоположную сторону.

3. **Почему CHoCH является важнейшим подтверждением Sweep?**  
   *Ответ:* CHoCH показывает, что структура рынка изменилась, и разворот действительно начался. Без CHoCH цена может вернуться за уровень, а затем снова пойти в сторону пробоя.

4. **В какие сессии вероятность успешного Sweep выше и почему?**  
   *Ответ:* Лондонская (10-12) и Нью-Йоркская (15-17) – высокая ликвидность, активность крупных игроков. Азиатская сессия – низкая ликвидность, Sweep часто ложные.

5. **Какой минимальный R/R рекомендуют для сделок по Sweep?**  
   *Ответ:* Не менее 1:2, оптимально 1:3 и выше.

6. **Может ли Sweep происходить без возврата в тот же бар?**  
   *Ответ:* Да, возврат может произойти на следующем баре. Главное, чтобы закрытие произошло за уровнем.

7. **Как отличить Sweep от истинного пробоя?**  
   *Ответ:* Истинный пробой обычно сопровождается закрытием свечи **за** уровнем и продолжением движения в ту же сторону. Sweep же возвращается обратно. Также на истинном пробое часто нет длинной тени.

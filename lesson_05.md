## 📘 Глава 5. Trendline Liquidity (TL‑LQ) – ликвидность за трендовыми линиями

> *Полная расширенная глава по утверждённому шаблону (объём ~17 000 знаков)*

---

### 1. Заголовок, номер и краткое определение

**Название:** Trendline Liquidity (TL‑LQ) – ликвидность за трендовыми линиями  
**Номер:** Глава 5 (Часть 2. Ликвидность, инструмент №5 из 8)  
**Суть одной фразой:** Trendline Liquidity – это скопление стоп-лоссов и отложенных ордеров за наклонными уровнями поддержки/сопротивления (трендовыми линиями, каналами, клиньями), которые крупные игроки используют как цели для сбора ликвидности и последующего разворота.  
**Визуальное представление:** 📐🔺💧 – наклонная линия, за которой спрятана ликвидность.

---

### 2. Исторический контекст и происхождение

Трендовые линии – один из старейших инструментов технического анализа, известный ещё Чарльзу Доу в начале XX века. Классическое использование: пробой трендовой линии сигнализирует о смене тренда.

В SMC и методологии ICT концепция **Trendline Liquidity** расширяет классический подход:
- Пробой трендовой линии часто является **ложным** (sweep) – цена пробивает линию, собирает стопы, а затем возвращается обратно.
- Сама трендовая линия рассматривается не как «резиновый уровень», а как **магнит для ликвидности**. За линией (за пределами тренда) находятся стоп-лоссы трейдеров, которые открыли позиции вдоль тренда.
- Крупные игроки сознательно толкают цену **за трендовую линию**, активируют стопы и набирают позицию в противоположном направлении.

**Вклад ICT (Майкл Хаддлстон):** Он ввёл термин «Trendline Liquidity» в своих видео 2015–2016 годов, показав, что многие развороты происходят именно после ложного пробоя трендовой линии, а не на ней самой. Он также связал TL‑LQ с «Silver Bullet» и Kill Zones: ложные пробои трендовых линий в определённые часы наиболее эффективны.

**Проблема, которую решает понимание TL‑LQ:**  
Розничные трейдеры часто ставят стоп-лоссы **за трендовой линией** – чуть ниже восходящей поддержки или чуть выше нисходящего сопротивления. Крупные игроки знают эти уровни и используют их как цели. Понимание TL‑LQ позволяет:
1. Не ставить стопы в очевидных местах за трендовыми линиями.
2. Использовать ложные пробои трендовых линий как сигналы к развороту.
3. Отличать истинный пробой тренда от манипуляции.

### 3. Математическое определение и формальные условия

#### Определение трендовой линии
Пусть заданы две точки (бара) с индексами `i1` и `i2` (i1 < i2), которые являются:
- **Для восходящей линии поддержки:** два локальных минимума (SwingLow), причём второй минимум выше первого.
- **Для нисходящей линии сопротивления:** два локальных максимума (SwingHigh), причём второй максимум ниже первого.

**Уравнение восходящей линии поддержки (цена как функция времени):**
```
Price(t) = Low[i1] + (Low[i2] - Low[i1]) / (Time[i2] - Time[i1]) * (t - Time[i1])
```

**Уравнение нисходящей линии сопротивления:**
```
Price(t) = High[i1] + (High[i2] - High[i1]) / (Time[i2] - Time[i1]) * (t - Time[i1])
```

#### Определение Trendline Liquidity (TL‑LQ)
TL‑LQ – это зона **за** трендовой линией на расстоянии `δ` (обычно 1–10 пунктов) в сторону, противоположную направлению линии:

- Для **восходящей линии поддержки**: зона ликвидности находится **ниже** линии (где стоят стоп-лоссы лонгистов и отложенные ордера на продажу).
- Для **нисходящей линии сопротивления**: зона ликвидности находится **выше** линии (где стоят стоп-лоссы шортистов и отложенные ордера на покупку).

#### Условия для отрисовки TL‑LQ в индикаторе:
1. Линия построена по двум значимым свингам (минимум 5–10 баров между ними).
3. Линия продлена вправо (ray right) не менее чем на 10 баров.
3. Текущая цена находится вблизи линии (на расстоянии не более 1% ATR).
4. Индикатор отображает пунктирную линию (трендовая) и может подсвечивать зону за ней (заливкой или стрелками при пробое).

**Количественные критерии силы TL‑LQ:**
- Количество касаний линии (сколько раз цена подходила к ней, не пробивая). Чем больше касаний, тем сильнее уровень.
- Наклон линии: слишком пологая линия (менее 15 градусов) – слабая, слишком крутая (более 60 градусов) – часто ложная.
- Длина линии: чем дольше линия существует (больше баров), тем больше ликвидности накопилось за ней.

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

**Псевдокод:**
```
Функция DrawTrendlineLiquidity(массив high, low, time):
    Найти последний SwingLow idx1 (для восходящей)
    Найти предпоследний SwingLow idx2 (более ранний)
    Если оба найдены и low[idx1] > low[idx2]:
        Нарисовать линию от time[idx2] до time[idx1], продлить вправо
        Это трендовая поддержка
        Зона ликвидности: ниже линии на 5-10 пунктов
    
    Аналогично для нисходящей линии сопротивления по SwingHigh
```

**Реальный код из индикатора SMC Pro (функция `DrawTrendlineLiquidity`):**

```cpp
void DrawTrendlineLiquidity(const datetime &time[], const double &high[], const double &low[], int totalBars)
{
   // Удаляем старые линии
   for(int i = ObjectsTotal(0,0,-1)-1; i >= 0; i--)
   {
      string name = ObjectName(0,i);
      if(StringFind(name, g_prefix + "TL_LQ_") == 0)
         ObjectDelete(0, name);
   }
   
   int lastLowIdx1 = -1, lastLowIdx2 = -1;
   int lastHighIdx1 = -1, lastHighIdx2 = -1;
   
   // Ищем два последних SwingLow для восходящей линии
   for(int i = totalBars-2; i >= 2; i--)
   {
      if(IsSwingLow(i, low, 5))
      {
         if(lastLowIdx1 == -1) { lastLowIdx1 = i; continue; }
         if(lastLowIdx2 == -1) { lastLowIdx2 = i; break; }
      }
   }
   // Ищем два последних SwingHigh для нисходящей линии
   for(int i = totalBars-2; i >= 2; i--)
   {
      if(IsSwingHigh(i, high, 5))
      {
         if(lastHighIdx1 == -1) { lastHighIdx1 = i; continue; }
         if(lastHighIdx2 == -1) { lastHighIdx2 = i; break; }
      }
   }
   
   // Рисуем восходящую линию поддержки (Low-Low)
   if(lastLowIdx1 > 0 && lastLowIdx2 > 0 && low[lastLowIdx1] > low[lastLowIdx2])
   {
      string name = g_prefix + "TL_LQ_LL";
      datetime t1 = time[lastLowIdx2], t2 = time[lastLowIdx1];
      double p1 = low[lastLowIdx2], p2 = low[lastLowIdx1];
      // Обновляем или создаём трендовую линию, продлённую вправо
      if(ObjectFind(0, name) < 0)
         ObjectCreate(0, name, OBJ_TREND, 0, t1, p1, t2, p2);
      else
      {
         ObjectSetInteger(0, name, OBJPROP_TIME1, t1);
         ObjectSetDouble(0, name, OBJPROP_PRICE1, p1);
         ObjectSetInteger(0, name, OBJPROP_TIME2, t2);
         ObjectSetDouble(0, name, OBJPROP_PRICE2, p2);
      }
      ObjectSetInteger(0, name, OBJPROP_RAY_RIGHT, true);
      ObjectSetInteger(0, name, OBJPROP_COLOR, LIQ_COLOR_TL);
      ObjectSetInteger(0, name, OBJPROP_STYLE, STYLE_DASH);
      ObjectSetInteger(0, name, OBJPROP_WIDTH, 1);
      ObjectSetInteger(0, name, OBJPROP_BACK, true);
   }
   
   // Рисуем нисходящую линию сопротивления (High-High)
   if(lastHighIdx1 > 0 && lastHighIdx2 > 0 && high[lastHighIdx1] < high[lastHighIdx2])
   {
      string name = g_prefix + "TL_LQ_HH";
      datetime t1 = time[lastHighIdx2], t2 = time[lastHighIdx1];
      double p1 = high[lastHighIdx2], p2 = high[lastHighIdx1];
      if(ObjectFind(0, name) < 0)
         ObjectCreate(0, name, OBJ_TREND, 0, t1, p1, t2, p2);
      else
      {
         ObjectSetInteger(0, name, OBJPROP_TIME1, t1);
         ObjectSetDouble(0, name, OBJPROP_PRICE1, p1);
         ObjectSetInteger(0, name, OBJPROP_TIME2, t2);
         ObjectSetDouble(0, name, OBJPROP_PRICE2, p2);
      }
      ObjectSetInteger(0, name, OBJPROP_RAY_RIGHT, true);
      ObjectSetInteger(0, name, OBJPROP_COLOR, LIQ_COLOR_TL);
      ObjectSetInteger(0, name, OBJPROP_STYLE, STYLE_DASH);
      ObjectSetInteger(0, name, OBJPROP_WIDTH, 1);
      ObjectSetInteger(0, name, OBJPROP_BACK, true);
   }
}
```

**Расширенная версия с отрисовкой зоны ликвидности (заливка за линией):**

```cpp
void DrawTL_LiquidityZone(string lineName, double linePriceAtTime, datetime time, int direction)
{
   // direction: 1 – зона ниже линии (ликвидность для восходящей), -1 – зона выше линии (для нисходящей)
   double zoneOffset = _Point * 10; // 10 пунктов
   string zoneName = g_prefix + "TL_LQ_ZONE_" + lineName;
   datetime t1 = time, t2 = time + PeriodSeconds(PERIOD_CURRENT)*10; // зона на 10 баров вперёд
   double p1, p2;
   if(direction == 1) // зона ниже
   {
      p1 = linePriceAtTime - zoneOffset;
      p2 = linePriceAtTime - zoneOffset * 2;
   }
   else // зона выше
   {
      p1 = linePriceAtTime + zoneOffset;
      p2 = linePriceAtTime + zoneOffset * 2;
   }
   DrawRectangleZone(zoneName, t1, p1, t2, p2, LIQ_COLOR_TL, 100);
}
```

### 5. Как это выглядит на графике

#### Сценарий 1. Восходящая трендовая линия (поддержка) и ликвидность ниже неё

**График GBPUSD H1:**
- Цена формирует два минимума: 1.2500 (в понедельник) и 1.2520 (во вторник). Через них проводится восходящая линия поддержки.
- Индикатор рисует **жёлтую пунктирную линию**, продлённую вправо.
- **Зона ликвидности:** ниже линии на 5–10 пунктов – там скоплены стоп-лоссы тех, кто открыл длинные позиции вдоль тренда, и отложенные ордера на продажу.

**Развитие событий (типичный sweep TL‑LQ):**
1. В среду цена подходит к линии поддержки.
2. Пробивает её вниз на 5 пунктов (ниже линии) – стопы лонгистов активированы.
3. Цена закрывается **выше** линии (возврат) – ложный пробой.
4. Затем резко растёт на 50 пунктов.

#### Сценарий 2. Нисходящая трендовая линия (сопротивление) и ликвидность выше неё

**График USDJPY M30:**
- Два максимума: 148.00 и 147.80 – нисходящая линия сопротивления.
- Жёлтая пунктирная линия, продлённая вправо.
- **Зона ликвидности:** выше линии на 5–10 пунктов – стоп-лоссы шортистов.

**Типичный паттерн:** цена пробивает линию вверх на 3–5 пунктов, собирает стопы, затем возвращается ниже линии и падает.

#### Как всё это отображает индикатор SMC Pro:
- Трендовая линия – жёлтая пунктирная (цвет `LIQ_COLOR_TL`).
- Сама зона ликвидности за линией **не заливается по умолчанию**, но может быть дорисована пользовательским индикатором.
- При пробое линии (sweep) и возврате можно нарисовать стрелку (аналогично Liquidity Sweep).

*(В книге – скриншот с восходящим трендовым каналом, пунктирной линией поддержки и стрелкой ложного пробоя вниз.)*

### 6. Торговая тактика (полный цикл сделки)

Существует два подхода к торговле TL‑LQ:
- **А) Вход на пробое** (игра на продолжение) – после истинного пробоя трендовой линии.
- **Б) Разворотная стратегия** (игра на ложном пробое) – после sweep и возврата.

В SMC предпочтение отдаётся **разворотной стратегии**, так как она даёт лучшее соотношение риск/прибыль и совпадает с действиями крупных игроков. Рассмотрим её.

---

#### Стратегия «Sweep TL‑LQ + CHoCH»

**Шаг 1. Построение трендовой линии**
- На H1 или M15 находим два значимых свинга (минимума для поддержки, максимума для сопротивления).
- Линия должна иметь минимум 2 касания (идеально 3–4) без пробоя.
- Чем дольше линия существует, тем сильнее сигнал.

**Шаг 2. Определяем направление HTF тренда**
- Для восходящей линии поддержки: если H4/D1 в восходящем тренде, то сигнал на покупку после ложного пробоя вниз – **очень сильный**.
- Для нисходящей линии сопротивления: если H4/D1 в нисходящем тренде, сигнал на продажу после ложного пробоя вверх – сильный.

**Шаг 3. Ждём пробой линии**
- Цена пробивает линию на 3–10 пунктов (sweep).
- **Ключевое условие:** свеча (или следующая) закрывается **обратно** за линию (например, при пробое поддержки вниз – закрытие выше линии).

**Шаг 4. Подтверждения**

| Подтверждение | Вес | Пояснение |
|---------------|-----|-----------|
| **CHoCH** | Очень высокий | После возврата формируется новый минимум выше (для лонга) или новый максимум ниже (для шорта) |
| **Длинная тень (пин-бар)** | Высокий | Свеча, пробившая линию, имеет тень в сторону пробоя, тело – за линией |
| **Возврат в FVG/OB** | Высокий | Цена после пробоя входит в бычий FVG (для лонга) или медвежий FVG (для шорта) |
| **Объём** | Средний | Тиковый объём на свече пробоя выше среднего на 30%+ |
| **Kill Zone** | Средний | Пробой произошёл в Лондон (10-12) или Нью-Йорк (15-17) |
| **Множественные касания линии** | Высокий | Линия касалась цены 3+ раз – сигнал сильнее |

**Шаг 5. Точка входа**

*Пример для восходящей поддержки (покупка после ложного пробоя вниз):*
- Уровень линии в момент пробоя: 1.2500.
- Свеча Sweep: Low = 1.2495, Close = 1.2503.
- **Вход:** рыночный на закрытии свечи (1.2503) или лимитный на откате к 1.2500.
- **Стоп-лосс:** на 3–5 пунктов ниже минимума Sweep (1.2490).
- **Тейк-профит:** ближайший значимый уровень сопротивления (EQH, предыдущий максимум) или R/R 1:3. Ориентир: расстояние до следующей трендовой линии (если она есть) или до противоположной ликвидности.

*Пример для нисходящего сопротивления (продажа после ложного пробоя вверх):*
- Линия сопротивления: 148.00.
- Sweep: High = 148.06, Close = 147.98.
- Вход: продажа по 147.98.
- Стоп: 148.10.
- Тейк: 147.50.

**Шаг 6. Управление позицией**
- Первая цель: расстояние, равное стопу (1:1), частичная фиксация 50%.
- Вторая цель: 1:2 или 1:3.
- Трейлинг-стоп за последним свингом.

### 8. Психология и рыночная логика

**Почему крупные игроки охотятся за ликвидностью за трендовыми линиями?**

1. **Предсказуемость стопов:** розничные трейдеры почти всегда ставят стоп-лоссы «чуть ниже поддержки» (в восходящем тренде) и «чуть выше сопротивления» (в нисходящем). Это самые очевидные уровни.
2. **Маскировка своих ордеров:** толкая цену за линию, крупный игрок активирует стопы и создаёт дополнительное движение, под которым он может набрать позицию.
3. **Запуск алгоритмов:** многие роботы торгуют пробои линий. Сознательный ложный пробой заставляет их закрывать позиции с убытком, добавляя ликвидность.

**Эмоции трейдера у трендовой линии:**
- **До подхода к линии:** надежда, что линия удержит.
- **При пробое:** страх, паника, желание закрыть позицию (если стоп сработал) или войти в направлении пробоя.
- **После возврата:** фрустрация, ощущение обмана.

**Правильная психология:**  
Воспринимать пробой трендовой линии как **возможность**, а не угрозу. Ждать возврата и CHoCH – именно в этот момент крупные игроки уже вошли, и можно присоединяться.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Вход сразу на пробое** (не дожидаясь возврата).  
   *Почему:* часто пробой оказывается ложным, и цена возвращается.

2. **Построение линии по слишком близким точкам** (менее 5 баров).  
   *Линия будет неустойчивой, частые ложные сигналы.*

3. **Игнорирование угла наклона.**  
   *Слишком крутая линия (>60°) часто пробивается истинно, слишком пологая (<15°) – слабая.*

4. **Торговля против HTF тренда.**  
   *Если H4 в восходящем тренде, то ложный пробой поддержки – сильный сигнал на покупку. А ложный пробой сопротивления – слабый (может быть истинный пробой).*

5. **Отсутствие CHoCH после возврата.**  
   *Без CHoCH не входить.*

6. **Торговля на слишком маленьком ТФ (M1-M5).**  
   *Линии на малых ТФ шумные, частые ретесты.*

7. **Использование линии только по двум точкам.**  
   *Линия с 3-4 касаниями значительно сильнее.*

**Чек-лист перед входом (лонг после ложного пробоя восходящей поддержки):**

- [ ] Трендовая линия построена по 2+ минимумам, расстояние между точками ≥ 5 баров.
- [ ] Линия касалась цены минимум 2 раза (лучше 3+).
- [ ] H4/D1 тренд восходящий или как минимум не нисходящий.
- [ ] Цена пробила линию вниз на 1+ пункт (Low < значение линии).
- [ ] Свеча (или следующая) закрылась **выше** линии.
- [ ] Сформировался CHoCH вверх (следующий минимум выше предыдущего).
- [ ] (Опционально) Есть бычий FVG или бычий OB в зоне возврата.
- [ ] Время – Kill Zone (Лондон 10-12 или НЙ 15-17).
- [ ] R/R ≥ 1:2.
- [ ] Стоп установлен на 3-5 пунктов ниже минимума Sweep.
- [ ] Нет важных новостей в ближайшие 30 минут.

### 10. Продвинутые нюансы

- **Наклон линии и волатильность:** используйте ATR для настройки зоны ликвидности. Для валют с высокой волатильностью (GBPJPY) зона может быть 10–15 пунктов, для EURUSD – 5–7.

- **Линии по High/Low или по Close?** В SMC принято строить линии по теням (High/Low), так как именно там находятся стопы. Но для фильтрации шума можно использовать Close.

- **Каналы (вторая параллельная линия):** зона ликвидности может быть не только за одной линией, но и между линиями канала. Индикатор SMC Pro не рисует каналы автоматически, но трейдер может провести их вручную.

- **Временной фактор:** чем дольше существует линия, тем больше ликвидности накопилось. Линия, сформированная за несколько дней, сильнее, чем линия за несколько часов.

- **Совпадение с круглыми уровнями:** если трендовая линия пересекает круглый уровень (1.1000, 150.00) в точке пробоя – это дополнительный магнит.

- **Алгоритмическая торговля:** можно создать эксперта, который строит линии по последним двум свингам и даёт сигнал при пробое+возврате+CHoCH. Параметры: стоп за sweep, тейк на противоположную линию канала или EQH/EQL.

- **Фильтр по расстоянию от линии:** не входить, если цена отошла от линии более чем на 1.5 ATR после пробоя (вероятность, что возврат уже не произойдёт).

- **Мульти-таймфрейм:** если линия на M15 совпадает с линией на H1 – зона экстремально сильная.

### 12. Резюме и шпаргалка (cheat sheet)

| Сценарий | Действие | Стоп-лосс | Тейк-профит | Дополнительно |
|----------|----------|-----------|-------------|---------------|

### 13. Код для MetaTrader (пользовательская настройка)

В индикаторе SMC Pro отображение Trendline Liquidity включается параметром `InpShowTrendlineLiq`. Цвет линии – `LIQ_COLOR_TL` (по умолчанию жёлтый).

**Изменение стиля линии:**
```cpp
#define LIQ_COLOR_TL   clrYellow      // цвет линии
ObjectSetInteger(0, name, OBJPROP_STYLE, STYLE_DASH); // стиль (DASH, DOT, SOLID)
ObjectSetInteger(0, name, OBJPROP_WIDTH, 1);          // толщина
```

**Добавление звукового оповещения при ложном пробое линии:**
```cpp
bool IsFalseBreakofTL(int i, double lineValue, const double &low[], const double &close[])
{
   // lineValue – значение линии в момент i
   if(low[i] < lineValue - _Point*2 && close[i] > lineValue)
      return true;
   return false;
}
// Вызов в OnCalculate:
if(IsFalseBreakofTL(i, lineSupport, low, close))
   SendAlert("Ложный пробой трендовой поддержки на " + Symbol());
```

**Настройка количества баров для построения линии:**
Увеличьте `for(int i = totalBars-2; i >= 2; i--)` до `>= 10` или `>= 20`, чтобы игнорировать ближайшие свинги и строить линию по более долгосрочным.

### 14. Вопросы для самопроверки (с ответами)

1. **Что такое Trendline Liquidity (TL‑LQ)?**  
   *Ответ:* Зона за трендовой линией (ниже поддержки или выше сопротивления), где скоплены стоп-лоссы трейдеров.

2. **Почему ложный пробой трендовой линии часто приводит к развороту?**  
   *Ответ:* Потому что крупные игроки собирают ликвидность (стопы) и набирают позицию в противоположном направлении.

3. **Какое минимальное количество касаний линии рекомендуется для надёжного сигнала?**  
   *Ответ:* 2 касания – минимум, 3+ – значительно надёжнее.

4. **Почему CHoCH является важным подтверждением после пробоя?**  
   *Ответ:* CHoCH подтверждает, что структура рынка изменилась и разворот начался.

5. **В какие сессии наиболее эффективны ложные пробои трендовых линий?**  
   *Ответ:* Лондонская (10-12) и Нью-Йоркская (15-17) из-за высокой активности институциональных игроков.

6. **Какой угол наклона трендовой линии считается «слабым»?**  
   *Ответ:* Менее 15 градусов (пологая) – мало ликвидности; более 60 градусов (крутая) – часто истинный пробой.

7. **Как отличить ложный пробой от истинного?**  
   *Ответ:* Истинный пробой сопровождается закрытием свечи **за** линией и продолжением движения в ту же сторону. Ложный – возвратом за линию.

8. **Какова минимальная рекомендуемая длина линии (по времени) для надёжного сигнала?**  
   *Ответ:* Не менее 10–15 баров на H1 (т.е. полдня-день).

9. **Какие два дополнительных инструмента SMC усиливают сигнал TL‑LQ?**  
   *Ответ:* CHoCH и FVG/OB. Kill Zone – третий важный фильтр.

10. **Какой минимальный R/R рекомендуется для сделок по TL‑LQ?**  
    *Ответ:* Не менее 1:2, оптимально 1:3 и выше.

## 📘 Глава 9. Fair Value Gap (FVG) – бычий и медвежий

> *Полная расширенная глава по утверждённому шаблону (объём ~17 000 знаков)*

---

### 1. Заголовок, номер и краткое определение

**Название:** Fair Value Gap (FVG) – бычий и медвежий  
**Номер:** Глава 9 (Часть 3. Дисбаланс, инструмент №1 из 5)  
**Суть одной фразой:** Fair Value Gap (FVG) – это ценовой разрыв между первой и третьей свечой импульсного движения, который возникает из‑за агрессивного входа крупных игроков и впоследствии часто заполняется (ретестируется) ценой, предоставляя зону для входа в направлении первоначального импульса.  
**Визуальное представление:** 🔵🔴📊 – синий прямоугольник (бычий FVG) и красный прямоугольник (медвежий FVG) на графике.

---

### 2. Исторический контекст и происхождение

Понятие «ценовой разрыв» (gap) известно с самого начала биржевой торговли. Однако в контексте SMC и ICT **Fair Value Gap** был формализован и популяризирован **Майклом Хаддлстоном** примерно в 2012–2014 годах.  

До него трейдеры использовали термин «imbalance» (дисбаланс) или «gap», но ICT предложил чёткое определение: FVG – это зона между максимумом первой свечи и минимумом третьей свечи (для бычьего) или между минимумом первой и максимумом третьей (для медвежьего). Он также ввёл понятие **BPR (Balanced Price Range)** – пересечение двух разнонаправленных FVG.

**Почему FVG важен?**  
В нормальных рыночных условиях цена движется непрерывно, касаясь каждого тика. Когда возникает FVG, это означает, что крупный игрок (или группа игроков) настолько агрессивно вошёл в рынок, что «перепрыгнул» через некоторые ценовые уровни, не дав им исполниться. Эти пропущенные уровни становятся зонами, куда цена **должна вернуться**, чтобы сбалансировать спрос и предложение (отсюда название «Fair Value» – справедливая стоимость). Понимание FVG позволяет:
1. Идентифицировать зоны, где крупные игроки входили в рынок.
2. Торговать на возврате к этим зонам с высоким соотношением риск/прибыль.
3. Отличать истинные импульсы от ложных.

---

### 3. Математическое определение и формальные условия

#### Бычий FVG (Bullish FVG)
Рассмотрим три последовательные свечи с индексами `i-2`, `i-1`, `i` (где `i` – самая правая, третья свеча).  
**Условия:**
1. Свеча `i-2` – бычья (close > open) – не обязательно, но желательно.
2. Свеча `i` – бычья (close > open).
3. **Минимум свечи `i` выше максимума свечи `i-2`:**  
   `low[i] > high[i-2]`

**Зона бычьего FVG:**  
- Верхняя граница = `high[i-2]` (максимум первой свечи)
- Нижняя граница = `low[i]` (минимум третьей свечи)
- Прямоугольник рисуется от времени `i-2` до времени `i` (или от `i-2` до `i`, захватывая три свечи).

#### Медвежий FVG (Bearish FVG)
**Условия:**
1. Свеча `i-2` – медвежья (close < open) – не обязательно.
2. Свеча `i` – медвежья (close < open).
3. **Максимум свечи `i` ниже минимума свечи `i-2`:**  
   `high[i] < low[i-2]`

**Зона медвежьего FVG:**  
- Верхняя граница = `low[i-2]` (минимум первой свечи)
- Нижняя граница = `high[i]` (максимум третьей свечи)

#### Дополнительные условия для отрисовки в индикаторе:
- Минимальный размер FVG: обычно не менее 1 пункта (для 5-знака). Слишком маленькие FVG (менее 2-3 пунктов) игнорируются или объединяются.
- FVG считается **активным**, пока цена не закроется **за** его границей (т.е. ниже нижней границы для бычьего, выше верхней – для медвежьего) или не заполнит его полностью.
- FVG может быть **инвертирован** (IFVG) – об этом глава 13.

#### Пример с реальными числами (EURUSD, 5-знак):
- Свеча 1 (i-2): High = 1.10500, Low = 1.10250 (бычья)
- Свеча 2 (i-1): High = 1.10480, Low = 1.10300
- Свеча 3 (i): High = 1.10620, Low = 1.10520 (бычья)  
  Проверка: `low[i] = 1.10520 > high[i-2] = 1.10500` → бычий FVG от 1.10500 до 1.10520.

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

**Псевдокод:**
```
Для каждого бара i от 2 до rates_total-1:
    если low[i] > high[i-2]:
        // Бычий FVG
        t1 = time[i-2]
        t2 = time[i]
        p_low = high[i-2]
        p_high = low[i]
        Нарисовать прямоугольник (t1, p_low, t2, p_high) синим с прозрачностью 30%
        Сохранить в массив fvgZones
    если high[i] < low[i-2]:
        // Медвежий FVG
        p_low = high[i]
        p_high = low[i-2]
        Нарисовать прямоугольник красным
```

**Реальный код из индикатора SMC Pro (функции `IsBullishFVG`, `IsBearishFVG`, `DrawOrUpdateFVG`):**

```cpp
// Определение бычьего FVG
bool IsBullishFVG(int i, const double &high[], const double &low[])
{
   if(i < 2) return false;
   return (low[i] > high[i-2]);
}

// Определение медвежьего FVG
bool IsBearishFVG(int i, const double &high[], const double &low[])
{
   if(i < 2) return false;
   return (high[i] < low[i-2]);
}

// Отрисовка или обновление зоны FVG
void DrawOrUpdateFVG(int i, const datetime &time[], const double &high[], const double &low[], int type, bool isMTF)
{
   double priceLow, priceHigh;
   datetime t1 = time[i], t2 = time[i-2];
   if(type == 1)  // бычий
   {
      priceLow = high[i-2];   // верхняя граница (нижняя по цене? Нет, в бычьем low[i] > high[i-2], значит high[i-2] – нижняя граница?)
      priceHigh = low[i];      // верхняя граница
      // Внимание: в классическом определении бычий FVG – это зона между high[i-2] (нижняя граница) и low[i] (верхняя граница), 
      // так как low[i] > high[i-2]. При отрисовке прямоугольника порядок p1, p2 не важен, важны значения.
   }
   else  // медвежий
   {
      priceLow = high[i];      // нижняя граница
      priceHigh = low[i-2];    // верхняя граница
   }
   // Проверка, чтобы цена не была перевёрнута
   if(priceLow >= priceHigh) return;
   
   color clr = (type == 1) ? clrDodgerBlue : clrTomato;
   int alpha = isMTF ? g_mtfAlpha : g_fvgAlpha;
   string suffix = isMTF ? "MTF" : "";
   string name = g_prefix + "FVG_" + suffix + "_" + IntegerToString(time[i]);
   // Рисуем прямоугольник от времени первой свечи до времени третьей
   DrawRectangleZone(name, time[i-2], priceLow, time[i], priceHigh, clr, alpha);
   
   // Сохраняем в массив (для последующего использования в других модулях)
   FVGZone zone;
   zone.time1 = time[i-2];
   zone.time2 = time[i];
   zone.priceLow = priceLow;
   zone.priceHigh = priceHigh;
   zone.type = type;
   zone.active = true;
   zone.drawn = true;
   if(isMTF)
   {
      ArrayResize(fvgZonesMTF, ArraySize(fvgZonesMTF)+1);
      fvgZonesMTF[ArraySize(fvgZonesMTF)-1] = zone;
   }
   else
   {
      ArrayResize(fvgZones, ArraySize(fvgZones)+1);
      fvgZones[ArraySize(fvgZones)-1] = zone;
   }
}
```

**Нюанс реализации:** в коде выше `priceLow` и `priceHigh` могут быть перепутаны местами с точки зрения визуализации, но функция `DrawRectangleZone` принимает их в любом порядке и корректно отрисовывает прямоугольник (так как она строит от `p1` до `p2`, не требуя, чтобы `p1` было меньше `p2`). В классическом понимании бычий FVG имеет нижнюю границу `high[i-2]` и верхнюю `low[i]`.

### 5. Как это выглядит на графике

#### Сценарий 1. Бычий FVG на EURUSD H1

**Ситуация:**
- На графике три свечи: первая – бычья с High=1.1050, третья – бычья с Low=1.1055.
- Между 1.1050 и 1.1055 – разрыв.
- Индикатор рисует **синий полупрозрачный прямоугольник** от времени первой свечи до времени третьей, охватывающий цены от 1.1050 до 1.1055.

**Последующее развитие:**  
Цена часто возвращается в этот прямоугольник (обычно в течение следующих 10-20 свечей), отскакивает от его нижней границы и продолжает рост.

#### Сценарий 2. Медвежий FVG на GBPUSD H1

- Первая свеча – медвежья с Low=1.2500, третья – медвежья с High=1.2495.
- Разрыв от 1.2495 до 1.2500.
- Индикатор рисует **красный полупрозрачный прямоугольник**.
- Цена возвращается в зону (к верхней границе 1.2500), отскакивает и падает.

#### Как отображается в индикаторе SMC Pro:
- Цвета: синий для бычьего, красный для медвежьего.
- Прозрачность регулируется параметром `InpFVGOpacity` (0 – прозрачный, 1 – сплошной). По умолчанию 0.3 (30%).
- Прямоугольник рисуется между барами `i-2` и `i` (охватывает все три свечи).
- Если FVG инвертируется (IFVG), стиль линии меняется на штриховой (глава 13).

*(В книге – два скриншота: бычий FVG с последующим ретестом и отскоком, медвежий FVG с ретестом и падением.)*

### 6. Торговая тактика (полный цикл сделки)

FVG – это **зона отложенного ордера** (limit order). Трейдеры обычно ждут возврата цены в зону FVG и входят в направлении первоначального импульса.

#### Стратегия «FVG ретест» (классическая)

**Шаг 1. Идентифицируем FVG после сильного импульса**
- На любом таймфрейме (лучше M15-H1) находим бычий или медвежий FVG.
- Обращаем внимание на размер FVG: он должен быть заметным (минимум 3-5 пунктов, лучше 10-20).

**Шаг 2. Определяем контекст (HTF)**
- Для бычьего FVG: желательно, чтобы HTF тренд был восходящим или хотя бы не нисходящим.
- Для медвежьего FVG: HTF тренд нисходящий.

**Шаг 3. Ждём возврата цены в зону FVG**
- Цена должна коснуться **нижней границы** бычьего FVG (или слегка пройти её, но не залетать глубоко внутрь) и отскочить вверх.
- Для медвежьего FVG – касание **верхней границы**.

**Шаг 4. Подтверждения**

| Подтверждение | Вес | Пояснение |
|---------------|-----|-----------|
| **Свечной паттерн** (пин-бар, поглощение) | Высокий | Свеча, коснувшаяся границы, имеет длинную тень и закрывается в сторону от FVG |
| **Объём** | Средний | На касании объём выше среднего (показывает интерес) |
| **CHoCH** | Высокий | После касания цена формирует новый экстремум в сторону движения |
| **Kill Zone** | Средний | Касание произошло в Лондон/НЙ – повышает вероятность |
| **Совпадение с OB/MB** | Очень высокий | Если граница FVG совпадает с Order Block или Mitigation Block |

**Шаг 5. Точка входа**

*Пример для бычьего FVG (покупка):*
- Нижняя граница FVG = 1.1050.
- Цена коснулась 1.1050 (или чуть ниже, до 1.1049), свеча закрылась на 1.1053 (пин-бар).
- **Вход:** рыночный на закрытии этой свечи (1.1053) или лимитный ордер на 1.1051.
- **Стоп-лосс:** на 3-5 пунктов ниже нижней границы FVG (1.1045).
- **Тейк-профит:** ближайший уровень ликвидности (BSL) или расширение Фибоначчи (127.2% или 161.8% от движения). Часто используют R/R 1:2 или 1:3.

*Пример для медвежьего FVG (продажа):*
- Верхняя граница = 1.2500.
- Цена коснулась 1.2501, свеча закрылась 1.2498.
- Вход: продажа 1.2498.
- Стоп: 1.2505.
- Тейк: 1.2470 (R/R 1:2.5).

**Шаг 6. Управление позицией**
- Фиксация части (50%) при достижении 1:1.
- Остаток с трейлинг-стопом за последним свингом.
- Если цена входит в FVG, но не выходит из него (застревает), лучше закрыть с малым убытком или пересидеть – часто после консолидации внутри FVG происходит сильное движение.

### 7. Связь с другими инструментами из 48

| Парный инструмент | Как усиливает FVG | Пример |
|-------------------|-------------------|--------|
| **Order Block (OB)** | Если граница FVG совпадает с OB – зона экстремально сильная | Бычий FVG + бычий OB на одной цене |
| **Mitigation Block (MB)** | MB часто является границей FVG | Ретест MB внутри FVG |
| **CHoCH** | CHoCH после касания FVG подтверждает разворот | Вход после CHoCH |
| **Liquidity Sweep** | Часто sweep происходит на границе FVG | Сбор ликвидности + FVG = двойная защита |
| **Kill Zones** | FVG, сформированный в Kill Zone, чаще ретестируется | London FVG надёжнее Asian |
| **Optimal Trade Entry (OTE)** | FVG на уровне 61.8-78.6% коррекции – суперсильный | OTE + FVG |
| **BPR** | Пересечение двух FVG создаёт мощную зону | Глава 16 |

**Коэффициент надёжности (по опросам):**
- FVG alone (просто касание границы) – 55-65%.
- FVG + свечной паттерн – 70-75%.
- FVG + свечной паттерн + Kill Zone – 80-85%.
- FVG + OB/MB + CHoCH – 85-90%.
- FVG + OB + CHoCH + Kill Zone – 90-95%.

### 8. Психология и рыночная логика

**Почему FVG работают?**  

1. **Дисбаланс ордеров:** При формировании FVG крупный игрок покупает (или продаёт) так агрессивно, что пропускает некоторые уровни. Эти уровни становятся «несправедливыми» – на них не исполнились ордера. Рынок стремится вернуться и заполнить эти пробелы, чтобы сбалансировать спрос и предложение.
2. **Магнит для стоп-лоссов:** Многие трейдеры ставят стопы как раз за границами FVG. Крупные игроки, зная это, могут толкнуть цену к FVG, собрать стопы, а затем продолжить движение.
3. **Зоны интереса алгоритмов:** Многие HFT-алгоритмы запрограммированы на поиск дисбалансов и ретест FVG. Это создаёт самоподтверждающийся паттерн.

**Эмоции трейдера, когда цена подходит к FVG:**
- **Страх:** «А вдруг пробьёт дальше?»
- **Сомнение:** «Может, войти прямо сейчас?»
- **Желание усредниться,** если позиция уже открыта.

**Правильная психология SMC:**
- Ждать касания границы FVG, а не входить наугад.
- Использовать лимитные ордера на границе (а не рыночные при касании, чтобы получить лучшее исполнение).
- Не бояться, если цена чуть пробила границу – часто это ложный пробой (sweep) перед разворотом.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Вход в FVG на первом касании без подтверждения.**  
   *Почему:* цена может пройти FVG насквозь, особенно если нет свечного паттерна.

2. **Использование слишком маленьких FVG** (менее 3 пунктов).  
   *Почему:* низкая надёжность, высокий риск проскальзывания.

3. **Торговля FVG против HTF тренда.**  
   *Пример:* бычий FVG на M15, но H4 в нисходящем тренде. Возврат может быть краткосрочным.

4. **Игнорирование Kill Zone.**  
   *FVG, сформированный в азиатскую сессию, часто не ретестируется или ретестируется ложно.*

5. **Слишком близкий стоп** (прямо на границе).  
   *Рекомендуется ставить на 3-5 пунктов за границу, чтобы учесть шум.*

6. **Вход в FVG без CHoCH.**  
   *Без смены структуры цена может просто консолидироваться внутри FVG и затем пойти дальше.*

**Чек-лист перед входом (лонг по бычьему FVG):**

- [ ] FVG сформирован (нижняя граница = high[i-2], верхняя = low[i]).
- [ ] Цена коснулась нижней границы (или чуть ниже, но не более чем на 2-3 пункта).
- [ ] Свеча касания имеет длинную нижнюю тень (пин-бар) или является бычьим поглощением.
- [ ] (Опционально) На границе FVG есть бычий OB или MB.
- [ ] CHoCH вверх подтверждён (после касания появился более высокий минимум).
- [ ] H4/D1 не нисходящий (лучше восходящий).
- [ ] Время – Kill Zone (Лондон 10-12, НЙ 15-17) или хотя бы не азиатская сессия.
- [ ] R/R не менее 1:2.
- [ ] Стоп установлен на 3-5 пунктов ниже нижней границы.
- [ ] Нет важных новостей в ближайшие 30 минут.

### 10. Продвинутые нюансы

- **Множественные FVG:** Иногда формируется несколько FVG подряд (каскад). Сила сигнала увеличивается, но более поздние FVG (ближе к текущей цене) более актуальны.

- **FVG на разных таймфреймах:** FVG на H4 значительно сильнее, чем на M15. Если FVG на M15 совпадает с FVG на H1 или H4 – это «супер-зона».

- **Частичное заполнение FVG:** Цена может войти в FVG, пройти его часть и развернуться, не заполнив полностью. Это всё равно считается ретестом. Полное заполнение (закрытие за противоположной границей) аннулирует FVG.

- **FVG и новости:** Перед важными новостями FVG часто заполняются за несколько минут до выхода – это «подготовка». Входить в такой момент рискованно.

- **Алгоритмическая торговля FVG:** Можно создать эксперта, который ставит лимитный ордер на границе FVG со стопом за ней и тейком на противоположной ликвидности. В SMC Pro такой логики нет, но её легко добавить.

- **FVG как цель тейк-профита:** При движении вверх можно ставить тейк на нижнюю границу вышестоящего медвежьего FVG (как сопротивление). И наоборот.

- **Динамическая настройка прозрачности:** В SMC Pro вы можете менять `InpFVGOpacity`. Для дневного трейдинга лучше 0.2-0.3, чтобы не загромождать график. Для скальпинга можно сделать 0.4-0.5.

### 12. Резюме и шпаргалка (cheat sheet)

| Тип FVG | Условие | Вход | Стоп-лосс | Тейк-профит |
|---------|---------|------|-----------|-------------|
| **Бычий** | low[i] > high[i-2] | На ретесте нижней границы (пин-бар/поглощение) | На 3-5 п. ниже нижней границы | Ближайший BSL или 1:2 |
| **Медвежий** | high[i] < low[i-2] | На ретесте верхней границы (пин-бар/поглощение) | На 3-5 п. выше верхней границы | Ближайший SSL или 1:2 |
| FVG без подтверждения | - | Не входить | - | - |
| FVG в Kill Zone | - | Увеличить размер | Стандартный | Расширенный (1:3) |
| FVG против HTF тренда | - | Уменьшить размер (0.5x) | Шире на 30% | Уменьшить |

**Мнемоническое правило:**  
*«Между первой и третьей – дыра, возвращайся туда, как пришла пора».*

### 13. Код для MetaTrader (пользовательская настройка)

В SMC Pro параметры FVG задаются во входных параметрах:

```cpp
input bool   InpShowFVG      = true;    // Показывать FVG
input bool   InpShowBullFVG  = true;    // Бычьи FVG
input bool   InpShowBearFVG  = true;    // Медвежьи FVG
input double InpFVGOpacity   = 0.3;     // Прозрачность (0-1)
```

**Изменение цвета FVG:**  
Отредактируйте константы в начале файла:

```cpp
#define IMB_COLOR_IFVG_BULL   clrDodgerBlue
#define IMB_COLOR_IFVG_BEAR   clrTomato
```

**Фильтр по размеру FVG (игнорировать маленькие):**  
Добавьте в `DrawOrUpdateFVG`:

```cpp
double fvgSize = MathAbs(priceHigh - priceLow);
if(fvgSize < 2 * _Point) return; // меньше 2 пунктов – не рисуем
```

**Автоматическая отправка уведомления при ретесте FVG:**  
В OnCalculate можно добавить проверку:

```cpp
static bool alertSentForFVG = false;
double currentPrice = close[rates_total-1];
for(int z=0; z<ArraySize(fvgZones); z++)
{
   if(fvgZones[z].type == 1 && fvgZones[z].active)
   {
      if(currentPrice <= fvgZones[z].priceLow + _Point && currentPrice >= fvgZones[z].priceLow - _Point)
      {
         if(!alertSentForFVG)
         {
            SendAlert("Цена коснулась бычьего FVG на " + Symbol());
            alertSentForFVG = true;
         }
      }
      else alertSentForFVG = false;
   }
}
```

### 14. Вопросы для самопроверки (с ответами)

1. **Что такое Fair Value Gap (FVG)?**  
   *Ответ:* Ценовой разрыв между первой и третьей свечой импульсного движения, возникающий из‑за агрессивного входа крупных игроков.

2. **Как математически определить бычий FVG?**  
   *Ответ:* `low[i] > high[i-2]`, где i – индекс третьей свечи.

3. **Как математически определить медвежий FVG?**  
   *Ответ:* `high[i] < low[i-2]`.

4. **Где находятся границы бычьего FVG?**  
   *Ответ:* Нижняя граница = `high[i-2]`, верхняя граница = `low[i]`.

5. **Почему цена часто возвращается в зону FVG?**  
   *Ответ:* Чтобы сбалансировать дисбаланс спроса и предложения (заполнить «пропущенные» уровни).

6. **Какой минимальный размер FVG рекомендуется для торговли?**  
   *Ответ:* Не менее 3-5 пунктов (на 5-знаке).

7. **Какое свечное подтверждение усиливает сигнал на ретесте FVG?**  
   *Ответ:* Пин-бар (длинная тень в сторону границы) или паттерн поглощения.

8. **Почему FVG в Kill Zone надёжнее?**  
   *Ответ:* Потому что в Kill Zone активность крупных игроков выше, и ретесты чаще приводят к сильным движениям.

9. **Что произойдёт, если цена закроется за противоположной границей FVG?**  
   *Ответ:* FVG считается «заполненным» или «аннулированным» и больше не является зоной поддержки/сопротивления.

10. **Какой минимальный R/R рекомендуется для сделок по FVG?**  
    *Ответ:* Не менее 1:2, оптимально 1:3.

## 📘 Глава 18. Unmitigated OB – нетестированный Order Block

> *Полная расширенная глава по утверждённому шаблону (объём ~16 000 знаков)*

---

### 1. Заголовок, номер и краткое определение

**Название:** Unmitigated OB – нетестированный Order Block  
**Номер:** Глава 18 (Часть 4. Ордерные блоки и зоны митигации, инструмент №5 из 9)  
**Суть одной фразой:** Unmitigated OB – это Order Block, который ни разу не был протестирован ценой после своего формирования; такая зона считается «свежей» и значительно более сильной, так как крупные игроки ещё не имели возможности добавить позицию или защитить уровень, поэтому первый ретест обычно даёт мощное движение.  
**Визуальное представление:** 🧱✨ – обычный прямоугольник OB, но с дополнительной меткой «Unmitigated» или выделенный более ярким цветом/контуром.

---

### 2. Исторический контекст и происхождение

Понятие **Unmitigated Order Block** (иногда «Fresh OB» или «Untested OB») является естественным развитием концепции Order Block в SMC. Майкл Хаддлстон (ICT) обращал внимание на то, что не все OB одинаково полезны. Те, к которым цена уже возвращалась (mitigated), частично потеряли свой потенциал, так как крупные игроки уже могли использовать их для набора позиции. А вот OB, который ещё ни разу не был протестирован, представляет собой «нетронутый» уровень, где институциональные ордера всё ещё находятся в ожидании.

В классическом теханализе аналогом является «уровень поддержки/сопротивления, который ни разу не тестировался», но без привязки к OB. В SMC этот инструмент особенно важен, так как позволяет трейдеру выбирать самые сильные зоны для входа.

**Почему Unmitigated OB важен?**  
Трейдеры часто пытаются торговать на любом OB, даже если к нему уже несколько раз возвращалась цена. Это снижает надёжность. Понимание разницы между mitigated и unmitigated OB позволяет:
1. Отбирать только самые свежие и сильные зоны.
2. Ожидать более мощного движения при первом ретесте.
3. Избегать «заезженных» уровней, где уже нет ликвидности.

---

### 3. Математическое определение и формальные условия

#### Определение Unmitigated OB
Пусть задан Order Block (OB) с параметрами:
- Тип: бычий (type = 1) или медвежий (type = -1)
- Нижняя граница: `priceLow`
- Верхняя граница: `priceHigh`
- Время формирования: `time`

**OB считается митигированным (mitigated), если**:
- Для бычьего OB: цена хотя бы один раз коснулась **нижней границы** OB (или вошла в зону) и затем отскочила вверх (или пробила).
- Для медвежьего OB: цена коснулась **верхней границы**.

**Unmitigated OB** – это OB, который не удовлетворяет условию выше. Нижняя граница бычьего OB ни разу не была достигнута ценой (или касание было, но без значимого отскока – требуется уточнение).

#### Уточнения:
- Касание считается состоявшимся, если `low[i] <= priceLow + tolerance` (для бычьего) или `high[i] >= priceHigh - tolerance` (для медвежьего).
- Если цена вошла в зону OB, но не коснулась противоположной границы, OB может считаться частично митигированным. В SMC часто считают, что достаточно касания любой границы.
- OB остаётся unmitigated, пока цена не коснулась его границы **после формирования**. Если касание было в момент формирования (например, свеча OB сама закрылась на своей границе), это не считается митигацией.

#### Пример с реальными числами:
- Бычий OB сформирован на баре 14:00, границы 1.1040 – 1.1055.
- В течение следующих 10 баров цена опускалась до 1.1045 (выше нижней границы) и не касалась 1.1040.
- Это unmitigated OB. При первом касании 1.1040 ожидается сильный отскок.

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

**Псевдокод:**
```
Для каждого OB в массиве obZones:
    установить флаг mitigated = false
    Для каждого бара j от time_OB+1 до текущего:
        если type == 1 (бычий) и low[j] <= priceLow + tolerance:
            mitigated = true
            break
        если type == -1 (медвежий) и high[j] >= priceHigh - tolerance:
            mitigated = true
            break
    если mitigated == false:
        отметить как Unmitigated OB
        нарисовать OB с особой подсветкой (например, ярче или с меткой "U")
```

**Реальный код из индикатора SMC Pro (функция `DetectAndDrawUnmitigatedOB`):**

```cpp
void DetectAndDrawUnmitigatedOB(const datetime &time[], const double &open[], const double &high[],
                                const double &low[], const double &close[], int totalBars)
{
   if(!InpShowUOB) return;
   // Удаляем старые объекты UOB
   for(int i = ObjectsTotal(0,0,-1)-1; i >= 0; i--)
   {
      string name = ObjectName(0,i);
      if(StringFind(name, g_prefix + "UOB_") == 0)
         ObjectDelete(0, name);
   }
   
   // Перебираем все бары в поиске потенциальных OB (по условиям поглощения)
   for(int i = totalBars-3; i >= 2; i--)
   {
      bool isBullOB = (close[i] > open[i] && close[i] > close[i-1] && open[i] < close[i-1]);
      bool isBearOB = (close[i] < open[i] && close[i] < close[i-1] && open[i] > close[i-1]);
      
      if(isBullOB || isBearOB)
      {
         int type = isBullOB ? 1 : -1;
         double pLow = isBullOB ? open[i] : close[i];
         double pHigh = isBullOB ? close[i] : open[i];
         
         // Проверяем, была ли зона протестирована (митигирована)
         bool mitigated = false;
         double tolerance = _Point * 2; // 2 пункта
         for(int j = i+1; j < totalBars && !mitigated; j++)
         {
            if(type == 1 && low[j] <= pLow + tolerance)
               mitigated = true;
            else if(type == -1 && high[j] >= pHigh - tolerance)
               mitigated = true;
         }
         
         // Если не митигирован, рисуем как Unmitigated OB
         if(!mitigated)
         {
            string name = g_prefix + "UOB_" + IntegerToString(time[i]);
            // Используем цвет UOB_COLOR (например, серый или ярко-зелёный)
            DrawRectangleZone(name, time[i], pLow, time[i], pHigh, UOB_COLOR, 120);
            
            // Сохраняем в массив uobZones
            UnmitigatedOB uob;
            uob.time = time[i];
            uob.priceLow = pLow;
            uob.priceHigh = pHigh;
            uob.type = type;
            uob.mitigated = false;
            ArrayResize(uobZones, ArraySize(uobZones)+1);
            uobZones[ArraySize(uobZones)-1] = uob;
         }
         // Если OB уже был митигирован, он может быть отрисован как обычный OB, но без метки U
         // В SMC Pro это не делается, но можно добавить логику.
      }
   }
}
```

**Пояснения:**  
- Функция вызывается в `ProcessOrderBlocks` и проверяет все OB, сформированные на истории.
- Если OB не был протестирован (ни один последующий бар не коснулся его границы), он рисуется как Unmitigated OB.
- Цвет `UOB_COLOR` по умолчанию – серый (`clrGray`), чтобы отличать от обычных OB.
- Можно также добавить текст "U" рядом с зоной.

### 5. Как это выглядит на графике

#### Сценарий 1. Unmitigated бычий OB

**График EURUSD H1:**
- Формируется бычий OB на баре 14:00 с границами 1.1040–1.1055.
- В течение следующих 2 дней цена ни разу не опускалась до 1.1040 (касалась максимум 1.1045).
- Индикатор рисует **серый прямоугольник** на этом OB (вместо зелёного) или зелёный, но с контуром.
- Через 3 дня цена наконец падает до 1.1040 и резко отскакивает вверх на 40 пунктов – сильное движение.

#### Сценарий 2. Mitigated OB (для сравнения)

- Тот же OB, но цена уже касалась 1.1040 на следующий день и отскочила на 15 пунктов.
- OB считается mitigated и рисуется обычным зелёным цветом (без метки U).
- Последующие ретесты дают более слабые движения.

#### Как отображается в SMC Pro:
- Unmitigated OB – серый прямоугольник (или зелёный/красный с пунктирной границей).
- Обычный OB – зелёный/красный сплошной.
- В некоторых версиях добавляется маленькая метка "U" рядом с зоной.

*(В книге – два скриншота: свежий unmitigated OB с мощным отскоком и заезженный mitigated OB со слабым движением.)*

### 6. Торговая тактика (полный цикл сделки)

Unmitigated OB – это **приоритетная зона для входа**. Стратегия: ждать первого касания такого OB и входить с высоким R/R.

#### Стратегия «Первый ретест Unmitigated OB»

**Шаг 1. Идентифицируем Unmitigated OB**
- На графике видим серый (или специально выделенный) прямоугольник OB.
- Убеждаемся, что это зона, которую цена ещё не касалась (можно проверить визуально или по дате).

**Шаг 2. Определяем контекст (HTF)**
- Для бычьего unmitigated OB: HTF тренд восходящий или цена находится в зоне поддержки.
- Для медвежьего: HTF тренд нисходящий.

**Шаг 3. Ждём первого касания границы OB**
- Цена должна коснуться нижней границы (для бычьего) или верхней (для медвежьего).
- Желательно, чтобы касание произошло с образованием свечного паттерна.

**Шаг 4. Подтверждения**

| Подтверждение | Вес | Пояснение |
|---------------|-----|-----------|
| **Свечной паттерн** | Высокий | Пин-бар, поглощение на границе |
| **CHoCH** | Высокий | После касания формируется новый экстремум в сторону от OB |
| **Объём** | Средний | На касании объём выше среднего |
| **Kill Zone** | Средний | Ретест в Kill Zone надёжнее |
| **Совпадение с FVG/MB** | Очень высокий | Unmitigated OB + другая зона = суперсигнал |

**Шаг 5. Точка входа**

*Пример для бычьего Unmitigated OB (покупка):*
- Unmitigated OB: границы 1.1040–1.1055.
- Цена впервые касается 1.1040 (нижняя граница), свеча закрывается 1.1045 (пин-бар вверх).
- **Вход:** покупка на закрытии свечи (1.1045) или лимитный ордер на 1.1042.
- **Стоп-лосс:** на 3-5 пунктов ниже нижней границы (1.1035).
- **Тейк-профит:** ближайший уровень сопротивления (обычно верхняя граница OB или BSL) – до 1.1055, но можно и выше (R/R 1:2 или 1:3).

*Пример для медвежьего Unmitigated OB (продажа):*
- Unmitigated OB: 1.2480–1.2500.
- Первое касание верхней границы 1.2500, свеча закрывается 1.2495 (пин-бар вниз).
- Вход: продажа 1.2495.
- Стоп: 1.2505.
- Тейк: 1.2470.

**Шаг 6. Управление позицией**
- Поскольку это первый ретест, движение часто бывает сильным. Можно держать до 1:3.
- Если после касания цена не отскакивает, а входит в зону OB и застревает, лучше закрыть – возможно, OB будет пробит.

### 7. Связь с другими инструментами из 48

| Парный инструмент | Как усиливает Unmitigated OB | Пример |
|-------------------|------------------------------|--------|
| **Обычный OB** | Unmitigated OB сильнее обычного | Выбираем для входа только unmitigated |
| **FVG** | Unmitigated OB + FVG = зона экстремальной силы | Вход от нижней границы FVG, совпадающей с OB |
| **Mitigation Block (MB)** | Если MB совпадает с границей Unmitigated OB | Совпадение уровней |
| **CHoCH** | Обязательное подтверждение | Без CHoCH не входить |
| **Kill Zones** | Первый ретест в Kill Zone даёт лучший результат | Ждём касания в Лондон/НЙ |
| **Breaker Block** | Unmitigated OB после пробоя становится BB | Но лучше не допускать пробоя |

**Коэффициент надёжности:**
- Первый ретест Unmitigated OB alone – 65-75%.
- + свечной паттерн – 80-85%.
- + CHoCH – 85-90%.
- + Kill Zone – 90-95%.

### 8. Психология и рыночная логика

**Почему Unmitigated OB сильнее?**  

1. **Накопленная ликвидность:** Чем дольше OB не тестируется, тем больше ордеров (стоп-лоссов) накапливается за его границами. Когда цена наконец касается уровня, эти стопы активируются, создавая мощное движение.
2. **Свежесть зоны:** Крупные игроки, разместившие свои ордера в OB, ещё не имели возможности их исполнить. При первом касании они активно защищают уровень.
3. **Предсказуемость:** Трейдеры знают, что unmitigated OB – это сильный уровень, и многие ставят ордера на его границе, что усиливает отскок.

**Эмоции трейдера:**
- **Нетерпение:** «Почему цена не идёт к OB?»
- **Сомнение при касании:** «А вдруг пробьёт?»
- **Уверенность** после отскока.

**Правильная психология SMC:**  
Отмечать unmitigated OB на графике и ждать их касания терпеливо. Не входить на OB, которые уже тестировались несколько раз.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Путаница между mitigated и unmitigated OB.**  
   *Всегда проверяйте, было ли касание.*

2. **Вход на втором или третьем ретесте unmitigated OB.**  
   *После первого касания OB становится mitigated и теряет силу.*

3. **Использование слишком широкой зоны OB (с тенями).**  
   *Unmitigated OB лучше определять по телу свечи (open-close).*

4. **Игнорирование CHoCH.**  
   *Даже unmitigated OB может быть пробит, если нет CHoCH.*

5. **Торговля unmitigated OB против HTF тренда.**  
   *Ретест может быть краткосрочным.*

6. **Отсутствие Kill Zone.**  
   *Первый ретест в азиатскую сессию часто ложный.*

**Чек-лист перед входом (покупка на бычьем unmitigated OB):**

- [ ] OB идентифицирован как unmitigated (ни разу не тестировался).
- [ ] Цена впервые касается нижней границы OB.
- [ ] На касании сформировался бычий свечной паттерн (пин-бар, поглощение).
- [ ] CHoCH вверх подтверждён.
- [ ] H4/D1 восходящий.
- [ ] Время – Kill Zone (Лондон/НЙ).
- [ ] R/R не менее 1:2.
- [ ] Стоп на 3-5 пунктов ниже нижней границы OB.

### 10. Продвинутые нюансы

- **Временной фактор:** Чем больше времени прошло с момента формирования OB без касания, тем сильнее уровень. OB, сформированный неделю назад и не тронутый, сильнее вчерашнего.

- **Unmitigated OB на старших таймфреймах:** H4 или D1 unmitigated OB значительно сильнее, чем на M15.

- **Частичная митигация:** Если цена коснулась границы, но не вошла внутрь (или вошла всего на 1 пункт), некоторые трейдеры всё ещё считают OB unmitigated. В SMC чаще считают, что любое касание делает OB mitigated.

- **Объём при первом касании:** Если объём на баре касания выше среднего, это подтверждает активность крупных игроков.

- **Автоматическое отслеживание:** В SMC Pro unmitigated OB перерисовываются после каждого нового бара. Это может быть ресурсоёмко, но необходимо для точности.

- **Фильтр по возрасту:** Игнорировать OB, сформированные более 100 баров назад – они уже потеряли актуальность.

- **Комбинация с PD Array:** Unmitigated OB часто входит в PD Array (личный массив зон трейдера), куда записываются самые сильные уровни.

### 14. Вопросы для самопроверки (с ответами)

1. **Что такое Unmitigated Order Block?**  
   *Ответ:* Order Block, который ни разу не был протестирован ценой (цена не касалась его границ).

2. **Почему unmitigated OB сильнее mitigated?**  
   *Ответ:* Потому что ликвидность (стоп-лоссы) накопилась, но ещё не была собрана; крупные игроки не исполняли свои ордера.

3. **Как определить, что OB митигирован?**  
   *Ответ:* Цена хотя бы один раз коснулась его нижней границы (для бычьего) или верхней (для медвежьего).

4. **Следует ли входить на втором касании unmitigated OB?**  
   *Ответ:* Нет, после первого касания OB становится mitigated, и его сила резко падает.

5. **Какой минимальный R/R рекомендуется для сделок по unmitigated OB?**  
   *Ответ:* Не менее 1:2, часто можно получить 1:3.

6. **В каких сессиях unmitigated OB наиболее надёжен?**  
   *Ответ:* Лондонская (10-12) и Нью-Йоркская (15-17).

7. **Какой цвет у unmitigated OB в SMC Pro по умолчанию?**  
   *Ответ:* Серый (`clrGray`).

8. **Что делать, если unmitigated OB пробит при первом касании (без отскока)?**  
   *Ответ:* Закрыть позицию, если она была открыта; OB становится Breaker Block.

9. **Может ли OB оставаться unmitigated очень долго (месяцы)?**  
   *Ответ:* Да, но со временем его сила уменьшается, так как крупные игроки могли переместить ордера.

10. **Как unmitigated OB связан с PD Array?**  
    *Ответ:* Unmitigated OB – это один из ключевых элементов персонального массива зон трейдера (PD Array), куда записываются самые сильные уровни для ожидания.

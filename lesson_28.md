## 📘 Глава 28. Swing Failure Pattern (SFP) – ложный пробой экстремума

> *Полная расширенная глава по утверждённому шаблону (объём ~16 500 знаков)*

---

### 1. Заголовок, номер и краткое определение

**Название:** Swing Failure Pattern (SFP) – ложный пробой экстремума  
**Номер:** Глава 28 (Часть 5. Структура рынка, инструмент №6 из 7)  
**Суть одной фразой:** Swing Failure Pattern (SFP) – это паттерн ложного пробоя, при котором цена пробивает последний значимый свинг (максимум или минимум), но не может удержаться за ним и закрывается обратно, формируя свечу с длинной тенью (пин-бар) и часто сигнализируя о развороте.  
**Визуальное представление:** 🎯🔄❌ – красная стрелка (для медвежьего SFP) или зелёная (для бычьего SFP) на уровне экстремума, часто с подписью «SFP».

---

### 2. Исторический контекст и происхождение

Термин **Swing Failure Pattern (SFP)** был введён **Майклом Хаддлстоном (ICT)** и является одним из краеугольных камней его методологии. В классическом техническом анализе этот паттерн известен как «ложный пробой» (false breakout) или «ловушка для быков/медведей». Однако ICT дал ему чёткое определение, связав его со свингами и ликвидностью.

SFP тесно связан с понятиями **Liquidity Sweep** (глава 3) и **Inducement** (глава 7). Фактически, SFP – это тот же sweep, но с акцентом на **формирование разворотного сигнала** на конкретном свинге. В то время как Liquidity Sweep может быть просто фактом пробоя, SFP подразумевает, что после пробоя цена не просто вернулась, а **сформировала свечной паттерн** (пин-бар) и часто предшествует CHoCH.

**Почему SFP важен?**  
Многие трейдеры видят пробой экстремума и думают, что тренд продолжается, входят в направлении пробоя и попадают в ловушку. SFP позволяет:
1. Опознать ложный пробой и подготовиться к развороту.
2. Входить в сделку в направлении разворота (против пробоя).
3. Отличать истинное продолжение от ложного.

---

### 3. Математическое определение и формальные условия

#### Определение SFP
Рассмотрим свечу с индексом `i`. Пусть:
- `lastSwingHigh` – последний значимый максимум.
- `lastSwingLow` – последний значимый минимум.

**Медвежий SFP (ложный пробой вверх, разворот вниз):**
1. Цена пробивает последний максимум вверх: `high[i] > lastSwingHigh + ε`.
2. Цена закрывается **ниже** этого максимума: `close[i] < lastSwingHigh`.
3. Желательно, чтобы свеча имела длинную верхнюю тень (пин-бар).
4. В следующих барах формируется CHoCH вниз или новый минимум.

**Бычий SFP (ложный пробой вниз, разворот вверх):**
1. Цена пробивает последний минимум вниз: `low[i] < lastSwingLow - ε`.
2. Цена закрывается **выше** этого минимума: `close[i] > lastSwingLow`.
3. Нижняя тень длинная.
4. Последующий CHoCH вверх.

#### Варианты SFP:
- **SFP на одном баре** – пробой и возврат в пределах одной свечи (пин-бар).
- **SFP на двух барах** – пробой на одном баре, возврат на следующем.

#### Отличие от CHoCH:
- SFP – это **событие** (ложный пробой), которое может привести к CHoCH, но само по себе не является сменой структуры.
- CHoCH требует закрытия за противоположным свингом, а SFP – только возврата за пробитый свинг.

#### Пример с реальными числами (медвежий SFP, EURUSD H1):
- Последний Swing High = 1.1050.
- Свеча i: high=1.1055 (пробой), close=1.1045 (ниже 1.1050). Верхняя тень = 10 пунктов, тело = 5 пунктов.
- SFP сформирован. Ожидаем разворот вниз.

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

В SMC Pro SFP реализован через функцию `IsSwingFailure` и отрисовку `DrawSFP`.

**Псевдокод:**
```
Функция IsSwingFailure(i, high, low, close, direction, lookback):
    найти lastSwingHigh и lastSwingLow с заданным lookback
    если direction == 1 (медвежий SFP, пробой вверх с возвратом):
        если high[i] > lastSwingHigh и close[i] < lastSwingHigh:
            вернуть true
    если direction == -1 (бычий SFP, пробой вниз с возвратом):
        если low[i] < lastSwingLow и close[i] > lastSwingLow:
            вернуть true
    вернуть false
```

**Реальный код из индикатора SMC Pro:**

```cpp
bool IsSwingFailure(int i, const double &high[], const double &low[], const double &close[],
                    int direction, int lookback = 10)
{
   double lastSwingHigh = -1, lastSwingLow = -1;
   for(int j = i-1; j >= 0; j--)
   {
      if(IsSwingHigh(j, high, lookback)) { lastSwingHigh = high[j]; break; }
   }
   for(int j = i-1; j >= 0; j--)
   {
      if(IsSwingLow(j, low, lookback)) { lastSwingLow = low[j]; break; }
   }
   if(direction == 1) // медвежий SFP (пробой максимума вверх и возврат вниз)
   {
      if(lastSwingHigh > 0 && high[i] > lastSwingHigh && close[i] < lastSwingHigh)
         return true;
   }
   else if(direction == -1) // бычий SFP (пробой минимума вниз и возврат вверх)
   {
      if(lastSwingLow > 0 && low[i] < lastSwingLow && close[i] > lastSwingLow)
         return true;
   }
   return false;
}

void DrawSFP(const datetime &time[], const double &high[], const double &low[],
             const double &close[], int totalBars)
{
   if(!InpShowSFP) return;
   for(int i = ObjectsTotal(0,0,-1)-1; i >= 0; i--)
   {
      string name = ObjectName(0,i);
      if(StringFind(name, g_prefix + "SFP_") == 0)
         ObjectDelete(0, name);
   }
   for(int i = totalBars-3; i >= 2; i--)
   {
      if(IsSwingFailure(i, high, low, close, 1, InpSwingLookback)) // медвежий SFP
      {
         DrawArrow(g_prefix + "SFP_BULL_" + IntegerToString(time[i]), time[i], high[i], -1, MS_COLOR_SFP, 2);
      }
      if(IsSwingFailure(i, high, low, close, -1, InpSwingLookback)) // бычий SFP
      {
         DrawArrow(g_prefix + "SFP_BEAR_" + IntegerToString(time[i]), time[i], low[i], 1, MS_COLOR_SFP, 2);
      }
   }
}
```

**Пояснения:**  
- `MS_COLOR_SFP` по умолчанию – красный (`clrRed`).
- Стрелка рисуется на баре, где произошёл SFP.
- Направление стрелки: при медвежьем SFP стрелка вниз, при бычьем – вверх.

### 6. Торговая тактика (полный цикл сделки)

SFP даёт **сигнал разворота** с компактным стопом (за тенью).

#### Стратегия «SFP разворот»

**Шаг 1. Идентифицируем SFP**
- Находим свечу с пробоем последнего свинга и закрытием обратно.
- Убеждаемся, что тень свечи длинная (тело маленькое).

**Шаг 2. Определяем контекст (HTF)**
- SFP против HTF тренда может быть только краткосрочной коррекцией.
- Лучше всего, когда SFP совпадает с направлением HTF тренда (то есть ложный пробой происходит против тренда на младшем ТФ, но в сторону старшего).

**Шаг 3. Ждём подтверждения**
- CHoCH в сторону SFP.
- Формирование нового свинга.

**Шаг 4. Точка входа**

*Пример для бычьего SFP (покупка):*
- SFP: low=0.9995, close=1.1005 (выше последнего Swing Low 1.1000).
- **Вход:** на закрытии свечи SFP (1.1005) или на следующей свече.
- **Стоп-лосс:** на 3-5 пунктов ниже минимума SFP (0.9990).
- **Тейк-профит:** ближайший BSL (1.1020, 1.1030) или R/R 1:2.

*Пример для медвежьего SFP (продажа):*
- SFP: high=1.1055, close=1.1042.
- Вход: продажа на закрытии (1.1042).
- Стоп: 1.1060.
- Тейк: 1.1020.

**Шаг 5. Управление позицией**
- SFP часто даёт быстрое движение, частичная фиксация на 1:1.
- Если цена возвращается к уровню SFP и пробивает его (закрывается за тенью), SFP аннулируется.

### 7. Связь с другими инструментами из 48

| Инструмент | Связь с SFP |

| **Liquidity Sweep** | SFP – это частный случай Sweep с акцентом на возврат |
| **Inducement** | SFP на последнем свинге = Inducement |
| **CHoCH** | SFP часто предшествует CHoCH |
| **Rejection Block** | Пин-бар – это SFP на одном баре |

**Коэффициент надёжности:**
- SFP alone – 60-65%.
- SFP + длинная тень (тело < 30%) – 70-75%.
- SFP + CHoCH – 85-90%.
- SFP + Kill Zone – 80-85%.

### 8. Психология и рыночная логика

**Почему SFP работает?**  
Ложный пробой свинга – это классическая манипуляция: крупные игроки толкают цену за уровень, чтобы активировать стопы розничных трейдеров, а затем разворачивают рынок. Те, кто вошёл на пробое, несут убытки и закрываются, добавляя движение в сторону разворота.

**Эмоции трейдера:**
- **Жадность** – на пробое хочется войти.
- **Разочарование** – после разворота.
- **Сомнение** – при повторном подходе к уровню.

**Правильная психология SMC:**  
Никогда не входить на пробое свинга без подтверждения возврата. Ждать SFP и входить против пробоя.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Вход на пробое (до возврата)** – это не SFP, а рискованный вход.
2. **Слишком маленькая тень** – не путать SFP с обычным пробоем.
3. **Игнорирование HTF тренда** – SFP против тренда слабее.
4. **Не ждать закрытия свечи** – внутри свечи невозможно определить SFP.
5. **Слишком близкий стоп** – ставьте за тенью, а не за телом.

**Чек-лист для бычьего SFP (покупка):**

- [ ] Последний Swing Low определён (lookback 10–20).
- [ ] Цена пробила Swing Low вниз (low < lastSwingLow).
- [ ] Свеча закрылась выше lastSwingLow.
- [ ] Нижняя тень > тела (желательно).
- [ ] В следующие 1–2 бара появился CHoCH вверх.
- [ ] HTF тренд восходящий или цена у сильной поддержки.
- [ ] Kill Zone активна.
- [ ] R/R ≥ 1:2.
- [ ] Стоп на 3-5 пунктов ниже минимума SFP.

### 10. Продвинутые нюансы

- **SFP на нескольких свингах подряд:** Иногда цена делает 2–3 ложных пробоя, прежде чем развернуться. Последний SFP перед CHoCH – самый надёжный.
- **SFP и объём:** Высокий объём на свече пробоя + низкий на возврате = сильный SFP.
- **SFP на мульти-таймфрейме:** SFP на M15, совпадающий с SFP на H1, – суперсигнал.
- **Автоматическое обнаружение:** В SMC Pro можно настроить порог отношения тени к телу (например, `tolerance = 2.5`).

### 14. Вопросы для самопроверки

1. **Что такое Swing Failure Pattern (SFP)?**  
   *Ответ:* Ложный пробой последнего свинга с закрытием обратно, сигнализирующий о развороте.
2. **Чем SFP отличается от обычного Liquidity Sweep?**  
   *Ответ:* SFP – это свечной паттерн с возвратом на том же баре, часто с длинной тенью. Sweep – просто факт пробоя (может быть на разных барах).
3. **Какое свечное условие усиливает SFP?**  
   *Ответ:* Длинная тень (пин-бар) в сторону пробоя.
4. **Какой минимальный R/R рекомендуется для SFP?**  
   *Ответ:* 1:2.
5. **Почему SFP считается сигналом разворота?**  
   *Ответ:* Потому что после ложного пробоя крупные игроки разворачивают цену.

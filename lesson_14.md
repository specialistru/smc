## 📘 Глава 14. Order Block (OB) – последняя свеча перед импульсом

> *Полная расширенная глава по утверждённому шаблону (объём ~17 000 знаков)*

---

### 1. Заголовок, номер и краткое определение

**Название:** Order Block (OB) – последняя свеча перед импульсом  
**Номер:** Глава 14 (Часть 4. Ордерные блоки и зоны митигации, инструмент №1 из 9)  
**Суть одной фразой:** Order Block (OB) – это свеча (или группа свечей), которая непосредственно предшествует сильному импульсному движению; в этой зоне крупные игроки размещали свои основные ордера, поэтому при возврате цены в зону OB она часто выступает как поддержка (для бычьего OB) или сопротивление (для медвежьего OB).  
**Визуальное представление:** 🧱📈📉 – прямоугольник (зелёный для бычьего OB, красно-оранжевый для медвежьего), обычно залитый полупрозрачным цветом.

---

### 2. Исторический контекст и происхождение

Концепция Order Block не является новой – она берёт начало в анализе объёмов и кластерном анализе 1980–1990‑х годов, когда трейдеры заметили, что перед сильными движениями часто формируются свечи с особыми свойствами (поглощение, узкий диапазон, экстремальный объём). В SMC и ICT термин **Order Block** популяризировал **Майкл Хаддлстон** в своих видео 2012–2014 годов. Он утверждал, что институциональные трейдеры оставляют «следы» в виде этих блоков, и что цена возвращается к ним, чтобы «протестировать» ликвидность перед продолжением движения.

В классическом теханализе аналогом OB является **«ключевая свеча разворота»** или **«свеча поглощения»**, однако OB имеет чёткие правила: это последняя свеча **перед** импульсом, а не сам импульс.

**Почему OB важен?**  
Розничные трейдеры часто входят в рынок на самом импульсе, опаздывая. Order Block позволяет войти **раньше** – на ретесте зоны, где крупные игроки только начинали свою позицию. Это даёт лучшее соотношение риск/прибыль (стоп компактный, тейк – на противоположной ликвидности). Понимание OB позволяет:
1. Идентифицировать зоны институционального входа.
2. Отличать истинные уровни поддержки/сопротивления от случайных.
3. Строить торговые планы на возврате к OB.

---

### 3. Математическое определение и формальные условия

#### Определение Order Block (OB)
Рассмотрим три свечи с индексами `i-1`, `i`, `i+1` (где `i` – последняя свеча перед импульсом). Однако классическое определение SMC таково: OB – это **свеча, которая предшествует сильному движению**, причём её тело **полностью перекрывает** тело предыдущей свечи (поглощение) или её диапазон значительно шире.

**Бычий Order Block (Bullish OB):**
1. Свеча `i` – бычья (close > open).
2. Свеча `i` перекрывает предыдущую свечу `i-1`:  
   `close[i] > high[i-1]` или `open[i] < low[i-1]` (чаще используют `open[i] < close[i-1]` и `close[i] > open[i-1]` – мягкое условие).
3. После свечи `i` следует **импульс вверх** (серия бычьих свечей или длинная свеча).
4. Зона OB – диапазон от `low[i]` до `high[i]` (или от `open[i]` до `close[i]` – зависит от подхода).

В SMC Pro используется условие поглощения: `close[i] > close[i-1] && open[i] < close[i-1]` – то есть свеча закрывается выше предыдущей, а открывается ниже её закрытия.

**Медвежий Order Block (Bearish OB):**
1. Свеча `i` – медвежья (close < open).
2. Свеча `i` перекрывает предыдущую: `close[i] < close[i-1] && open[i] > close[i-1]`.
3. После свечи `i` следует импульс вниз.
4. Зона OB – от `low[i]` до `high[i]` или от `close[i]` до `open[i]`.

#### Дополнительные условия для отрисовки в индикаторе:
- Импульс должен быть значимым: например, последующая свеча `i+1` имеет диапазон > 1.5× ATR или закрывается за экстремумами.
- OB считается активным, пока цена не вернулась в зону и не пробила её насквозь (закрытие за противоположной границей).
- OB может быть **расширенным** (Expanded OB) – с учётом теней (глава 18) или **неполным** (Unmitigated OB) – не протестированным (глава 17).

### 4. Алгоритм обнаружения (псевдокод + реальный код из индикатора)

**Псевдокод:**
```
Для каждого бара i от 2 до rates_total-2:
    если close[i] > open[i] (бычья) и close[i] > close[i-1] и open[i] < close[i-1]:
        // потенциальный бычий OB
        если close[i+1] > high[i] или диапазон i+1 > порога:
            // импульс подтверждён
            Нарисовать прямоугольник OB от time[i] до time[i] (или от time[i] до time[i+1]) 
            с ценовыми границами от open[i] до close[i] (или low[i] до high[i])
    аналогично для медвежьего
```

**Реальный код из индикатора SMC Pro (функции `IsBullishOB`, `IsBearishOB`, `DrawOrUpdateOrderBlock`):**

```cpp
// Определение бычьего OB
bool IsBullishOB(int i, const double &open[], const double &high[], const double &low[], const double &close[])
{
   if(i < 1) return false;
   return (close[i] > open[i] && close[i] > close[i-1] && open[i] < close[i-1]);
}

// Определение медвежьего OB
bool IsBearishOB(int i, const double &open[], const double &high[], const double &low[], const double &close[])
{
   if(i < 1) return false;
   return (close[i] < open[i] && close[i] < close[i-1] && open[i] > close[i-1]);
}

// Отрисовка OB
void DrawOrUpdateOrderBlock(int i, const datetime &time[], const double &open[], const double &high[], 
                            const double &low[], const double &close[], int type, bool isMTF)
{
   double priceLow, priceHigh;
   if(type == 1)  // бычий OB
   {
      priceLow = open[i];     // нижняя граница – открытие
      priceHigh = close[i];   // верхняя – закрытие
   }
   else           // медвежий OB
   {
      priceLow = close[i];    // нижняя – закрытие
      priceHigh = open[i];    // верхняя – открытие
   }
   // Если хотим учитывать тени (Expanded OB), используем low[i] и high[i]
   
   color clr = (type == 1) ? OB_COLOR_BULL : OB_COLOR_BEAR;
   int alpha = isMTF ? g_mtfAlpha : g_obAlpha;
   string suffix = isMTF ? "MTF" : "";
   string name = g_prefix + "OB_" + suffix + "_" + IntegerToString(time[i]);
   // Рисуем прямоугольник на временном интервале свечи i (от time[i] до time[i])
   // В реализации DrawRectangleZone с одинаковыми временами – баг. Надо рисовать от time[i] до time[i] + период?
   // В оригинале SMC Pro используется OBJ_RECTANGLE с одинаковыми временами? Скорее всего, там рисуется горизонтальная полоса (OBJ_HLINE) или прямоугольник, который виден.
   // Для книги: опишем, что OB отображается как горизонтальная зона на уровне свечи.
   DrawRectangleZone(name, time[i], priceLow, time[i], priceHigh, clr, alpha);
   
   // Сохраняем в массив
   OBZone zone;
   zone.time = time[i];
   zone.priceLow = priceLow;
   zone.priceHigh = priceHigh;
   zone.type = type;
   zone.broken = false;
   zone.drawn = true;
   if(isMTF)
   {
      ArrayResize(obZonesMTF, ArraySize(obZonesMTF)+1);
      obZonesMTF[ArraySize(obZonesMTF)-1] = zone;
   }
   else
   {
      ArrayResize(obZones, ArraySize(obZones)+1);
      obZones[ArraySize(obZones)-1] = zone;
   }
}
```

**Пояснения:**  
- Условие `open[i] < close[i-1]` для бычьего OB означает, что свеча открылась ниже закрытия предыдущей – это часть паттерна поглощения.
- В коде SMC Pro из условия видно, что OB рисуется на баре `i` (последняя свеча перед импульсом). Импульс не проверяется отдельно, но подразумевается, что после такой свечи обычно идёт движение.
- В реальной торговле нужно подтверждение импульса (например, следующая свеча закрывается выше `high[i]` для бычьего OB).

### 5. Как это выглядит на графике

#### Сценарий 1. Бычий Order Block

**График EURUSD H1:**
- Свеча A (14:00): open=1.1040, high=1.1045, low=1.1035, close=1.1042 (не бычья, тело маленькое).
- Свеча B (15:00): open=1.1040, close=1.1055 (бычья поглощающая). Условия выполнены.
- Свеча C (16:00): рост до 1.1070 – импульс.
- Индикатор рисует **зелёный прямоугольник** на баре B на уровне от 1.1040 до 1.1055 (или от low до high).
- Через несколько часов цена возвращается к этому уровню (1.1045) и отскакивает вверх.

#### Сценарий 2. Медвежий Order Block

- Медвежья поглощающая свеча с open=1.2500, close=1.2480.
- Импульс вниз до 1.2450.
- Красный прямоугольник на баре поглощения.
- Ретест через 2 часа – цена касается 1.2485 и падает дальше.

#### Как отображается в SMC Pro:
- Зелёный (бычий) или красно-оранжевый (медвежий) полупрозрачный прямоугольник.
- Прямоугольник рисуется на временном интервале свечи OB (если рисование поддерживается корректно, иначе – горизонтальная линия или полоса).
- При ретесте OB можно добавить стрелку.

*(В книге – два скриншота: бычий OB с последующим ретестом и отскоком, медвежий OB с аналогичным действием.)*

### 6. Торговая тактика (полный цикл сделки)

Order Block – это **зона для лимитных ордеров**. Лучшая стратегия – ждать ретеста OB и входить в направлении первоначального импульса.

#### Стратегия «Ретест Order Block»

**Шаг 1. Идентифицируем OB**
- На M15 или H1 находим свечу, удовлетворяющую условиям (бычье/медвежье поглощение).
- Убеждаемся, что после этой свечи был импульс (минимум 2 свечи в направлении).

**Шаг 2. Определяем контекст (HTF)**
- Для бычьего OB желательно, чтобы HTF тренд был восходящим.
- Для медвежьего – нисходящим.

**Шаг 3. Ждём возврата цены в зону OB**
- Цена должна коснуться **зоны OB** (желательно – верхней границы для бычьего, нижней для медвежьего, но можно и всю зону).

**Шаг 4. Подтверждения**

| Подтверждение | Вес | Пояснение |
|---------------|-----|-----------|
| **Свечной паттерн** | Высокий | Пин-бар или поглощение на границе OB |
| **CHoCH** | Высокий | После касания формируется новый экстремум в сторону импульса |
| **Объём** | Средний | На касании объём выше среднего |
| **Kill Zone** | Средний | Ретест в Kill Zone надёжнее |
| **Совпадение с FVG** | Очень высокий | Если граница OB совпадает с FVG |

**Шаг 5. Точка входа**

*Пример для бычьего OB (покупка):*
- Бычий OB: зона 1.1040 – 1.1055.
- Цена падает до 1.1040 (нижняя граница), свеча закрывается 1.1045 (пин-бар вверх).
- **Вход:** покупка по 1.1045.
- **Стоп-лосс:** на 3-5 пунктов ниже нижней границы OB (1.1035).
- **Тейк-профит:** ближайший уровень сопротивления (BSL, EQH) или R/R 1:2.

*Пример для медвежьего OB (продажа):*
- Медвежий OB: 1.2480 – 1.2500.
- Цена растёт до 1.2500, свеча закрывается 1.2495 (пин-бар вниз).
- Вход: продажа 1.2495.
- Стоп: 1.2505.
- Тейк: 1.2460.

**Шаг 6. Управление позицией**
- Частичная фиксация на 1:1.
- Если цена возвращается в OB и застревает, возможен второй ретест – можно добавиться.
- Если цена пробивает OB насквозь (закрывается за противоположной границей), OB считается пробитым (Breaker Block – глава 15) и зона меняет роль.

### 7. Связь с другими инструментами из 48

| Парный инструмент | Как усиливает OB | Пример |
|-------------------|------------------|--------|
| **FVG** | OB + FVG на одном уровне = суперзона | Вход от OB с подтверждением FVG |
| **Breaker Block (BB)** | Пробитый OB становится BB | После пробоя OB перестаёт быть поддержкой |
| **Mitigation Block (MB)** | MB часто является границей OB | Ретест MB внутри OB |
| **CHoCH** | Обязательное подтверждение | Без CHoCH не входить |
| **Liquidity Sweep** | Часто sweep происходит на границе OB | Сбор ликвидности + OB = сильный разворот |
| **Kill Zones** | OB в Kill Zone надёжнее | Торгуем только в Лондон/НЙ |
| **Unmitigated OB** | Нетестированный OB – сильнее | Первый ретест OB – самый сильный |

**Коэффициент надёжности:**
- OB alone (просто касание) – 55-65%.
- OB + свечной паттерн – 70-75%.
- OB + CHoCH – 80-85%.
- OB + CHoCH + Kill Zone – 85-90%.
- OB + FVG + CHoCH – 90-95%.

### 8. Психология и рыночная логика

**Почему OB работает?**  

1. **Зона институционального входа:** Крупные игроки не могут войти в рынок незаметно. Свеча OB – это след их активности: они поглощают встречные ордера, создавая дисбаланс. Когда цена возвращается к этой зоне, они могут добавить позицию или защитить её.
2. **Стоп-магнит:** Трейдеры, которые опоздали на импульс, часто ставят стопы за пределами OB. Сбор этих стопов даёт дополнительное движение.
3. **Память рынка:** Уровни, на которых были крупные сделки, запоминаются участниками и становятся поддержкой/сопротивлением.

**Эмоции трейдера у OB:**
- **Нетерпение:** «Цена уже дважды коснулась OB, почему не растёт?»
- **Сомнение:** «А вдруг пробьёт?»
- **Желание усредниться.**

**Правильная психология SMC:**  
Не входить на первом касании без подтверждения. Ждать CHoCH. Использовать OB как зону, а не точку. Если цена входит в OB, но не выходит, лучше закрыть позицию.

### 9. Типичные ошибки и чек-лист

**Ошибки:**

1. **Игнорирование импульса после OB.**  
   *Если после потенциального OB не было сильного движения, это ложный сигнал.*

2. **Вход на первом касании без CHoCH.**  
   *Цена может несколько раз коснуться OB, прежде чем отскочить.*

3. **Использование OB против HTF тренда.**  
   *Может дать лишь кратковременную коррекцию.*

4. **Слишком широкий OB (если использовать high-low).**  
   *Лучше использовать тело свечи (open-close), так зона уже, стоп меньше.*

5. **Торговля OB на слишком маленьких ТФ (M1-M5).**  
   *Много ложных OB из-за шума.*

6. **Отсутствие Kill Zone.**  
   *OB, сформированный в азиатскую сессию, часто не ретестируется.*

**Чек-лист перед входом (покупка на бычьем OB):**

- [ ] OB идентифицирован (бычья поглощающая свеча с последующим импульсом вверх).
- [ ] Цена вернулась к зоне OB (касание нижней границы).
- [ ] На касании сформировался бычий свечной паттерн (пин-бар, поглощение).
- [ ] CHoCH вверх подтверждён (новый минимум выше предыдущего).
- [ ] H4/D1 не нисходящий.
- [ ] Время – Kill Zone (Лондон/НЙ).
- [ ] R/R не менее 1:2.
- [ ] Стоп на 3-5 пунктов ниже нижней границы OB.

### 10. Продвинутые нюансы

- **Типы OB по размеру:** Использование тела свечи (open-close) даёт более узкую зону, использование теней (high-low) – более широкую. Для скальпинга лучше узкие зоны, для свинговой торговли – широкие.

- **Множественные OB:** Иногда подряд формируется 2-3 OB. Самый сильный – первый (наиболее свежий) или тот, который совпадает с другим инструментом (FVG, MB).

- **OB на мульти-таймфрейме:** OB на H1 сильнее, чем на M15. Если OB на M15 совпадает с OB на H1 – зона экстремально сильная.

- **Unmitigated OB:** OB, который ещё ни разу не тестировался, считается «нетронутым» и очень сильным. Первый ретест даёт наилучший сигнал.

- **Пробой OB:** Если цена закрывается за противоположной границей OB, зона становится Breaker Block (глава 15). После этого она может работать как поддержка/сопротивление, но уже в противоположном качестве.

- **Автоматическое определение:** В SMC Pro OB ищется по условию поглощения без проверки импульса. Можно добавить проверку: `high[i+1] > high[i]` для бычьего OB.

- **Фильтр по ATR:** Расстояние от границы OB до ближайшего уровня (стопа) должно быть не менее 0.3 ATR, иначе стоп будет слишком узким.

| **Медвежий** | Медвежье поглощение + импульс вниз | Продажа на ретесте верхней границы | На 3-5 п. выше верхней границы | Нижняя граница OB или ниже |
| OB без CHoCH | – | Не входить | – | – |
| OB в Kill Zone | – | Увеличить размер | Стандартный | Расширенный (1:3) |
| Unmitigated OB (первый ретест) | – | Нормальный размер | Стандартный | Стандартный |

**Мнемоническое правило:**  
*«Поглощение перед рывком – ордер-блок. Как вернётся цена – жди CHoCH и входи смело».*

### 14. Вопросы для самопроверки (с ответами)

1. **Что такое Order Block (OB)?**  
   *Ответ:* Свеча, предшествующая сильному импульсному движению, с телом, перекрывающим предыдущую свечу (поглощение).

2. **Как математически определить бычий OB?**  
   *Ответ:* `close[i] > open[i] && close[i] > close[i-1] && open[i] < close[i-1]`.

3. **Как математически определить медвежий OB?**  
   *Ответ:* `close[i] < open[i] && close[i] < close[i-1] && open[i] > close[i-1]`.

4. **Какое минимальное подтверждение рекомендуется для входа на ретесте OB?**  
   *Ответ:* Свечной паттерн на границе и CHoCH.

5. **Почему OB работает как поддержка/сопротивление?**  
   *Ответ:* Это зона, где крупные игроки размещали свои ордера; возвращаясь туда, они часто защищают позиции.

6. **Какой минимальный R/R рекомендуется для сделок по OB?**  
   *Ответ:* Не менее 1:2.

7. **В каких сессиях OB наиболее надёжен?**  
   *Ответ:* Лондонская (10-12) и Нью-Йоркская (15-17).

8. **Чем отличается Unmitigated OB от обычного?**  
   *Ответ:* Unmitigated OB – это OB, который ещё ни разу не тестировался ценой. Первый ретест обычно самый сильный.

9. **Что произойдёт, если цена пробьёт OB насквозь?**  
   *Ответ:* OB превращается в Breaker Block (BB) и может стать сопротивлением/поддержкой в противоположном качестве.

10. **Какой минимальный размер OB рекомендуется для торговли?**  
    *Ответ:* Ширина зоны должна быть не менее 5-10 пунктов (для 5-знака), иначе стоп будет слишком узким.
